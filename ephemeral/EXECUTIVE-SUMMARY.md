## The 24 Generative Questions (Overview)

### Stage 1: Density Dimensions (6 questions)
What if information density could follow natural metaphors?
- PULSE: Can pages breathe? (alternating dense/sparse)
- CRESCENDO: Can density build progressively? (sparse→dense)
- ISLANDS: Can content exist as isolated clusters? (dense islands in sparse sea)
- GEOLOGICAL: Can density be stratified? (4-layer metaphor: Atmosphere/Surface/Topsoil/Subsoil)
- TIDAL: Can information flow like rivers? (variable width channels)
- FRACTAL: Can density be self-similar at 4 scales? (page/section/component/character)

### Stage 2: Organizational Dimensions (6 questions)
What if organizational structure generates density naturally?
- CONVERSATIONAL: What if docs were Q&A conversations? (generates PULSE)
- NARRATIVE: What if docs were story arcs? (generates CRESCENDO)
- TASK-BASED: What if docs were discrete tasks? (generates ISLANDS)
- CONFIDENCE: What if docs were stratified by certainty? (generates GEOLOGICAL)
- SPATIAL: What if docs were hub-spoke maps? (generates WAVE)
- CREATIVE: Are organization and density the SAME phenomenon?

### Stage 3: Axis Dimensions (6 questions)
What if reading patterns could serve organizational patterns?
- Z-PATTERN: Does diagonal sweep serve conversational flow? (temporal equivalence)
- F-PATTERN: Does scan degradation serve crescendo density? (proportional equivalence)
- BENTO GRID: Can grid structure contain islands? (cell hierarchy equivalence)
- SPIRAL: Can spirals be geological? (golden ratio nesting equivalence)
- CHOREOGRAPHY: Does direction serve wave-islands rhythm? (kinetic equivalence)
- COMPOUND: Does meta-equivalence unify all mechanisms?

### Stage 4: Combination Dimensions (6 questions)
Can multiple patterns coexist at scale?
- REASONING INSIDE CODE: Can 3 components + dual axis transitions work?
- TASK CONTAINING DECISION: Can matrices embed as natural pauses?
- FILE TREE WITH CALLOUTS: Can components share visual DNA?
- ESSENCE AS BACKGROUND: Can elements pervade without depth?
- MULTI-AXIS TRANSITION: Can 3 axis patterns coexist sequentially?
- PILOT MIGRATION: Can the system explain itself end-to-end?

## The Critical Discovery: Two Different Question Types

### TC (Tension-Composition) Skill Asks:
**"What does the reader FEEL/UNDERSTAND/NEED?"**

14 different axes, all focused on the reader's internal experience:
- Emotional states (FEEL, RECONCILE, BECOME)
- Cognitive states (UNDERSTAND, EVALUATE, COMPARE)
- Behavioral states (DO, DISCOVER, NAVIGATE)
- Orientational states (TRUST, ORIENT, SEE)

**Output:** Metaphor + emotional description
**Example:** "Refinery cross-section" -- implies structure but doesn't specify it

### Exploration Questions Ask:
**"What if information was organized LIKE THIS?"**

Systematic progression through 4 structural dimensions:
- Phenomenological (what metaphors work?)
- Organizational (what cognitive structures work?)
- Geometric (what axis patterns serve them?)
- Integrative (can all these coexist?)

**Output:** Pre-specified architecture with testable predictions
**Example:** "4 confidence strata with inverse density-certainty mapping, GEOLOGICAL rhythm, fractal at 4 scales"

## The Gap: Structural Propositions

### What TC Produces
- "The reader should feel like they're moving through a controlled industrial process"
- "Stacked operational levels from foundation to output"
- "Progressive disclosure of complexity"

### What TC Does NOT Produce
- "Density correlates inversely with confidence"
- "Established content gets sparse density, speculative gets dense"
- "Conversational structure generates PULSE naturally"
- "Fractal self-similarity at 4 scales: page, section, component, character"

### This Missing Category is Called: STRUCTURAL PROPOSITIONS

A structural proposition is: "THIS content property maps to THIS visual property, producing THIS outcome."

**TC tells the builder HOW IT SHOULD FEEL.**
**Explorations tell the builder WHAT IT SHOULD BE.**
**Nothing tells the builder HOW FEELING BECOMES BEING.**

---

## Why This Matters: The Richness Gap

### Builder with TC Brief Only
"The reader should feel gradual disclosure"
→ Builder thinks: "I'll add a heading and some spacing"
→ Result: Adequate but conventional

### Builder with Exploration Framework
"Implement OD-004 Confidence structure: 4 strata (established/probable/speculative/open) with inverse density-to-certainty mapping"
→ Builder thinks: "For speculative, tight 3px borders and high density. For established, wide spacing and sparse layout"
→ Result: Inherently varied and rich because the RULE generates different output per input

**The specificity difference explains why exploration-era pages score higher.** Not because explorations are better, but because they pre-constrain the architecture, forcing variation.

## The Five Structural Differences

| Dimension | TC Questions | Exploration Questions |
|-----------|--------------|----------------------|
| **Subject** | Reader-centric | Structure-centric |
| **Framework** | Experiential/emotional | Architectural/spatial |
| **Direction** | Bottom-up (content→feeling) | Top-down (research→hypothesis) |
| **Specificity** | Metaphor implies structure | Metaphor IS structure |
| **Knowledge** | Fresh each time | Cumulative across stages |

## Key Insights

### 1. Exploration Questions Are Not Creative Prompts
They are systematic research instruments designed to explore a design space incrementally. Each question:
- References specific prior findings
- Makes falsifiable predictions
- Builds on predecessor discoveries
- Tests specific mechanisms

### 2. TC and Explorations Operate on Different Axes
Not competing -- complementary. TC is essential for emotional conviction. Explorations are essential for architectural certainty. Both needed; neither sufficient.

### 3. The Missing Bridge Is Structural Propositions
TC → Metaphor → (MISSING: explicit structural rules) → Exploration framework → CSS

The gap is where "warm" becomes "wide spacing" or "industrial" becomes "3px borders."

### 4. Specificity Drives Richness
Pre-specified architecture (explorations) forces variation because rules generate different outputs for different inputs. Open-ended metaphor (TC) leaves room for homogeneity because the builder must independently infer constraints.

### 5. Cumulative Knowledge Compounds Quality
DD→OD→AD→CD progression produces emergent properties (equivalence mechanisms, meta-unification) that couldn't emerge from 4 independent question sets.
