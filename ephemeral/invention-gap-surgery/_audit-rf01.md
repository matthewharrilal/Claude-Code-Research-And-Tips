## Categories still missing:
- **Self-Aware Components (Cat 2):** MISSING. Section 10 has one structural question family ("Structural Self-Awareness" — minimap/progress/annotation from DD-006, OD-006) but this is a 0-1 optional question, not a mechanism pathway. The 7 self-aware inventions (CSS-only scroll witness via @property, fractal annotation layer with writing-mode, geological minimap, visual index, recursive TOC with font cascade, confidence switcher, chapter progress dots) have no upstream agent that would generate them. S3 could extract the structural logic from OD-006 case study, but the CSS techniques (@property, animation-timeline, writing-mode) remain absent from the pipeline's technical vocabulary.
- **Advanced CSS Techniques (Cat 3):** MISSING. The 5 advanced CSS inventions (animation-timeline: view(), @property, clip-path: polygon(), sticky section headers, 3-column page grid) have zero representation in any pipeline file. No specialist prompt, synthesizer section, or builder instruction mentions these CSS features. The pipeline's CSS vocabulary is bounded by what tokens.css and components.css contain — advanced features require either mechanism catalog expansion or explicit builder permission to use modern CSS.

## Specific gaps:
1. **CSS vocabulary boundary:** The pipeline has no mechanism for introducing CSS features not already in tokens.css / components.css / mechanism catalog. Even with perfect structural propositions, a builder cannot deploy `animation-timeline`, `@property`, `clip-path`, or `writing-mode` because these are outside the known vocabulary. Section 11 blueprint CSS could theoretically include them, but the synthesizer has no upstream source for these techniques (S3 case studies don't catalog raw CSS features, only structural logic).
2. **Self-aware components require per-page invention:** Minimaps, scroll witnesses, and visual indexes are inherently bespoke — each page needs its own self-referential component. No upstream agent can prescribe "build a minimap of your own page structure." This is a category that resists pipeline systematization; it requires builder-level creative autonomy that the pipeline's current TEXTURE-vs-STRUCTURE distinction doesn't accommodate (these are neither texture mechanisms nor structural layouts — they are meta-components).
3. **Section 10 structural questions are 0-1 optional:** The "Structural Self-Awareness" family is the only pathway to Cat 2 inventions, but it competes with 4 other structural families for a single 0-1 slot. In practice, most builds will select 0 structural questions (since most content is Linear Narrative), leaving Cat 2 entirely unaddressed.
4. **S3 extracts process, not CSS technique:** Specialist 3 extracts structural LOGIC and decision SEQUENCES from case studies, but explicitly avoids copying CSS (line 339: "You are NOT a CSS copier"). The inventions in Cat 2 and Cat 3 ARE their CSS — the technique IS the invention. Process extraction misses them.

## Impact: MAJOR
The pipeline now has genuine pathways for Spatial Organization (Cat 1) and partial coverage for Content-as-Form (Cat 4) and Compound Components (Cat 5) — this is a significant improvement from the catalog's finding of 0/29 producible. However, Self-Aware Components (Cat 2, 7 inventions) and Advanced CSS (Cat 3, 5 inventions) remain structurally unreachable — 12 of 29 inventions (41%) have no pipeline pathway. The gap is not prompt wording but architectural: the pipeline lacks a CSS vocabulary expansion mechanism and a meta-component generation layer.
