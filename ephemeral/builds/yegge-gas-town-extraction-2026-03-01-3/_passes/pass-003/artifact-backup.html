"Claude Code is the world's biggest fuckin' ant. Everyone is focused on making their ant run longer… when nature prefers colonies."

Steve Yegge

This is the fundamental shift Gas Town represents:

Traditional Approach

  • One Big Ant
  • Make it run longer
  • Context window limits everything
  • You are the programmer

Gas Town Approach

  • Ant Colony
  • Build the factory
  • Multiple workers share the load
  • You are the Product Manager

Gas Town isn't an agent — it's a factory for agents:

Role Transformation
Traditional Dev Gas Town
You write code You file features
You debug Agents debug
You're the implementer You're the PM

Section 03 · Architecture

The 8 Roles Architecture

The colony is organized into eight specialized roles across three tiers. Town-level roles govern the entire settlement — one instance each, always running. Rig-level roles operate within individual projects — multiple instances, project-scoped. And you, the Overseer, sit above it all with your own inbox and identity in the system.

Role Deep Dives

Town Level — Global, One Instance Each
01
Mayor
Town-Level Coordination

Function: Chief coordinator — your concierge and chief-of-staff

  • Receives high-level directives from Overseer
  • Routes work to appropriate rigs
  • Manages cross-rig coordination
  • Handles town-level decisions
  • Primary AI coordinator for the entire town
gt mayor up # Start mayor gt mayor status # Check mayor state gt mayor down # Stop mayor
Model: Opus 4.5
02
Deacon
Monitoring / Handshakes

Function: Health monitoring and agent coordination

  • Monitors agent health across the town
  • Handles agent-to-agent handshakes
  • Reports anomalies to Mayor
  • Tracks work progress
  • Manages handoff protocols and worker recycling
gt deacon up # Start deacon gt deacon status # View monitoring state
Model: Haiku
03
Dogs
Quality Gates / Watchdogs

Function: Quality assurance and safety gates — town-level grunts

  • Reviews work before acceptance
  • Enforces quality standards
  • Guards against regressions
  • Can reject/rollback work
  • Stale branch cleanup
gt dogs up # Start dogs gt dogs bark # Trigger quality check
Model: Sonnet
Rig Level — Per-Project, Multiple Instances
04
Refinery
Task Decomposition

Function: Breaks down large tasks into agent-sized work

  • Receives feature requests from Mayor
  • Decomposes into smaller, atomic tasks
  • Assigns to Polecats or Crew
  • Tracks decomposition progress
  • Manages the merge queue with sequential rebasing
gt refinery up <rig> # Start refinery for rig gt refinery decompose <task> # Manually trigger decomposition
Model: Opus 4.5
05
Polecat
Named Persistent Workers

Function: Named, persistent agent workers with specializations

  • Execute assigned tasks
  • Maintain context across sessions
  • Specialize in rig-specific knowledge
  • Report completion to Refinery
  • Ephemeral but can be persistent workspaces
gt polecat up <rig> alice # Start polecat named "alice" gt polecat up <rig> bob # Start polecat named "bob" gt polecat list <rig> # List polecats for rig gt polecat assign alice <task> # Assign task to alice

Naming convention: Human names (alice, bob, carol) help with mental model

Model: Sonnet
06
Witness
Per-Rig Observer

Function: Observes and records rig activity — watches over polecats

  • Watches all work in assigned rig
  • Records history and context
  • Provides state to new workers
  • Reports anomalies to Deacon
  • Unsticks drifting agents
gt witness up <rig> # Start witness for rig gt witness report # Get current state report
Model: Haiku
Worker Level — Ephemeral, Task-Specific
07
Crew
Ephemeral Workers

Function: Short-lived, task-specific burst workers

  • Execute single tasks
  • No persistent state
  • Cheap and disposable
  • Used for parallel bursts
  • Spawn, complete, disappear
gt crew spawn <rig> --task "fix this bug" gt crew count <rig> # Count active crew gt crew purge <rig> # Kill all crew
Model: Haiku
Tip

Crew members are the cheapest way to parallelize work. When a Refinery decomposes a feature into 10 tasks, Crew workers can execute all 10 simultaneously. They cost nothing to spin up and nothing to throw away. Use Haiku for Crew — simple, isolated tasks don't need expensive reasoning.

Human Level — You
08
Overseer
Human Boss — YOU

Responsibilities

  • Define features and priorities
  • Review inbox decisions
  • Approve major changes
  • Provide human judgment
  • Has identity in the system with own inbox

Interface

  • Human inbox for decisions
  • tmux panes for observation
  • gt CLI for commands
  • Can send and receive town mail

"That's you, Human. As the Overseer, you have an identity in the system, and your own inbox, and you can send and receive town mail."

Steve Yegge

Essence

The eight roles are not eight programs. They are eight jurisdictions. Each has authority over its domain and communicates through the town mail system. When you understand this, you stop thinking about agents and start thinking about governance.
