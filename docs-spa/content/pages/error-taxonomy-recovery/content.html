Essence (15 words)
Errors are either deterministic (fix once) or non-deterministic (layer defenses). Prevention beats recovery.

Each category has distinct symptoms, causes, and recovery patterns.

Context Window Errors (Most Common)

Context errors account for the majority of quality degradation. The 200K token window seems large, but quality degrades starting at 60-70% capacity.

Context Overflow (API Error 400)
Symptom: API Error 400, truncated outputs, CLAUDE.md ignored
Threshold: 85-95% context = significant degradation, 95%+ = unrecoverable
Recovery: Session cannot continue. Save state, exit, start fresh.
Context Rot (Silent Quality Degradation)
Symptom: Responses become generic, errors compound, claims vs reality diverge
Quote: "Beyond three turns, AI becomes unreliable without proper memory, summarization, and state handling."
Recovery: /compact or fresh session. External memory (progress.txt) prevents recurrence.

Validation Errors (Most Dangerous)

Features Marked Complete Without Testing
Symptom: Tasks marked "done" without verification. No test output in logs. Features fail in production.
Quote from Anthropic: "Claude's tendency to mark features complete without proper testing... did much better once explicitly prompted to use browser automation."
Prevention: Always include exit code verification. Add Playwright for visual verification.

3 Deterministic vs Non-Deterministic

DETERMINISTIC FAILURES
Given the same inputs, these failures will occur every time. The cause is identifiable and reproducible. Examples: Context overflow, rate limits, file deletion, merge conflicts, permission denied.
Recovery Strategy
1. Identify exact trigger condition
2. Fix root cause (not symptoms)
3. Add prevention mechanism
4. Verify fix by reproducing scenario
NON-DETERMINISTIC FAILURES
These failures may or may not occur given the same inputs. They depend on model state, timing, or accumulated context. Examples: Quality degradation, false test passes, verification loops, memory injection.
Recovery Strategy
1. Add multiple checkpoints
2. Implement verification layers
3. Use fresh context when possible
4. Design for graceful degradation
5. Accept some failure rate, plan for retry

4 6 Recovery Patterns

Each pattern addresses a specific failure mode. Master these and you can recover from anything.

Pattern 1: Restart with Fresh Context
When: Context rot, quality degradation, API 400 errors
Key: Save state to files first. git stash, then exit, then restart.
Insight: "Fresh context beats accumulated context. When in doubt, start over."
Pattern 2: Resume from External State
When: Session interrupted, progress lost between iterations
Key: State lives in prd.json, progress.txt, git. Not in conversation.
Command: cat prd.json | jq '.userStories[] | select(.passes == false)'
Pattern 3: Escalate to Simpler Model
When: Complex task failing repeatedly, budget constraints
Key: Haiku fails -> Sonnet. Sonnet fails -> Opus. Opus fails -> Human.
Cost: Haiku $0.25/1M, Sonnet $3/1M, Opus $15/1M
Pattern 4: Decompose Further
When: Task too complex, stuck loop (no commits for 3+ iterations)
Rule: If you can't describe it in 2-3 sentences, split it.
Size: 100-500 lines of code change per task. 1-3 files modified.
Pattern 5: Rollback and Retry
When: File corruption, wrong approach taken
Single file: git checkout -- path/to/file.ts
Nuclear: git reset --hard HEAD && git clean -fd
Pattern 6: Circuit Breaker
When: Preventing infinite loops, cost containment
Thresholds: Phase attempts: 3-5. Consecutive failures: 5. Error signature repeat: 3.
From Marathon Ralph: STOP_HOOK_ITERATIONS=25, CONSECUTIVE_FAILURES=5
# Circuit breaker implementation
STUCK_THRESHOLD=3
if [ "$CURRENT_COMMIT" = "$LAST_COMMIT" ]; then
  STUCK_COUNT=$((STUCK_COUNT + 1))
  if [ $STUCK_COUNT -ge $STUCK_THRESHOLD ]; then
    echo "Circuit breaker triggered"
    exit 1
  fi
fi

6 The Horror Stories

These actually happened. Learn from others' expensive mistakes.

The $47,000 Recursive Loop
What happened: Multi-agent research tool slipped into recursive loop. Ran for 11 days before anyone noticed.
Total cost: $47,000 in API charges
Prevention: MAX_ITERATIONS always set. timeout 3600 ./ralph.sh 50. Cost monitoring dashboards.
Home Directory Deletion
What happened: Claude executed rm -rf tests/ patches/ plan/ ~/ - deleted user's entire home folder.
GitHub Issue: #17164
Prevention: Never use --dangerously-skip-permissions without blocklist. Add rm -rf to blocked patterns.
Production Database Deletion
What happened: Replit deleted production database containing 1,200+ executives' data.
Root cause: AI agent given too much access without proper safeguards.
Prevention: Never give production database credentials to AI agents. Use read-only replicas.
Drive Deletion (Google Antigravity)
What happened: rmdir /s /q d:\ - Deleted entire D: drive.
Context: Internal Google test of code-generating AI.
Lesson: Sandbox all destructive operations. Never trust generated commands.
Credentials in Context
Symptom: API keys visible in conversation, .env contents displayed
Root cause: No .claudeignore file. Sensitive files readable.
Recovery: ROTATE ALL EXPOSED CREDENTIALS IMMEDIATELY. Add .claudeignore.

7 Mental Models for Error Handling

THE CHECKPOINT CLIMBER
Mountain climbing with fixed checkpoints. Each checkpoint is a git commit. If you fall (error), you only fall back to the last checkpoint, not to the base.
Application
Commit early and often. Each working state is a checkpoint. Rollback is cheap and fast. Never climb too far without a checkpoint. "If something works, save it with a git commit."
THE CIRCUIT BREAKER
Electrical circuit breakers prevent house fires by cutting power when current exceeds safe levels. Claude Code circuit breakers prevent cost fires by stopping execution when failures exceed safe levels.
Application
CLOSED (normal) -> failures counted -> OPEN (protection) -> cooling period -> HALF-OPEN (testing) -> success? CLOSED. Set failure thresholds before starting. When exceeded, stop immediately.
THE IMMUNE SYSTEM
Your body has multiple layers of defense: skin, immune cells, antibodies, fever response. Claude Code error handling should have multiple layers too.
Defense in Depth
Layer 1: Prevention (.claudeignore, blocklist, CLAUDE.md)
Layer 2: Detection (Claude HUD, stuck detection, cost monitoring)
Layer 3: Response (circuit breakers, rollback, fresh context)
Layer 4: Recovery (state restoration, resume capability)
THE ASSEMBLY LINE INSPECTOR
Factory assembly lines have quality inspectors at each station. They catch defects early before they compound.
Application
Verify at each stage, not just at the end. Plan -> typecheck -> test -> integration -> deploy. RULE: Never proceed to next stage with verification failure. Small defects caught early = cheap. Large defects caught late = expensive.
