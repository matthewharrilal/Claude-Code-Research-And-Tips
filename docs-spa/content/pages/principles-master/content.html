Principles Index
The physics of agent engineering: 8 core principles from which all patterns derive. Understanding principles enables pattern creation, not just pattern usage.

1 What Are Principles?

Principles are the "physics" of Claude Code agent engineering. They explain WHY patterns work, not just WHAT they are. When you understand principles, you can:

  • Create new patterns for novel constraints
  • Diagnose why existing patterns fail
  • Adapt patterns to different contexts
  • Make architectural decisions with confidence

The 8 core principles were distilled from 40+ research extractions, community innovations, and production implementations. They form a hierarchy where Principle 1 (Context Is Finite) is the root constraint from which everything else flows.

4 The Eight Principles

# Principle Core Insight
P1 Context Is Finite All patterns optimize around limited context windows (~200K nominal, ~100K effective)
P2 External State > Internal Memory Files persist, context evaporates. Memory is filesystem, not conversation.
P3 Fresh Context > Extended Sessions New instances beat long conversations. Compaction is lossy, files are not.
P4 Separation of Concerns Orchestrators coordinate, workers execute. Never mix planning and doing.
P5 Atomic, Verifiable Tasks Complete in one iteration, verify before commit. 2-3 sentence test.
P6 Quality Gates Before Commit Tests must pass, no broken code inheritance. CI green mandate.
P7 Isolation Prevents Contamination Own your space, don't conflict. Worktrees, branches, containers.
P8 Explicit Communication File-based handoffs, not implicit coordination. JSON over conversation.

Meta-Principle: When in doubt, externalize, isolate, verify, and restart fresh.
