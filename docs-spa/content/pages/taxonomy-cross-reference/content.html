3 The 10 Major Themes

These are the cross-cutting concerns that appear across ALL patterns in the knowledge base. Understanding these themes gives you the mental models to understand any new pattern.

THEME 1: CONTEXT AS PRIMARY CONSTRAINT
The single most universal theme. Context window management shapes every pattern. Every successful pattern addresses context management differently.
4 Approaches
Panopticon: Domain isolation (8 separate directories)
Ralph: Fresh context per iteration
CC Mirror: Orchestrator/worker separation
Claude-Mem: Semantic compression to summaries
THEME 2: FRESH CONTEXT PER ITERATION (THE RALPH PRINCIPLE)
The discovery that wiping context and starting fresh produces better results than maintaining long conversations. Counterintuitive but validated across multiple practitioners.
Key Sources
@ryancarson (PRD-driven Ralph), @mattpocockuk (tutorials + 11 tips), @arvidkahl ("We learn from failure"), @ghuntley (original inventor)
THEME 3: FILE-BASED STATE (FILESYSTEM AS DATABASE)
Using the filesystem rather than conversation memory for state persistence. Git history becomes memory. Files become the source of truth.
File Types and Purposes
prd.json = Task tracking (passes: true/false)
progress.txt = Append-only learnings
AGENTS.md = Permanent documentation
CLAUDE.md = Project context and rules
THEME 4: VERIFICATION LOOPS (NON-NEGOTIABLE QUALITY GATES)
The consensus that feedback loops are essential for production-quality AI-generated code. Without verification, errors compound across iterations.
Verification Hierarchy
1. Typecheck - npm run typecheck (minimum)
2. Unit tests - Core logic validation
3. Integration tests - API contracts
4. E2E browser tests - Playwright/dev-browser
5. Human review - Security, architecture decisions
THEME 5: ORCHESTRATOR/WORKER SEPARATION
The architectural pattern of separating coordination from execution. The orchestrator decides WHAT to do; workers execute HOW.
The Iron Law (CC Mirror)
"YOU DO NOT WRITE CODE. YOU DO NOT RUN COMMANDS. YOU DO NOT EXPLORE CODEBASES. You are the CONDUCTOR. Your agents play the instruments."

Pattern Conflicts

Long context sessions vs Ralph fresh-context
Conflict: Mutually exclusive philosophies
Resolution: Pick one approach. Don't mix in the same workflow.
Subagent recursion vs Worker preamble rules
Conflict: CC Mirror workers must NOT spawn sub-agents
Resolution: Keep workers flat. Only orchestrator spawns tasks.
Serial execution vs Parallel agents
Conflict: Merge conflicts on shared files
Resolution: Use git worktrees or file-locking patterns.

The Fundamental Challenge
The tension: Community documentation evolved organically. Different authors invented different terms for identical concepts.
Why it's hard: You might search for "worker" and miss relevant docs about "crew" or "polecat".
Mitigation: Use this cross-reference. Search for multiple synonyms. Check the Author Expertise Map for who uses which terms.
