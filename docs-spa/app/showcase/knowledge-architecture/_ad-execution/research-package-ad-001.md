## 5. EXT-AXIS-Z FINDINGS (Bespoke External Research)

### EXT-AXIS-Z-001: Z-Pattern Diagonal as Implicit Grid Line

**Description:** In graphic design (Josef Muller-Brockmann's grid systems), the diagonal is not random — it follows an implicit grid line from top-left to bottom-right that creates mathematical balance. When the Z-pattern's diagonal follows a specific angle (e.g., the golden angle ~38.2 degrees from horizontal), it creates harmonic proportions between the four Z quadrants.

**Soul Test:** PASS — Pure 2D geometric principle. No CSS depth, shadows, or rounded corners. Diagonal is a compositional guide, not a rendered element.

**Signature Elements:**
1. `grid-template-rows: 1.618fr 1fr` — golden ratio between Z-line heights creates balanced quadrants
2. Implied diagonal at ~38.2 degrees creates harmonic Z-quadrant proportions
3. No visible diagonal line — the diagonal is implied by content placement weight

### EXT-AXIS-Z-002: Z-Pattern Velocity Control via Typography Weight

**Description:** Eye movement speed along the Z-pattern can be controlled via typographic weight. Bold/heavy type at Z-entry points (top-left, bottom-left) slows the eye, creating PULSE pauses. Light body text at Z-traverse points (top-right, bottom-right) allows faster scanning. This maps directly to PULSE: heavy type = dense zone (slow), light type = sparse zone (fast).

**Soul Test:** PASS — Typography-based hierarchy (IS-2). Uses locked fonts (Instrument Serif, Inter). No decoration needed. Aligns with IS-1 (hierarchy through SIZE, COLOR, POSITION).

**Signature Elements:**
1. Questions in `font-weight: 600` + `font-family: 'Instrument Serif'` at Z-entry (slow/heavy)
2. Answers in `font-weight: 400` + `font-family: 'Inter'` at Z-traverse (fast/light)
3. Typography weight shift at each Z quadrant transition

### EXT-AXIS-Z-003: Z-Pattern Breakpoint Collapse Strategy

**Description:** Z-pattern geometry must degrade gracefully at narrow viewports. Below 768px, the Z collapses to a linear stack (natural mobile reading). The PULSE rhythm is preserved because PULSE is inherently vertical (question/answer alternation is top-to-bottom regardless of viewport width). The TIDAL width variation collapses: questions go from 60% to 100% width, but density differentiation shifts to background color instead.

**Soul Test:** PASS — Responsive design principle. No soul implications. Zone tokens (--color-zone-sparse, --color-zone-dense) maintain density signals when width variation is unavailable.

**Signature Elements:**
1. `@media (max-width: 768px) { .question { max-width: 100%; background: var(--color-zone-sparse); } }` — width collapse with color compensation
2. Z-pattern becomes linear stack below breakpoint — PULSE survives, TIDAL transfers to color
3. Desktop: width=TIDAL signal. Mobile: background-color=TIDAL signal.

### EXT-AXIS-Z-004: Z-Pattern as Conversational Turn-Taking

**Description:** Conversational analysis (Sacks, Schegloff & Jefferson 1974) identifies turn-taking as the fundamental structure of conversation. Each Z-cycle maps to one conversational turn: the question is the "current speaker" (left-positioned, narrow, distinct), and the answer is the "respondent" (full-width, detailed). The Z-diagonal is the conversational "transition relevance place" — where turn-taking occurs.

**Soul Test:** PASS — Content-level structural pattern, soul-neutral. Maps conversational analysis theory to Z-pattern geometry without requiring any CSS decoration.

**Signature Elements:**
1. Questions left-positioned with visual distinction (Instrument Serif, --color-zone-sparse bg)
2. Answers full-width with detailed content (Inter, --color-zone-dense bg)
3. 32px+ gap at each Q→A transition = turn-taking boundary
