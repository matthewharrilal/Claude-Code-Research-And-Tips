AD-003: Bento Grid

What if axis geometry used a modular grid of varying cell sizes to spatially encode the ISLANDS density pattern? Does the bento grid's rigid structure fight or contain the discontinuous spacing that ISLANDS demands?

Building with Bento Grid Layouts

The bento grid pattern uses CSS Grid with varied cell sizes to create modular, self-contained documentation layouts. Each grid cell is a discrete task island — its size signals its importance, its position signals its complexity level, and the grid gaps between cells form the sparse ocean of the ISLANDS density pattern.

This guide walks through implementing bento grid layouts for task-based documentation, progressing from single cells to compound grid compositions.

Getting Started

Begin with the foundational grid setup. Each cell below is a self-contained task — complete them in any order.

Beginner
Create the Grid Container Beginner
~5 min

The bento grid begins with a container element using display: grid. The 4-column layout with fractional units creates equal-width cells that form the structural backbone.

View implementation steps
  1. Define the grid container with display: grid and set grid-template-columns: repeat(4, 1fr) for equal columns.
  2. Set the gap using a spacing token: gap: var(--space-6) creates the 24px ocean between task islands.
  3. Add grid items as child elements. Each child automatically becomes a 1x1 cell occupying one column and one row. 
/* EXT-AXIS-B-001: Bento grid as ISLANDS spatial encoding */
.bento-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: var(--space-6); /* 24px — the sparse ocean */
}
Set Cell Borders Beginner
~3 min

Every cell uses border: 3px solid for Cat 1 structural framing. No 2px borders. The sharp edges are non-negotiable.

View border rules 
.bento-cell {
  border: 3px solid var(--color-border);
  border-radius: 0; /* Soul Piece #1 */
  box-shadow: none; /* Soul Piece #4 */
}
Apply Zone Tokens Beginner
~3 min

Use zone tokens for backgrounds. Dense zones use #FFFFFF, sparse zones use #FEF9F5, breathing zones use #FAF5ED.

Configure Typography Tokens Beginner

Cell Size and Hierarchy

Cell size is the primary hierarchy signal in a bento grid. A 2x2 cell commands four times the visual weight of a 1x1 cell. This is IS-1 (hierarchy through SIZE) — no decoration needed.

Essence
The bento grid does not decorate hierarchy — it embodies it. A large cell is not marked "important" with a badge or color. It IS important, because it occupies more space. The space is the signal. The signal is the space.
Intermediate
Span Syntax for Cell Sizing Intermediate
~8 min

Use grid-column: span N and grid-row: span N to create cells of varying sizes. The span value directly maps to the importance of the content.

View span patterns 
/* R4-020: Large = big deal */
.bento-item--large {
  grid-column: span 2;
  grid-row: span 2;
}

/* Wide content strip */
.bento-item--wide {
  grid-column: span 2;
}

/* Tall sidebar content */
.bento-item--tall {
  grid-row: span 2;
}
Tip

The largest cell in each grid section should contain your most critical content. Size IS hierarchy — let the grid do the work.

Gap as Density Intermediate
~5 min

The grid gap is the sparse ocean between dense island cells. Vary gaps between sections to create meta-ISLANDS structure. 

/* 3-level gap hierarchy */
gap: var(--space-4);  /* tight */
gap: var(--space-6);  /* standard */
gap: var(--space-8);  /* wide */
Breaking the Grid Intermediate
~6 min

A featured 2x2 cell that spans a grid boundary creates intentional disruption: "this task is special." The break is structural, not decorative.

Compound Grid Patterns

Advanced bento layouts combine multiple grid techniques — nested grids within cells, responsive collapse strategies, and compound density patterns where ISLANDS and BOOKENDS work simultaneously at different scales.

Advanced
Responsive Grid Collapse Advanced
~12 min

At viewport widths below 768px, all multi-column grids collapse to single column. This is a binary rule from AD-CONVENTION-SPEC Section 12.5. The span classes reset to span 1 so cells stack vertically.

View responsive implementation

Desktop (>768px): Full 4-column bento grid with varied cell sizes. The spatial hierarchy is visible — large cells command attention.

Mobile (<768px): Single-column stack. Hierarchy shifts from SPATIAL (cell size) to SEQUENTIAL (reading order). Source order matters.

@media (max-width: 768px) {
  .bento-grid {
    grid-template-columns: 1fr;
  }

  .bento-item--large,
  .bento-item--wide {
    grid-column: span 1;
    grid-row: span 1;
  }
}
Nested Grid Cells Advanced
~8 min

A bento cell can contain its own sub-grid for internal organization. The outer grid defines the macro ISLANDS; the inner grid creates micro-islands within each cell. 

.bento-cell {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: var(--space-4);
}
Named Grid Areas Advanced
~10 min

Use grid-template-areas for semantic grid definitions. Named areas make the layout self-documenting and easier to rearrange. AD-CONVENTION-SPEC Section 12.1 requires named areas for structural layout. 

grid-template-areas:
  "hero  hero  side"
  "main  main  side"
  "foot  foot  foot";
Compound Density: ISLANDS Inside BOOKENDS Advanced
~15 min

The bento grid creates a compound density pattern: BOOKENDS at the page level (dense header + dense verification section bracketing sparse middle) with ISLANDS at the section level (discrete task cells separated by grid gap ocean). This dual-scale density is the central structural insight of bento grid layouts.

View density structure
Key Pattern

Page scale: Dense header (bookend) → Sparse grid ocean → Dense verification (bookend)
Section scale: Dense task cells (islands) separated by grid gaps (ocean)
Cell scale: Dense content blocks separated by internal padding (micro-ocean)
Character scale: Dense code logic alternating with sparse comments

Verification Checklist

After implementing a bento grid layout, verify these requirements. Each is a binary pass/fail check.

Verify

  • [✓] border-radius: 0 on all grid cells — no rounded corners
  • [✓] box-shadow: none on all elements — no shadows
  • [✓] All cell backgrounds fully opaque — no rgba() with alpha < 1.0
  • [✓] Cell borders are Cat 1 (3px+) — no 2px borders
  • [✓] Grid gap uses spacing tokens — no arbitrary pixel values
  • [✓] Grid collapses to single column below 768px

Reflect

The bento grid answers a structural question about ISLANDS: does a rigid grid fight or contain discontinuous density? The answer from this exploration is that the grid contains islands. The grid cells ARE the islands. The grid gaps ARE the ocean. The rigidity of the grid provides the structural discipline that prevents islands from drifting into visual chaos, while the varied cell sizes preserve the discontinuous, non-uniform density that ISLANDS demands.
