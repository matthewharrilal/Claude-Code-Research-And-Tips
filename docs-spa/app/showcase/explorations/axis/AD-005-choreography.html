Soul-Compliant Transitions

Choreographic movement in the KortAI system is structural, not visual. The reader's journey through content is paced by spacing changes, typography scale shifts, and background zone alternations — never by CSS animation that simulates physical movement. This distinction is the core of soul-compliant choreography: the content choreographs itself through its own density rhythm.

The allowed transition set is deliberately narrow. Each transition must pass the ANTI-PHYSICAL gate: no scale transforms (implies z-depth), no rotation (implies physical manipulation), no hover-lift (implies object floating), no blur (implies material depth). What remains is a focused vocabulary of flat, honest state changes.

Allowed Transition Properties

The following CSS transitions are permitted within choreographic layouts. Each has been tested against the soul compliance checklist and the 20/20 SOUL FAIL analysis from the research pipeline.

Scroll-Triggered Reveal Implementation

The "arriving wisdom" pattern allows content to appear as the reader scrolls to it. The element starts hidden and transitions to visible once, then stays. The prefers-reduced-motion prophylactic is mandatory: users who prefer reduced motion see all content immediately without animation. 

/* Scroll-triggered reveal — fires ONCE, then stays */
/* Progressive enhancement: visible by default */
.reveal-on-scroll {
  opacity: 1;
  transform: translateY(0);
}

/* When JS is available, prepare for animation */
.js-enabled .reveal-on-scroll {
  opacity: 0;
  transform: translateY(20px);
  transition:
    opacity 0.3s ease,
    transform 0.3s ease;
}

/* Revealed state — set by IntersectionObserver */
.js-enabled .reveal-on-scroll.revealed {
  opacity: 1;
  transform: translateY(0);
}

Notice the progressive enhancement: without JavaScript, content is fully visible. The .js-enabled class is added to the document root only when the IntersectionObserver API is available. This ensures content is accessible regardless of JavaScript state.

Caution

Scroll-triggered animations fire ONCE. No repeat, no reverse. The element must remain in its final state (opacity: 1, translateY: 0) permanently. Animations that replay on scroll-out-and-back violate the ANTI-PHYSICAL identity by treating content as a physical object that can appear and disappear.

Prohibited Transition Properties

These CSS transitions are banned in all choreographic layouts. Each implies physical object behavior that contradicts the flat, editorial ANTI-PHYSICAL identity. The prohibition is absolute — no edge cases, no exceptions. 

/* PROHIBITED — every line here is a soul violation */

/* Implies z-depth change (3D space) */

/* Implies physical object manipulation */

/* Implies lift/float — treats content as button */

Essence

Choreographic movement is the absence of movement. The content arranges itself through density variation, spacing rhythm, and axis direction — the reader experiences a journey without anything ever literally moving. The flattest possible surface creates the richest possible experience.

Hub ▸ Diagonal Accents

Diagonal Clip-Path as Axis Direction Signal

Diagonal section dividers mark axis direction changes. When the reader transitions from horizontal hub scanning to vertical spoke reading, a diagonal clip-path section creates visual momentum without animation. The diagonal is a flat 2D graphic element — like a diagonal rule in a printed magazine — consistent with the editorial tradition.

The angle encodes direction: top-left to bottom-right signals "descending into detail" (hub to spoke). Bottom-left to top-right signals "ascending to overview" (spoke to hub). This directional encoding is purely compositional — no CSS animation, no parallax, no perspective transforms.

Descending Diagonal Implementation

The descending diagonal clips the bottom-right corner upward, creating a visual slope that draws the eye downward into the next content section. This is the primary diagonal used between hub overview and spoke detail. 

/* Descending diagonal — hub to spoke */
.axis-transition {
  background: var(--axis-zone-transition);
  clip-path: polygon(
    0 0,       /* top-left */
    100% 0,    /* top-right */
    100% 85%,  /* bottom-right (cut upward) */
    0 100%      /* bottom-left (full height) */
  );
  padding: var(--space-20) var(--space-6);
}

/* Ascending diagonal — spoke to hub */
.axis-transition--ascending {
  clip-path: polygon(
    0 15%,    /* top-left (cut downward) */
    100% 0,    /* top-right (full height) */
    100% 100%, /* bottom-right */
    0 100%      /* bottom-left */
  );
}

At mobile viewports (below 768px), the diagonal clip-path simplifies to a horizontal 3px structural border. The WAVE density oscillation is preserved through spacing tokens regardless of the axis direction signal simplification.

Usage Constraints

Gotcha

Maximum 2-3 diagonal accents per page. Diagonals supplement the flat grid system — they do not replace it. The grid remains the primary structural mechanism. Excessive diagonals create visual chaos that contradicts the editorial calm of IS-7 (Unhurried Editor).

Responsive Axis Collapse

Choreographic layouts adapt to viewport width through a 3-stage collapse strategy. At each breakpoint, the grid simplifies while the WAVE density oscillation is preserved through spacing tokens. The choreography's axis direction signals (diagonal clip-paths) simplify to horizontal structural borders at mobile viewports — the density rhythm continues even when the geometric signal cannot.

The 4-2-1 Column Cascade

/* Desktop: 4-column hub grid */
.hub-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: var(--space-6); /* 24px */
}

/* Tablet: 2-column collapse */
@media (max-width: 1440px) {
  .hub-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Mobile: single column stack */
@media (max-width: 768px) {
  .hub-grid {
    grid-template-columns: 1fr;
  }

  /* Diagonal → horizontal border */
  .axis-transition {
    clip-path: none;
    border-top: 3px solid
      var(--color-border);
  }
}

The featured territory tile (2-column span at desktop) becomes a full-width tile at mobile. The WAVE density oscillation persists because density is controlled by spacing tokens, not by column count. A single-column hub is still sparse (overview text, generous gaps), and a single-column spoke is still dense (long-form content, code blocks, tables).

WAVE Preservation Across Breakpoints

Tip

Density is independent of axis geometry. The hub-spoke WAVE oscillation is preserved at all breakpoints because density is encoded in spacing tokens (hub: --space-6 gaps, spoke: --space-between content gaps), not in column count. A 4-column hub and a 1-column hub both maintain the sparse phase of the WAVE — the column reduction changes scanning direction (horizontal to vertical) but not density.

Gathering the Threads

Choreographic Identity Synthesis

Choreography is the fifth axis pattern, extending the validated 3-way unification of axis geometry and density. Where Z-Pattern and F-Pattern discovered temporal equivalence, Bento Grid discovered spatial equivalence, and Spiral discovered proportional equivalence, choreography reveals a fourth mechanism: kinetic equivalence.

Kinetic equivalence means: the transition between density states IS the axis direction change. The reader does not traverse a sparse zone, then encounter a direction change, then enter a dense zone. The direction change and the density shift are the same event. The diagonal clip-path that marks "descending into detail" simultaneously signals the hub-to-spoke WAVE transition. Axis and density are not parallel phenomena that correlate — they are a single phenomenon observed from two angles.
