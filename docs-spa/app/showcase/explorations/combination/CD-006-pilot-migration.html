Section 02 · Z-Pattern · Pulse Density

Understanding the Soul

Before writing a single line of CSS, you need to understand the five soul pieces that constrain everything. These are not guidelines. They are invariants -- violations produce immediate, perceptible wrongness.

What IS the KortAI Soul?

Five locked constraints discovered through 337 research findings across five stages. The soul emerged from observation: sharp edges command authority, flat planes enforce honesty, limited color prevents distraction, the serif voice carries weight, and consistent structure builds trust.

The 5 Pieces

  1. border-radius: 0
  2. box-shadow: none
  3. Instrument Serif for Essence only
  4. 2-zone callouts, 4px left border
  5. #E83025 only accent

Each piece constrains differently. The geometry lock (border-radius: 0) constrains shape. The depth lock (box-shadow: none) constrains perception. The typography lock constrains voice. Together, they create the ANTI-PHYSICAL identity: a system that refuses to pretend it has depth.

Sharp edges.
Flat planes.
Limited color.

Why border-radius: 0?

Round corners signal friendliness. This system does not seek to be friendly -- it seeks to be authoritative. Sharp edges are a command: "this boundary is real." When every corner is sharp, the reader trusts that edges mean something.

tokens/geometry.md -- the geometry lock
/* LOCKED -- T1: geometry.md */ --border-radius: 0; /* Sharp edges command authority */ --box-shadow: none; /* Flat planes enforce honesty */ /* Enforce globally -- Soul Piece #1 */ *:where(:not(input, button, select)) { border-radius: 0; }
Why no box-shadow?

Shadows imply depth. This system has no depth -- it is flat, honest, and refuses to create illusions of z-axis layering. The ANTI-PHYSICAL identity means: if it does not physically exist on a flat plane, it does not appear. No floating cards, no elevated buttons, no material design depth cues.

Gotcha

The most common soul violation is accidental box-shadow from CSS frameworks or component libraries. Always check computed styles -- a reset might not be enough if a framework injects shadows via specificity.

Why Instrument Serif for Essence only?

The serif typeface carries weight and authority. If used everywhere, it loses its power. Instrument Serif appears only in Essence Callouts and Core Abstraction principles -- the moments when the system speaks with its deepest voice. This scarcity makes the serif voice unmistakable when it appears.

Tip

The quickest soul test: squint at your page. Does anything pop forward? If any element appears to float above the surface, you have a shadow or depth violation. Flatten it.

Section 03 · Bento Grid · Islands Density

Token Reference Cards

The design system is built on four categories of tokens. Each category is a discrete island -- you can configure them in any order, and each is valid independently.

Colors

The locked palette has 8 colors. The primary accent #E83025 is the only warm color. Everything else is deliberate: cream backgrounds, dark text, earth-tone borders.

tokens/colors.md -- the color lock
--color-primary: #E83025; /* The ONLY accent */ --color-background: #FEF9F5; /* Warm cream paper */ --color-text: #1A1A1A; /* Near-black body */ --color-border: #E0D5C5; /* Earth-tone border */
Gotcha

#E83025 is the ONLY accent color. The callout accent palette (--accent-blue, --accent-green, etc.) is reserved exclusively for callout left borders. Never use callout accent colors for general UI elements.

Typography

Three fonts, strict roles. No substitutions.

/* Font trio -- LOCKED */ --font-display: 'Instrument Serif'; --font-body: 'Inter'; --font-mono: 'JetBrains Mono';
Tip

Load fonts via <link> in the document <head>, not @import in CSS. The link tag allows parallel loading; @import serializes it.

Spacing

A numbered scale from 4px to 80px, plus three semantic aliases for gestalt relationships.

--space-1: 4px; --space-2: 8px; --space-4: 16px; --space-8: 32px; --space-12: 48px; --space-16: 64px; /* Semantic aliases */ --space-within: 8px; --space-between: 32px; --space-chapter: 64px;

Geometry & Structure

The border system has exactly three categories. There is no middle ground, no 2px compromise. This was the #1 systemic audit finding: the 2px border epidemic affected 108 CSS declarations and over 1,000 computed instances.

Border categories
/* Cat 1: Structural/Accent */ border: 3px solid; border-left: 4px solid; /* Cat 2: Data separators */ border-bottom: 1px solid; /* Cat 3: Micro-element */ border: 1px solid; /* BANNED: 2px borders */

Movement 3: The Recommendation

Reasoning
For a tutorial page, should I use F-Pattern or Z-Pattern?

F-Pattern

  • Progressive complexity (simple to hard)
  • CRESCENDO builds momentum through the tutorial
  • Narrative arc creates a sense of journey
  • Reader follows a single thread
  • Natural for "do this, then this, then this"

Z-Pattern

  • Alternating instruction/code rhythm
  • PULSE creates breathing room between steps
  • Conversational tone (ask, then answer)
  • Good for troubleshooting mid-tutorial
  • Natural for "what if?" / "here's how" pairs

Recommendation: Use F-Pattern for the overall tutorial arc (sections build progressively) and Z-Pattern within individual implementation sections (where step-by-step Q&A serves the reader). This is compound sequencing -- one pattern per section, the compound effect through scrolling.

Section 05 · Z-Pattern · Pulse Density

Building Your First Component

Time to build. We will construct a page step by step, alternating between instruction and code. Each step answers one question and produces one tangible result.

Build Checklist
  • Create the HTML structure with semantic elements
  • Apply the token stylesheet via :root custom properties
  • Add your first callout component with 4px left border
  • Apply density rhythm through spacing tokens
  • Verify fractal compliance at 5 scales
Step 1: What is the page skeleton?

Start with semantic HTML. Every KortAI page follows the same skeleton: a skip link, a dark header, a <main> container with a width-constrained page container, and a dark footer. This structure is not optional -- it is documented in Section 4 of the convention spec.

Step 1 -- Page skeleton
<a href="#main" class="skip-link">Skip to content</a> <header> <div class="header-inner"> <h1>Your Page Title</h1> </div> </header> <main id="main"> <div class="page-container"> <!-- max-width: 1100px --> <!-- Your sections here --> </div> </main> <footer><!-- mirrors header --></footer>
Step 2: How do I apply the token stylesheet?

Copy the complete :root block from Section 10 of the convention spec. This single block contains every locked token: colors, typography, spacing, geometry, callout accents, and zone backgrounds. Do not cherry-pick tokens -- include the full block to ensure consistency.

Tip

Use CSS custom properties so tokens cascade naturally. When you change --color-primary in :root, every element using that token updates automatically. This is the power of token-based design.

Step 3: How do I add a callout?

Every callout shares the same two-zone structure: a label zone (sparse, uppercase, small) and a body zone (dense, prose content). The only difference between callout types is the accent color and the semantic weight.

Step 3 -- An Info Callout
<div class="callout callout--info"> <div class="callout-label">Info</div> <div class="callout-body"> <p>Your message here.</p> </div> </div> <!-- CSS: the 2-zone callout structure --> .callout { border-left: 4px solid; /* Soul Piece #3 */ padding: var(--space-6) var(--space-8); }
Gotcha

The left border is always 4px, never 2px. This is a Category 1 structural border per the 3-category border system. Using 2px reintroduces the border epidemic.

Step 4: How do I apply the density rhythm?

Density is not a separate layer to apply -- it emerges from your spacing decisions. Use the semantic spacing aliases: --space-within (8px) for related elements, --space-between (32px) for distinct elements, and --space-chapter (64px) for section breaks. The density pattern is already present in these choices.

Step 5: How do I verify fractal compliance?

Check each of the five scales independently. Navigation (does your TOC mirror page rhythm?), page (sparse introduction, dense middle, sparse conclusion?), section (each section has intro/detail/summary?), component (label sparse, body dense?), character (headings sparse, paragraphs dense?). Each scale must echo the others.

Step 5 -- Fractal verification pseudocode
// Check 5 fractal scales -- DD-F-006 const scales = [ 'navigation', // TOC mirrors page density 'page', // sparse -> dense -> sparse 'section', // intro -> detail -> summary 'component', // label(sparse) -> body(dense) 'character' // headings -> paragraphs ]; // Each scale: is there sparse/dense alternation? // If YES at all 5 scales -> PASS (AD-F-027) // If any scale missing -> identify and fix
Challenge

Now try building the same section with Bento Grid instead of Z-Pattern. What changes about the density? Does the content still work when each step becomes a discrete island instead of a conversational flow? The answer reveals whether your content is inherently sequential or inherently modular.

Section 06 · Bento Grid · Islands Density

Auditing for Compliance

Every page must pass four audit domains before it can ship. These domains are independent -- you can run them in any order, and each produces a binary PASS/FAIL result.

Soul Compliance

Soul Compliance Checklist
  • border-radius: 0 on every element (including pseudo-elements)
  • box-shadow: none globally -- no exceptions
  • No filter: drop-shadow() anywhere
  • All backgrounds fully opaque (opacity: 1)
  • Font trio only: Instrument Serif, Inter, JetBrains Mono
  • Zero 2px borders (only 1px, 3px, 4px)
  • #E83025 is the only accent color

Density

Verify PULSE rhythm is present, density waves oscillate (not plateau), recovery zones exist after dense sections, and fractal self-similarity holds at all 5 scales.

Quick Test

Scroll through at reading speed. Does density alternate? If three consecutive sections feel equally heavy, you have a plateau problem.

Fractal

Each of the 5 fractal scales must independently show sparse/dense alternation. Navigation mirrors page rhythm. Page has arc. Sections have intro/detail/summary. Components have label/body. Characters have headings/paragraphs.

Tip

Each scale is independent -- verify each separately. A page can have perfect page-level fractal rhythm but broken component-level rhythm.

Accessibility

Accessibility Checklist
  • Color contrast ratios meet WCAG AA (4.5:1 for text, 3:1 for large)
  • :focus-visible outline on all interactive elements
  • Semantic HTML: proper heading hierarchy (h1 > h2 > h3)
  • Every section has aria-label
  • Skip link works and targets #main-content
  • Source order matches logical reading order

Deployment Overview

Stage

Deploy to staging environment. Run the perceptual audit against the live page. Verify that font loading completes before visual testing -- document.fonts.ready is critical.

Font Verification
