## BUILD PROTOCOL REFERENCE
═══════════════════════════════════════════════════════════════════════════════

### Per-Section Build Process

```
1. PLAN
   - Review relevant research from RESEARCH-ACTIVE.md
   - Review relevant explorations from DISCOVERIES-LOG.md
   - Identify components needed from COMPONENTS-REGISTRY.md

2. BUILD V1
   - Create page.tsx
   - Use realistic content (not lorem ipsum)
   - Apply KortAI design tokens

3. CAPTURE V1
   - Take Playwright screenshots (full + sections)
   - Save to screenshots/sections/

4. AUDIT V1 (Deploy 7 agents)
   - ZONE-TYPOGRAPHY
   - ZONE-SPACING
   - ZONE-CONTAINERS
   - ZONE-HIERARCHY
   - ZONE-COMPOSITION
   - ZONE-COLOR
   - ZONE-LAYOUT-INFRASTRUCTURE

5. DOCUMENT V1
   - Record findings
   - Note what works, what doesn't

6. BUILD V2
   - Address audit issues
   - Enhance what's working

7. ITERATE
   - Repeat until quality ≥8/10
   - Minimum 3 iterations

8. VERIFY
   - Check RESEARCH-ACTIVE.md for unapplied research
   - Ensure all relevant findings considered

9. FINALIZE
   - Lock section
   - Update this document

10. CHECKPOINT
    - Update MASTER-STATE.md
    - Update COMPONENTS-REGISTRY.md
```
