# STAGE 2: DENSITY EXPLORATION (DD)
## Where Visual Rhythm Was Discovered

═══════════════════════════════════════════════════════════════════════════════
## THE STORY
═══════════════════════════════════════════════════════════════════════════════

The components were solid. 11 verified, 5 soul pieces documented, ~92% compliance.
Now the question: How do these components BREATHE together on a page?

This stage explored DENSITY — not just "how much stuff" but the rhythm, the
contrast, the flow. Six explorations, each testing a hypothesis:

**DD-001 Breathing (33/40):** Can a page inhale and exhale? Dense sections
(INHALE) alternate with sparse sections (EXHALE). The PULSE pattern emerged.
Initially violated traffic-light anti-pattern (green Tip adjacent to red Gotcha),
but the breathing concept proved valuable.

**DD-002 Gradient (35/40):** Can density build progressively? CRESCENDO pattern:
sparse introduction → medium context → dense details → sparse resolution. Key
insight: Remove density indicators after section 2 — users learn the pattern.

**DD-003 Islands (32/40):** Can dense content float in sparse ocean? Archipelago
metaphor — clusters of information as navigable islands. Works conceptually but
underutilized the multi-island potential.

**DD-004 Layers (35/40):** Can information stratify like geological strata?
Atmosphere → Surface → Topsoil → Subsoil → Bedrock. 5-bar depth indicator
shows current position. Expert vs Novice paths emerge naturally.

**DD-005 Rivers (36/40):** Can width communicate density? Narrow channels =
dense (code). Wide rivers = sparse (context). Parallel rivers for comparison.
Confluence points for decisions. River delta for choices.

**DD-006 Fractal (36/40 — HIGHEST):** Can the same rhythm repeat at every scale?
Page level: █░░█░░█░░. Section level: █░░█░░█░░. Component level: █░░█░░█░░.
Character level: █░░█░░█░░. Self-similarity creates coherence.

**The breakthrough:** DD-006's fractal principle isn't just one pattern among six.
It's the META-PATTERN. Apply the same dense/sparse rhythm at ALL scales, and
documentation feels coherent and navigable. This constrains every future stage.

## KEY DECISIONS (With Provenance)
═══════════════════════════════════════════════════════════════════════════════

### Decision 1: FRACTAL is the META-PATTERN (DD-F-006)
**What:** Same dense/sparse rhythm at page, section, component, and character levels
**Why:** DD-006 scored highest (36/40). Self-similarity creates coherence that
variation cannot. When in doubt, apply the same rhythm everywhere.
**Impact:** ALL OD explorations must apply fractal self-similarity. Not optional.

### Decision 2: Traffic-Light is CRITICAL Anti-Pattern (DD-F-008)
**What:** Never place green (#4A9D6B) and red (#E83025) callouts adjacent
**Why:** DD-001 violated this and scored lowest initially (30.5). Creates
kindergarten-level visual language, undermines editorial authority.
**Impact:** All future stages must verify against traffic-light anti-pattern.

### Decision 3: CRESCENDO for Learning Content (DD-F-002)
**What:** Sparse → Medium → Dense → Resolution flow for tutorials
**Why:** DD-002 demonstrated effective progressive build (35/40). Users learn
the density pattern after section 2 — can remove indicators.
**Impact:** OD-002 Narrative should pair with CRESCENDO density.

### Decision 4: WIDTH = INVERSE DENSITY (DD-F-005)
**What:** Narrow channels for dense content, wide rivers for sparse context
**Why:** DD-005 proved width is a density signal (36/40). Parallel rivers
enable comparison; confluence points mark decisions.
**Impact:** OD-001 Conversational should use width variation.

### Decision 5: LAYERS for Expert/Novice Paths (DD-F-004)
**What:** Geological model (Atmosphere → Bedrock) with skip options
**Why:** DD-004 showed experts can surface→bedrock skip while novices traverse
all layers (35/40). 5-bar depth indicator aids navigation.
**Impact:** OD-004 Confidence should pair with GEOLOGICAL density.
