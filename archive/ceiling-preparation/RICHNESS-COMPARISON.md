**Date:** 2026-02-16
**Analyst:** richness-analyst
**Team:** ceiling-preparation

## Executive Summary

**Core finding:** Showcase richness is achieved through **MECHANISM SATURATION + ITERATIVE DISCOVERY**, not single techniques. DD-006 uses 12+ mechanisms across 4 scales; CD-006 uses 15+ across 5 scales. Middle-tier experiment (8-10 mechanisms, 2 scales) captured ~60-65% of showcase richness. The gap is **TECHNIQUE DENSITY** (mechanisms deployed per scale) and **SCALE DEPTH** (how many zoom levels show coherence).

**What richness ACTUALLY is:** Multi-channel semantic encoding. Showcases say the SAME thing in 4-5 CSS channels simultaneously (border-weight + spacing + zone-bg + typography all encode "depth"). Middle says things in 2-3 channels. Floor says things in 1 channel.

**Provenance contribution:** Showcases had 337 research findings (R1-R5) → 50+ findings per exploration → iterative refinement → 12-18 mechanism deployment. Middle had SAME catalog (18 mechanisms) but NO iteration → baseline deployment. Provenance = iteration budget, not vocabulary access.

## 3. Richness Gap Analysis — What's MISSING in Middle vs DD-006?

### Gap Dimension 1: Mechanism Count (QUANTITY)

**DD-006 deployed:** 12 mechanisms
**Middle deployed:** 12 mechanisms
**Gap:** ZERO

**Conclusion:** Mechanism count is NOT the gap. Middle deployed the SAME number of mechanisms as DD-006.

**BUT:** DD-006 was built at a time when only ~8-10 mechanisms existed (pre-extraction). DD-006 DISCOVERED solid offset, fractal nesting, recursive 2-zone DNA. Middle APPLIED pre-extracted mechanisms.

**The real gap:** Discovery vs application. DD-006 invented mechanisms. Middle used them.

---

### Gap Dimension 2: Scale Depth (COHERENCE)

**DD-006 scales:** 4 (page/section/component/character)
**Middle scales:** 2 (page/component)
**Gap:** 2 missing scales (section + character)

**Concrete difference:**

**DD-006 Section scale (fractal rhythm at section level):**
Each of the 4 scale demonstration blocks has INTERNAL progression:
- Sparse section header (12px label)
- Dense demonstration box (24px padding, content)
- Sparse example code (16px body text)
- Dense explanation paragraph (18px body text)

**Middle Section scale (uniform internal structure):**
Architecture section:
- Heading (uniform treatment)
- Problem callout (uniform padding 20px 24px)
- Diagram (uniform treatment)
- Benefit subsections (uniform border-bottom 1px, uniform padding)

**The gap:** DD-006's sections INTERNALLY echo the page-level pattern (sparse→dense→sparse). Middle's sections have UNIFORM internal structure. No fractal coherence at section scale.

**Impact on richness:** Section-scale coherence is ~15-20% of perceived richness. When you read ONE section in isolation, DD-006 feels "designed" because the section itself has rhythm. Middle sections feel "assembled" because components are just placed.
