Component Placement

8-IC Architecture
System Bus -- Town Level Components (Singletons)
U1 -- CPU
Central Processing Unit

Mayor

Primary system coordinator. Receives directives from the I/O controller (you), routes tasks to appropriate peripheral buses, manages inter-bus arbitration, handles system-level decisions. All data paths route through this IC.

gt mayor up # Power on U1 gt mayor status # Read status register gt mayor down # Power down U1
Package: Opus 4.5 -- BGA-2048 (highest pin count for complex routing)

System bus: U1 data/address lines to peripheral components

U2 -- Watchdog Timer

Deacon

System health monitoring IC. Checks heartbeat of all components, handles inter-IC handshake protocols, reports faults to CPU, manages component recycling after timeout.

Package: Haiku -- SOIC-8 (minimal pin count)
U3 -- Voltage Regulator

Dogs

Quality regulation IC. Monitors output quality before signals leave the system. Enforces voltage standards, guards against signal degradation. Can reject and reset faulty outputs. The regulator that keeps everything in spec.

Package: Sonnet -- QFP-44 (quality check requires moderate logic)

Peripheral bus: System level to rig-level components

Peripheral Bus -- Rig Level Components (Per-Project)
U4 -- DSP (Signal Processor)

Refinery

Task decomposition processor. Receives complex signal packets from CPU, decomposes into atomic operations, dispatches to worker ICs. Manages the data merge queue.

Package: Opus 4.5 -- BGA-1024 (complex signal processing)
U5 -- ADC (Sensor)

Witness

Analog-to-digital observer. Monitors all signals on the rig bus, converts operational state to digital records, provides diagnostic data to new components, reports anomalies.

Package: Haiku -- SOIC-16 (observation and logging)

I/O Interface -- Human Controller
J1 -- I/O Connector

Overseer (You)

External interface port. Product Manager and system designer. Sends feature request signals, reads inbox status register, approves output data, provides human judgment. You have a unique bus address, your own mailbox register, and bidirectional data transfer capability.

Ground Plane: Beads Memory Controller

PCB Layer 4 (Bottom) -- Persistent Storage IC -- 225K Lines Go

Every component on the board shares this ground plane. Agents suffer from "50 First Dates" syndrome -- they power up with cleared registers. Beads provides the flash memory: git-backed persistent storage, shared memory bus, hash-addressed locations to prevent bus contention, and garbage collection through compaction.

L1 -- Hot Cache (SRAM)

  • Current task registers
  • Last N transaction log
  • Active pointer table

L2 -- Warm Buffer (DRAM)

  • Recent commit history buffer
  • Cross-component message queue
  • Bus state snapshots

L3 -- Cold Storage (Flash)

  • Full project archive
  • Previous execution traces
  • Archived decision log

Memory hierarchy: L1 cache hit -> L2 buffer -> L3 flash (only on cache miss)

Board Revision History

6 Revisions
Rev 1.0 -- 3.0

Single-Chip Era

Manual coding (1x) to completions (2x) to chat-based (5x). Progressively larger dies, higher clock speeds, but always one processor. Thermal limits approached.

Rev 4.0 -- 5.0

Multi-Chip Module

Coding agents (10x) and multi-agent clusters (25x). Multiple dies on one substrate. Parallel execution begins. Bus contention becomes the bottleneck.

Rev 6.0 -- Current

Full System-on-Board

Gas Town is Rev 6.0. A complete multi-IC system with dedicated buses, shared memory controller, and specialized processing elements. 50x+ throughput. The board manages itself -- you just feed it requirements through the I/O port.

Design Rules

Operating Constraints
DRC-001 -- GUPP

Universal Propulsion

"If there is work on your hook, YOU MUST RUN IT." Signals persist on bus lines (git worktrees). Components sample on power-up. If signal present, execute immediately.

DRC-002 -- Zero Logic

No On-Board Intelligence

"Claude makes all decisions." All logic delegated to AI ICs. No hardwired heuristics in the PCB traces. The board routes signals, not decisions.

DRC-003 -- Hot Swap

Component Replaceability

Individual ICs are field-replaceable. The bus protocol is persistent. The component is a socket, not a solder joint. Hot-swap freely.

DRC-004 -- Fault Tolerance

Nondeterministic Routing

Signal paths are nondeterministic. Different executions use different trace routes. System functions as long as components are populated. Tolerant of individual IC failures.

Design Rule Violation -- Input Requirements

"If you're not at Stage 7, you'll get your face ripped off by superintelligent chimpanzees." This board requires Stage 7+ input voltage, tmux bus controller, $50-200/day power supply, high noise tolerance, and 100+ hours of multi-chip operating experience. Underpowered inputs will produce undefined behavior.
