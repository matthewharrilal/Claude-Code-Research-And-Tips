## EXECUTIVE SUMMARY

The 20 flagship preparation reports converge on a single thesis: **spatial confidence is THE gate, and everything else is optimization on top of it.** The ceiling experiment scored 9/9 novelty, deployed 14 mechanisms, achieved 8/8 soul compliance, and was declared DO NOT SHIP because 70-80% of the page was blank cream void. Every preparation report traces back to this root cause.

The flagship experiment must solve THREE problems simultaneously:

1. **Prevent the void** (spatial confidence gate, content-first methodology, void budget calculation)
2. **Produce genuine composition** (compositional fluency through multi-channel encoding, not vocabulary accumulation)
3. **Achieve beauty** (confident intention at every scale of attention -- warm, authoritative, unhurried)

The recommended pipeline: **22 agents, 5 passes, 7 gates, ~320 minutes, 55-65% probability of Full Flagship, 85-90% probability of Minimum Viable Flagship.**

A calibration run is STRONGLY recommended before committing to the flagship.

## FLAGSHIP PIPELINE

### Overview: 5 Passes, 7 Gates, 22 Agents

```
PASS 0: Content Architecture (Opus, 20-30 min)
  └── GATE 0: Spatial Budget Gate (content-to-void >= 60:40)

PASS 1: Spatial Skeleton (2x Sonnet competitive, 35-50 min)
  └── GATE 1: Spatial Confidence Gate (7 binary checks)
  └── GATE 2: Landmark Completeness Gate (header + footer + sections)

PASS 2: Mechanism Deployment (Sonnet, 25-40 min)
  └── GATE 3: Composition Verification Gate (mechanism count, reinforcing pairs, transitions)

PASS 3: Metaphor Integration (Opus, 20-35 min)
  └── GATE 4: Metaphor Coherence Gate (2 fresh-eyes: structural or announced?)

PASS 4: Intentionality Layer (Opus, 15-25 min)
  └── GATE 5: Ship-Ready Gate (Mode 4 audit, dual PA, programmatic audit)
```

### Pass 0: Content Architecture (Opus 4.6, 20-30 min)

**Single concern:** What goes where, how much space each zone gets.

**Inputs:** Raw content + metaphor derivation + content inventory
**Outputs:** Content zones, section plan, spatial budget table, transition type assignments

**Key deliverables:**
1. Content-to-zone mapping with word counts per zone
2. Spatial budget table (content height vs structural overhead per zone)
3. Transition type assignment for each zone boundary (HARD CUT / SPACING SHIFT / CHECKPOINT -- see Report 02, A1)
4. Density arc specification (where the page peaks and rests)

**Gate 0: Spatial Budget Gate (binary)**
- Content-to-void ratio >= 60:40? PASS/FAIL
- Zone count <= maximum for content volume (per A3 table)? PASS/FAIL
- Content/Zone ratio >= 1.5? PASS/FAIL
- On FAIL: Reduce zones, compress spacing, recalculate. Cannot proceed until PASS.

### Pass 1: Spatial Skeleton (2x Sonnet 4.5, competitive, 35-50 min)

**Single concern:** HTML/CSS skeleton -- containers, flow, rhythm, proportions.

**Innovation: Competitive Build.** Two builders execute in parallel with the same plan but different approaches. Team-lead selects the better skeleton. This doubles P(good skeleton) from ~70% to ~91%. (Source: Report 17, Section 2)

**Embedded Auditor** (Sonnet 4.5) monitors the build in real-time. Screenshots every 100 CSS lines. Hard alert if void ratio exceeds 30%. (Source: Report 17, Section 4)

**Mandatory Communication (Checkpoint A):** Builder MUST send CLARIFICATION-REQUEST to planner before writing any HTML. Content: biggest spatial concern, checkpoint whitespace calculation, confirmation of plan read. (Source: Report 06)

**Mandatory Communication (Checkpoint B):** Builder MUST send POST-BUILD REPORT after writing file. Content: total page height, content-to-void ratio, section count, any deviations from plan. (Source: Report 06)

**Builder Self-Check (before reporting build complete):**
1. Container width 940-960px (verify via getBoundingClientRect)
2. CPL 45-80 (formula: content-width / font-size / 0.6)
3. Content-to-void >= 60:40
4. No empty structural elements (every div with class has text or visible children)
5. Soul 8/8
6. Header + footer present with content
7. Token compliance >= 80% (var() / (var() + raw))

**Gate 1: Spatial Confidence Gate (7 binary checks)**
- SC-01: Container width 940-960px
- SC-02: Maximum contiguous void <= 1.5 viewport heights (2,160px)
- SC-03: Content distribution >= 70% of scroll depth has content
- SC-04: Squint test -- 3+ distinct content regions at 25% zoom (team-lead judgment)
- SC-05: Header <= 20% of first viewport
- SC-06: >= 5 content sections with >= 200px content height each
- SC-07: Footer reachable, gap <= 1 viewport height

Max 2 fix cycles. After 2 failures: ABORT or downgrade to ceiling tier. (Source: Report 03)

**Gate 2: Landmark Completeness (team-lead verifies)**
- Header exists with visible content
- Footer exists with visible content
- All planned sections present
- Container width confirmed (redundant with SC-01 but team-lead confirms)

### Pass 2: Mechanism Deployment (Sonnet 4.5, 25-40 min)

**Single concern:** CSS technique application. No metaphor vocabulary yet.

**Embedded Auditor** continues from Pass 1, monitoring for spatial degradation.

**Key constraints:**
- Per-category minimums: Spatial: 1+, Hierarchical: 1+, Chromatic: 1+, Density: 1+, Navigational: 1+
- Mechanism cap: <= 16 (hard), 12-14 (recommended)
- Density cap per viewport: <= 4 simultaneous mechanisms visible
- Distribution requirement: every third of the page has >= 2 distinct mechanisms
- 2+ reinforcing pairs (same semantic dimension, different CSS channels)
- 3+ transition types, no identical adjacent
- >= 1 designed moment per scroll quartile

**Restraint Protocol:**
- Planner documents >= 15 rejected placements in Restraint Map (Phase A)
- Builder documents >= 6 additional rejections (Phase B)
- Total >= 21 rejections before proceeding to Pass 3
- Format: REJECTED / PROPOSED LOCATION / SCALE / REASON / WHAT IS PRESERVED (Source: Report 04)

**Signal-to-Silence Ratio:** 0.6-0.8:1. For every unit of signal, 0.6-0.8 units of designed silence. >= 2 documented silence zones (200px+ height). (Source: Report 04)

**Gate 3: Composition Verification**
- CP-01: Mechanism count <= 16
- CP-02: Restraint ratio (rejected/deployed) >= 1.5
- CP-03: Zone boundaries perceptible without labels (invisible_boundaries <= 1)
- CP-04: Spatial re-check (SC-01 + SC-03 still passing after Pass 2)
- On CP-04 FAIL: ROLLBACK to Pass 1 skeleton, re-attempt with reduced elaboration

### Pass 3: Metaphor Integration (Opus 4.6, 20-35 min)

**Single concern:** Vocabulary, zone naming, channel encoding.

**Why separate from mechanism deployment:** The ceiling experiment combined both and produced mechanisms that existed in CSS but were invisible under metaphor vocabulary labels. Separating ensures the Mechanism Builder focuses on "does this CSS technique work here?" while the Metaphor Builder focuses on "does this vocabulary express the right meaning?" (Source: Report 17)

**Key constraints:**
- Metaphor vocabulary layered on top of verified mechanisms
- Zone naming must be structural (perceivable without labels)
- Channel encoding: metaphor vocabulary maps to CSS channels already deployed
- No NEW mechanisms added -- metaphor uses existing mechanism deployment

**Gate 4: Metaphor Coherence (2 fresh-eyes agents, zero context)**
- Q: "Is there a metaphor? What is it?"
- Q: "Does it feel structural or announced?"
- Q: "Can you identify zone boundaries without reading labels?"
- PASS: Both identify metaphor without labels. Structural score >= 70%.
- FAIL: Label-dependent -> FLAG for review. < 50% structural -> REDESIGN.

### Pass 4: Intentionality Layer (Opus 4.6, 15-25 min)

**Single concern:** Bookending, self-reference, cognitive transitions, meta-annotation.

**6 intentionality dimensions** (from Report 14, derived from crown jewel forensics):

1. **Self-reference:** Content describes its own container/styling. >= 1 instance. CSS cost: 0 lines. (Example: code block showing the CSS that styles it)
2. **Pedagogical sequencing:** Content reveals in the order the READER needs, not the order the SYSTEM organizes. Confirmed by checking: would reorganizing sections reduce understanding?
3. **Cognitive mode transitions:** Bridge text between sections that shifts the reader's mental state. >= 2 instances. (LEARN -> BUILD, UNDERSTAND -> APPLY)
4. **Structural bookending:** Opening section echoes closing section in form or content. Confirms the page is a complete thought, not a truncated document.
5. **Meta-annotation:** Section labels that illuminate the architecture (not just navigation). Examples: "FIRST CONTACT" instead of "Overview."
6. **Provenance threading:** References to the design system's own evolution. CSS cost: 0 lines.

**Height budget: <= 10% page height increase from Pass 4.** Intentionality is lightweight (content text, minor structural elements). If Pass 4 adds more than 10%, the additions are content, not intentionality. Revert heaviest additions. (Source: Report 07, IT-01)

**Crown Jewel reference:** Intentionality builder receives Crown Jewel Technique Inventory (from Report 11). Not to copy techniques, but to calibrate quality level. "This is what compositional fluency feels like. Match the QUALITY LEVEL, not the specific patterns." (Source: Report 17)

**Gate 5: Ship-Ready Gate (Full Mode 4 audit)**

Pre-audit steps:
1. Post-Build Metacognitive Reviewer (cold-eyes, emotional response test)
2. Team-lead screenshot pre-capture (cold look + scroll-through at all viewports: 1440px, 768px)

Mode 4 audit:
- 9 independent auditors, 48 PA questions, 3 viewports
- Screenshot pre-capture pattern (eliminates Playwright contention)
- All auditors read saved images via Read tool
- Zero shared context between auditors

Additional audits (parallel with Mode 4):
- Programmatic audit (soul compliance, token compliance, CPL, heading spacing)
- Comparative audit (vs Middle experiment output)
- Blind novelty evaluation (D3: structural fingerprint, CSS value overlap, mechanism combination)

## CROSS-CUTTING THEMES

The 20 reports converge on these principles. They are not new findings -- they are the CONFIRMED patterns that every report independently reinforced.

### 1. Spatial Confidence Is THE Gate

Every high-quality result had well-managed spatial composition. Every quality failure involved spatial breakdown. This is the only hypothesis with ZERO counter-evidence across all 4 experiments (Phase D, Middle, Ceiling, Crown Jewels). The spatial confidence gate (7 binary checks) is the single highest-leverage addition to the pipeline. (Sources: Reports 01, 02, 03, 07, 08, 09, 15, 16)

### 2. Binary Rules -> 100% Compliance; Judgment Rules -> ~0%

This is the most important architectural principle in the project. Every binary rule in the ceiling prompt achieved full compliance (18/18). Every judgment rule failed. The flagship prompt must contain ZERO judgment rules in the execution spec. All judgment content goes to Layer 1 (Conviction) as context, not instruction. (Sources: Reports 06, 07, 08, 10, 20)

### 3. Attention Is the Limiting Resource (~100 Units Per Agent)

Agent attention budget is approximately 100 units. The ceiling prompt demanded ~128 (28% over budget). Binary identity rules cost ~0.5 units each; judgment rules cost ~4-6 units each. A 56-rule prompt with 26 judgment rules consumes ~152 attention units. The solution: keep builder prompt to ~55 lines, move everything else to GATES (which cost the builder zero attention). (Source: Report 20)

### 4. Restraint > Saturation

The ceiling experiment deployed 14 mechanisms. Only 1 was perceptually visible. 13 mechanisms were present in CSS but not perceivable. The problem was not insufficient mechanisms -- it was insufficient DISTRIBUTION and excessive CONCENTRATION. Mechanism cap per viewport: 4. Distribution requirement: every third of the page has >= 2. The restraint ratio (rejected/deployed >= 1.5) forces documented evaluation, not default deployment. (Sources: Reports 04, 11, 13, 19)

### 5. Compositional Fluency Is the Real Differentiator

Vocabulary fluency (knowing many mechanisms) maxes out at "professionally stiff." Compositional fluency (mechanisms encoding SHARED semantic dimensions through DIFFERENT CSS channels) produces genuine beauty. The acid test: describe the page without listing mechanisms. If you can't, the mechanisms are decoration, not composition. Reinforcing pairs (2+ mechanisms encoding the same meaning through different CSS properties) are the minimum unit of compositional fluency. (Source: Report 13)

### 6. "Wall of Content" Is the Next Risk (Overcorrection)

9 of the 20 reports warn about overcorrection from void to claustrophobia. The breathing zone exception (1 scroll position of transition text between content-bearing sections) prevents over-stuffing. Signal-to-silence ratio 0.6-0.8:1 ensures rhythm. The void budget is a FLOOR, not a target -- 70% content is the minimum, not the goal. (Sources: Reports 02, 04, 09, 15)

### 7. Intentionality Requires NO Extra CSS

All 6 intentionality dimensions (self-reference, pedagogical sequencing, cognitive transitions, bookending, meta-annotation, provenance threading) are content/structure decisions, not styling decisions. Total cost: ~93-168 lines HTML, ~26-48 min, zero additional CSS. Crown jewel benchmarks: DD-006 4/6, CD-006 6/6, Ceiling 0/6. (Source: Report 14)

### 8. Prompt Must Be ~240 Lines Max

The ceiling prompt was 1,004 lines and achieved worse compliance than the Middle experiment's shorter prompt. The meta-analysis identifies ~200 lines as the preparation ceiling: beyond this, additional rules DEGRADE output by consuming attention budget. The 4-layer architecture achieves ~240 lines across Layers 1-3 (76% reduction from ceiling prompt). (Sources: Reports 10, 20)

### 9. Beauty = Confident Intention at Every Scale of Attention

Beauty is not the sum of richness + novelty + compliance. It is "the feeling of encountering a page that is warm without being soft, austere without being cold, authoritative without being aggressive, and restrained without being empty." The Ceiling page fell off "restrained/not-empty." The Middle page walked all four tightropes at low height. The crown jewels walk all four at height. Beauty cannot be directly specified -- but its prerequisites (spatial confidence, compositional fluency, intentionality, restraint) can be gated. (Source: Report 12)

### 10. Content Inventory Has VETO POWER Over Metaphor

The ceiling experiment chose "Secure Facility" for semantic alignment (security content -> security metaphor). The content naturally divided into 5 sections with peak density in the MIDDLE, but the metaphor demanded escalating density toward the END. The mismatch produced the void. Content-first methodology reverses this: inventory content FIRST, derive natural structure, THEN select a metaphor that amplifies what already exists. If the metaphor conflicts with the content's natural density arc, the content wins. (Source: Report 05)
