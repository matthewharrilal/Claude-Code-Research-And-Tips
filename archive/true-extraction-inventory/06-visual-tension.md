## Executive Summary

The tension-test files represent THE MOST CRITICAL OUTPUT for extraction analysis — they are real content with metaphor-driven compositional layouts generated by the tension-composition pipeline. This audit cataloged **what's EXTRACTABLE (shared vocabulary) vs what's GENERATED (metaphor-specific composition)**.

### The Critical Discovery

**The metaphor shapes the layout architecture entirely.** Each of the 5 metaphors for the same content produces:
- **DIFFERENT page structure** (sections, dividers, containers)
- **DIFFERENT visual hierarchy** (headings, labels, callouts)
- **DIFFERENT component types** (tables vs definition lists vs nested containers)
- **SAME core content** (text, data, quotes — identical across all 5)

**The Ratio: ~80% GENERATED, ~20% EXTRACTABLE**

**Average: ~75% GENERATED, ~25% EXTRACTABLE**

### What This Means for Extraction

**The pipeline's VALUE is in COMPOSITION, not COMPONENTS.**

The extractable vocabulary is:
- Typography system
- Color palette
- Basic component shapes (table, blockquote, code block, heading)
- Core layout primitives (container, section, separator)

The generated output is:
- Page architecture
- Section structure
- Metaphor-specific labels
- Container relationships
- Visual hierarchy decisions
- Component selection (table vs list vs nested divs)

**Implication:** You CANNOT extract "a Boris template" or "a Gas Town template". The metaphor IS the template. The extraction target is the PRIMITIVES used to BUILD the metaphor-driven layouts.

## Part 7: Extraction Planning Implications

### What to Extract

**Tier 1: Universal Primitives (100% extractable)**
- Typography system (font-family, sizes, weights, line-heights)
- Color palette (background, text, accent, border colors)
- Spacing system (padding, margins)
- Base component styles (table, blockquote, code)

**Tier 2: Compositional Patterns (50% extractable)**
- Section container patterns
- Callout box patterns (can be parameterized)
- Separator patterns
- Header/footer patterns

**Tier 3: Metaphor-Specific (NOT extractable as-is)**
- Section labels and terminology
- Page architecture
- Container relationships
- Component selection logic

### What NOT to Extract

1. **Specific metaphor layouts** — these are GENERATED, not reusable
2. **Metaphor terminology** — "Floor 1" vs "Lesson 1" is content, not style
3. **Section structure** — this varies completely by metaphor
4. **Container hierarchies** — these are composition decisions

### The Extraction Strategy

**Instead of extracting "templates", extract:**
1. **Token system** — colors, spacing, typography
2. **Component library** — tables, blockquotes, code blocks, callouts
3. **Composition grammar** — rules for combining components
4. **Metaphor engine** — the PIPELINE, not the OUTPUT

**The files themselves are EXAMPLES of what the pipeline can generate, not TEMPLATES to reuse.**

## Conclusion

The tension-test files confirm the hypothesis: **the tension-composition pipeline generates LAYOUTS, not components**. The extractable design system is the VOCABULARY (tokens, components, patterns), not the SENTENCES (complete page layouts).

**Critical Numbers:**
- 15 files audited
- 3 content sets × 5 metaphors
- ~75% of each file is metaphor-generated
- ~25% is from extractable vocabulary
- 0 soul violations
- 6-8 NEW component patterns vs core explorations

**Recommendation:**
Do NOT attempt to extract "Boris templates" or "Gas Town templates". Instead, extract:
1. The token system (colors, spacing, typography)
2. The component library (tables, callouts, code, etc.)
3. The composition patterns (how components combine)
4. The metaphor-to-layout transformation rules (the pipeline logic)

The files are THE PROOF that the pipeline works, not THE PRODUCT to extract.
