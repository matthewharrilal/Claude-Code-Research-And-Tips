SYSTEM — Remote Mac Control for AI Agents

Self-hosted AI assistant enabling remote macOS control through natural language

MIT License | 285+ Stars | by ygwyg

SYSTEM is a self-hosted AI assistant that enables remote control of macOS through natural language commands. Created by ygwyg, it's an MIT-licensed project with 285+ GitHub stars.

Core Capability: Tell Claude "play some jazz music" from your phone, and your home Mac starts playing Apple Music.

Why It Matters: The Brain/Tunnel/Body Architecture

SYSTEM introduces a distributed execution model that solves fundamental problems in AI agent design.

The Problem
Traditional AI agents face a dilemma: Local execution is fast/secure but tied to one machine. Cloud execution is accessible but can't touch local resources.

The Solution: Three-Layer Separation

┌──────────────────────────────────────────────────────────────┐
│  BRAIN (Cloudflare Worker)                                   │
│  - Claude AI with MCP                                        │
│  - HTTP-only, no system access                               │
│  - Stateless: reads/writes to KV store                       │
└────────────────┬─────────────────────────────────────────────┘
                 │
                 │  HTTP requests (JSON)
                 │
┌────────────────▼─────────────────────────────────────────────┐
│  TUNNEL (Cloudflare Tunnel / ngrok)                          │
│  - Exposes local server to internet                          │
│  - Zero firewall config                                      │
└────────────────┬─────────────────────────────────────────────┘
                 │
                 │  Local HTTP bridge
                 │
┌────────────────▼─────────────────────────────────────────────┐
│  BODY (macOS System)                                         │
│  - AppleScript execution                                     │
│  - Shell command allowlist                                   │
│  - Filesystem access                                         │
└──────────────────────────────────────────────────────────────┘

Why This Architecture Matters

Security Through Separation

Brain never has direct system access. Even if the AI is compromised, it can only send HTTP requests that the Body validates and sanitizes.

Scalability

One Brain, multiple Bodies (control home Mac + work Mac) or vice versa (multiple users sharing one Mac server).

Reliability

Brain failure doesn't affect Body operation. Your Mac keeps running even if Cloudflare Workers goes down.

Flexibility

Replace Brain with different AI providers (OpenAI, Anthropic direct, local LLM) without touching Body code.

Security

Threat Model

Threat Mitigation
Unauthorized API access Bearer token authentication (timing-safe comparison)
Command injection via shell Allowlist filtering + argument sanitization
AppleScript injection Pattern blocklist (system events, do shell script)
Rate-limit abuse 60 requests/minute per IP
Man-in-the-middle HTTPS-only tunnel (Cloudflare Tunnel enforces TLS)
Credential theft Token rotation, environment variables, no hardcoding

What SYSTEM Does NOT Protect Against

Threat Why Mitigation
Compromised API token Token = full access by design Rotate tokens, use Cloudflare Access, monitor audit logs
Malicious LLM output AI can request harmful commands Strict allowlists, manual command approval mode
Local privilege escalation Body runs with user permissions Run as limited user, never root
Physical Mac access Out of scope FileVault, screen lock, physical security

Security Layers

Layer 1: Authentication

Timing-safe bearer token comparison prevents timing attacks.

const isValid = crypto.timingSafeEqual(
  Buffer.from(providedToken),
  Buffer.from(expectedToken)
);
Layer 2: Command Filtering

Shell commands matched against explicit allowlist.

const ALLOWED_COMMANDS = [
  'osascript',
  'shortcuts',
  'open'
];

if (!ALLOWED_COMMANDS.includes(command)) {
  throw new Error('Command not allowed');
}
Layer 3: AppleScript Sanitization

Block dangerous patterns in AppleScript inputs.

const BLOCKED_PATTERNS = [
  /do shell script/i,
  /system events/i,
  /keystroke/i
];

if (BLOCKED_PATTERNS.some(p => p.test(script))) {
  throw new Error('Blocked pattern detected');
}
Layer 4: Rate Limiting

60 requests per minute per IP address.

const rateLimit = {
  windowMs: 60 * 1000,
  max: 60
};

Hardening Recommendations

  • Use Cloudflare Tunnel

    Avoid exposing ports. Cloudflare Tunnel creates outbound-only connections with automatic HTTPS.

    cloudflared tunnel --url http://localhost:3000
  • Enable Cloudflare Access

    Add email/SSO authentication before requests reach the Body.

  • Rotate Tokens Regularly

    Generate new bearer tokens monthly and update both Brain and Body configs.

  • Monitor Audit Logs

    Review executed commands for anomalies. Enable logging to file or syslog.

  • Restrict Shell Allowlist

    Remove unused commands from ALLOWED_COMMANDS. The smaller the allowlist, the smaller the attack surface.
