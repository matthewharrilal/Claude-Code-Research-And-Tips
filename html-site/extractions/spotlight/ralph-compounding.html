Core Insight

"What's amazing is I can ask it to look at previous runs to learn what worked or didn't work (I archive each PRD and user story JSON). So every time I run Ralph, it gets better and better." - Ryan Carson

The key insight: Each Ralph run can learn from all previous runs. Archive your PRDs, user stories, and learnings. Point the next run at those archives. Watch performance compound.

The Pattern

Traditional Ralph

Each run starts fresh, memory via:

  • progress.txt
  • git commits
  • Current prd.json

Compounding Ralph

Same as above PLUS:

  • Archived PRDs from previous runs
  • Archived user story JSONs
  • Agent references past successes/failures
The Difference

Traditional Ralph forgets everything between sessions. Compounding Ralph builds institutional knowledge.

Workflow

Run 1: Initial attempt -> Complete stories -> Archive prd.json, learnings Run 2: References Run 1 archives -> Better prompts, fewer failures -> Archive this run too Run 3: References Runs 1+2 -> Even better performance -> Continues compounding...

    └── ...

After Each Run

DATE=$(date +%Y-%m-%d)
mkdir -p archives/run-$DATE
cp scripts/ralph/prd.json archives/run-$DATE/
cp scripts/ralph/progress.txt archives/run-$DATE/

Prompt Strategy

Add this section to your prompt.md:

## Learning from Previous Runs

Before starting, review archives in `archives/` folder:
- What patterns worked well?
- What caused failures?
- What shortcuts were discovered?

Apply these learnings to current implementation.

The agent will then scan prior runs and extract:

  • Successful patterns (reuse them)
  • Common failures (avoid them)
  • Project-specific knowledge (apply it)

Parallel Execution

Ryan runs 3 simultaneous Ralph instances:

# Terminal 1: Payments feature
cd ~/project && ./scripts/ralph/ralph.sh 25 > logs/payments.log 2>&1 &

# Terminal 2: Auth feature (different worktree)
cd ~/project-worktree-auth && ./scripts/ralph/ralph.sh 25 > logs/auth.log 2>&1 &

# Terminal 3: Dashboard feature (different worktree)
cd ~/project-worktree-dashboard && ./scripts/ralph/ralph.sh 25 > logs/dashboard.log 2>&1 &

Mental Model

"Think of AI coding as an extremely persistent but context-limited junior developer who works 24/7. You become the product manager/architect."

The Transformation

Human Role AI Role --------------------------------------- Product Manager Implementation Army Architect Tireless Executor 25-30 min planning 8 hours grinding Morning review Overnight shipping

Key Principles

  1. Heavy upfront planning pays massive dividends
  2. Persistence > perfection in single prompt (iterate many times)
  3. Memory via artifacts (JSON, git, logs) > relying on long context
  4. Compounding returns - each run makes the next better
  5. Parallelization - run multiple Ralphs simultaneously
