2 The Core Abstraction

"Select and configure, don't build from scratch."
"Use @bootstrap-orchestrator to analyze and setup this project"
That command auto-detects your stack and selects optimal agents.

Claude 007 Agents is an agent library, not a framework. You get 117+ production-ready agents across 26 categories. The bootstrap system analyzes your codebase, detects your tech stack, and configures the right agents automatically.

This is analogous to using React vs building your own component system. Without 007, you write every agent from scratch. With 007, you npm install react—except it's agents.

3 Design Decisions

WHY PRE-BUILT AGENTS?
Most Claude Code users reinvent the wheel. Writing a good security review agent takes hours of prompt engineering and testing. Claude 007 encodes that work into reusable agents with battle-tested prompts.
What this means for you
Instead of writing 500 lines of security expertise encoding, invoke @security-specialist. The library handles prompt quality, tool configuration, and coordination patterns.
WHY BOOTSTRAP AUTO-DETECTION?
Manual agent selection is error-prone. A React project needs different agents than a Rails project. The bootstrap engine detects package.json, .py files, Gemfile, etc., and maps them to optimal agent combinations.
What this means for you
One command configures everything: @bootstrap-orchestrator. It generates your CLAUDE.md, selects tech-specific agents, and configures MCP servers. Setup time drops from hours to minutes.
WHY CHOREOGRAPHY WORKFLOWS?
Individual agents are easy. Making them work together is hard. @feature-development-dance encodes the sequence: requirements analysis -> technical design -> security review -> implementation -> testing -> documentation.
What this means for you
Instead of manually coordinating 6 agents, invoke one choreography agent. It handles handoffs, ensures security reviews happen before implementation, and maintains state across the workflow.

6 Gotchas

Real problems you'll hit, with concrete fixes:

#1: Bootstrap fails with "Cannot detect project type"
Symptom: Empty directory or unusual project structure
Time to fix: 30 seconds
Fix: Create minimal indicator file: touch package.json or touch requirements.txt
#2: Agent not found: "@react-expert"
Symptom: Agent files not in correct location
Time to fix: 2 minutes
Fix: cp claude-007-agents/.claude/agents/frontend/react-expert.md ~/.claude/agents/frontend/
#3: Too many agents activated, confusing responses
Symptom: Over-aggressive agent routing causing conflicts
Time to fix: 5 minutes
Fix: In CLAUDE.md, limit "Active Agents" to core only, move others to "On-Demand Agents"
#4: MCP connection fails
Symptom: Session state not persisting, memory not loading
Time to fix: 5 minutes
Fix: Test manually: npx -y @anthropic/mcp-basic-memory. Check GITHUB_TOKEN for GitHub MCP.
#5: Commit attribution format rejected
Symptom: Pre-commit hook enforcing 007 format
Time to fix: 1 minute
Fix: Use format: feat(auth): add OAuth2 - @security-specialist @nodejs-expert
#6: Session state lost after interruption
Symptom: Progress disappears between Claude restarts
Time to fix: 2 minutes
Fix: Always start with @session-manager start session for [task]. Resume with @session-manager resume last session
#7: Agents duplicating work or conflicting
Symptom: Multiple agents modifying same files
Time to fix: 5 minutes
Fix: Add Anti-Duplication Protocol to CLAUDE.md: "Before generating, search existing codebase for similar implementations"
#8: Node.js version mismatch
Symptom: Bootstrap engine fails silently
Time to fix: 5 minutes
Fix: Requires Node.js v18+. Check with node --version. Use nvm to upgrade.
#9: Vibe Coding takes longer than expected
Symptom: 15-20 minute prep phase feels slow
Time to fix: N/A - by design
Fix: This is intentional. Phase 1 (5-7 min) explores codebase, Phase 2 (8-10 min) plans implementation, Phase 3 (3-5 min) creates context. Don't skip.
#10: Global installation vs per-project
Symptom: Agents available in one project but not another
Time to fix: 2 minutes
Fix: Choose one: global (~/.claude/agents/) or per-project (.claude/agents/). Don't mix.

7 What's Hard

These are fundamental tensions, not bugs to fix:

Library Churn
The tension: 007 is actively maintained (121+ commits) but evolving rapidly. Agent APIs may change. New agents added frequently.
Symptoms: Agent behavior changes between updates. Deprecated agents cause errors.
Mitigation: Pin to specific commits for production. Check CHANGELOG before updating. Prefer stable, widely-used agents.
Agent Complexity Hidden
The tension: Pre-built agents hide their implementation. When they fail, debugging requires understanding 500+ lines of prompt engineering.
Symptoms: Agent produces wrong output. You don't know why. Black box frustration.
Mitigation: Read agent source files before using. Understand the prompts. Start with simpler agents, build up.
Coordination Overhead
The tension: More agents = more coordination complexity. 40+ agents requires sophisticated routing. Agent conflicts increase non-linearly.
Symptoms: High-complexity projects (7-10) have agents stepping on each other. Context pollution.
Mitigation: Start with fewer agents. Add only when needed. Use choreography workflows to manage handoffs.
MCP Dependency Chain
The tension: Full 007 functionality requires 4 MCP servers (basic-memory, github, task-master, sequential-thinking). Each is a potential failure point.
Symptoms: Some features work, others don't. Partial functionality is confusing.
Mitigation: Test each MCP independently. Use 007 in degraded mode (without all MCPs) for simpler projects.
Knowledge Transfer Gap
The tension: 007 encodes expert knowledge, but using it doesn't teach you WHY it works. You use @security-specialist without learning security.
Symptoms: Dependence on agents. Can't debug or customize. Blind trust.
Mitigation: Read agent prompts as learning material. Use @rubber-duck-debugger for Socratic learning. Build one agent from scratch first.

8 When to Use / When Not

USE CLAUDE 007 WHEN
Need quick multi-agent setup (< 5 minutes to productive)
Standard tech stack (React, Node, Rails, Django, Go)
Want production-tested agent prompts without writing them
Need session persistence and recovery
Integrating with Ralph, Gas Town, or CC-Mirror (pre-built Dogs)
DON'T USE CLAUDE 007 WHEN
Highly custom domain (agents won't fit your needs)
Need to understand every prompt decision (black box)
Minimal project (single agent is simpler)
Sensitive to library churn (need stability)
Learning Claude Code from scratch (start vanilla first)
