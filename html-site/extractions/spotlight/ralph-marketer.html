Core Insight

Claude Code plugin that iteratively drafts, self-critiques, and refines marketing content until it passes "founder quality" standards in your voice.

The Loop

DISCOVER --> LEARN --> RESEARCH --> IDEATE --> WRITE --> CRITIQUE | "Would founder publish?" | No --> ITERATE --> back to CRITIQUE Yes --> PUBLISH

This is the Ralph pattern applied to content creation instead of code. Same fundamental loop - iterate until quality bar is met - but with domain-specific phases for marketing content.

Key Innovation: Discovery vs Synthesis

From the author's reply to @mattyp:

"Ralph doesn't have the LLM 'discover' things on its own. Its discovery is a database-driven input system... The agent's job is synthesis."

Critical Insight

Don't let AI discover - it hallucinates. Pre-populate a SQLite DB with verified information. AI then SYNTHESIZES unique angles from known data, not discovering new facts.

What to Pre-Populate

  • Your content/voice samples - Existing writing that captures your style
  • Competitor analysis - What others are saying in your space
  • Market trends - Verified data points from reliable sources
  • Internal communications - Product updates, key messages

This separation is the key to reliable content generation. The AI synthesizes, humans verify the input data.

Architecture

Multi-Agent Ecosystem

+---------------------------------------------+ | External Agents (feed the DB) | | - TrendScout | | - Research Agent | | - Product/Marketing Intel | +---------------------------------------------+ | v +---------------------------------------------+ | SQLite Database (single source of truth) | | - trends | | - research | | - communications | | - content_plan | | - drafts | | - published | +---------------------------------------------+ | v +---------------------------------------------+ | Ralph Copywriter Agent | | - Reads prd.json for tasks | | - Checks progress.txt for learnings | | - Writes --> Critiques --> Iterates | | - Commits to git | +---------------------------------------------+

Database Schema

-- Core Tables
trends (topic, description, source, relevance_score, status)
research (title, summary, key_findings, data_points, category, status)
communications (type, title, details, key_messages, target_audience, priority, status)
content_plan (content_type, title, brief, target_keywords, status)
drafts (plan_id, version, content, word_count, feedback)
published (plan_id, final_content, meta_description)
agent_log (action, details, created_at)

This schema separates concerns clearly: external data goes into trends/research/communications, the agent's work goes into content_plan/drafts/published.

Mental Model

"View Claude Code as a persistent, file-memory-based loop where AI acts as an autonomous copywriter, breaking projects into small, verifiable stories that compound over iterations - emphasizing eventual consistency over perfect first attempts."

Key Principles

  1. Small stories - Complete in one iteration
  2. Explicit acceptance criteria - Verifiable completion
  3. Fast feedback via critiques - Self-correction loop
  4. Compound learnings in files - External memory
  5. Separate discovery (human/DB) from synthesis (AI) - No hallucination risk

The Original Ralph Loop

At its core, this is the same pattern:

while :; do cat PROMPT.md | claude-code ; done

Everything else is scaffolding around this fundamental loop.

Caveats and Limitations

Discovery MUST be pre-populated

AI can't reliably find new facts. The database needs real content before you can generate quality output.

Each iteration = fresh context

Persist learnings in files. The agent doesn't remember between iterations without external memory.

Initial outputs may be bad

"Deterministically bad" failures are expected. They improve with iteration. Don't judge by first draft.

Mock DB needs real content

For personalized output, you need to seed with YOUR voice samples and YOUR product details.

Long loops consume resources

Use /ralph-cancel if stuck. Set reasonable iteration limits.
