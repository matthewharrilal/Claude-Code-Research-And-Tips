2 The Core Abstraction

"You're a hiring manager with a pool of AI developers. Each explores a different implementation universe. Pick the best timeline."
Traditional: You -> Single Agent -> Wait -> Maybe Retry
Superconductor: You -> [Agent 1, 2, 3, 4] -> Compare -> PR Best
Parallel exploration replaces serial iteration. 90 minutes becomes 3 minutes.

Superconductor inverts the traditional AI coding workflow. Instead of running one Claude session, waiting, evaluating, maybe trying again, you launch 3-5 agents simultaneously on the same task. Each agent works in its own isolated cloud sandbox with your full repository.

The magic: each agent gets a live browser preview of its implementation. You swipe between them on your phone, comparing actual visual results---not just code diffs. When you find the winner, one tap creates the PR.

3 Design Decisions

WHY PARALLEL AGENTS (NOT SEQUENTIAL)?
Traditional workflow: Attempt 1 (30 min) -> Bad -> Attempt 2 (30 min) -> Better -> Attempt 3 (30 min) -> Good. Total: 90 minutes serial. With parallel agents: All 3 attempts run simultaneously (~30 seconds to previews). Pick best in 2 minutes. Total: ~3 minutes. Time collapses by 30x.
What this means for you
Pay ~$5 more in API costs (~$20 parallel vs ~$15 serial with retries), save 87 minutes. For most developers, time is worth more than the marginal cost.
WHY BYOK (BRING YOUR OWN KEY)?
Superconductor doesn't sell AI access. You bring your own API keys (Anthropic, OpenAI, Amp, Gemini, Opencode). This means no markup on AI costs, you use existing subscriptions, and you choose which providers to use.
What this means for you
You manage your own API costs directly with providers. Claude Max subscribers can use their existing subscription. No hidden fees on the AI side.
WHY NATIVE iOS (NOT WEB-ONLY)?
Native iOS provides push notifications (agents come to you), swipe gestures (natural preview comparison), offline ticket list, and iPad multitasking (preview + chat side-by-side). A web app can't match this mobile UX.
What this means for you
Requires Apple device (iOS 16.6+, iPadOS 16.6+, or macOS 13.5+ with Apple Silicon). No Android app yet. The tradeoff for excellent mobile UX is platform lock-in.
WHY LIVE BROWSER PREVIEWS?
Code diffs are hard to evaluate on mobile. "Did Agent 1's dark mode look right?" is easier to answer by seeing it than by reading CSS. Visual comparison surfaces bugs that code review misses.
What this means for you
Your app must be preview-able (web frameworks work well). Backend-only changes get less value from visual previews. Static sites, React, Next.js, Rails---all work great.
WHY GITHUB INTEGRATION?
One-click PR creation is the goal. Deep GitHub integration means PRs appear with proper attribution, CI runs immediately, and you can @ mention Superconductor in comments to spawn agents directly from GitHub.
What this means for you
GitHub-centric workflow assumed. If you use GitLab, Bitbucket, or other providers, Superconductor may not fit your workflow today.

7 Gotchas

Real problems you'll hit with Superconductor, with fixes:

#1: Cost multiplied by agents
Symptom: $20/feature feels expensive compared to $5 single-agent
Reality: Parallel = 3-4x single cost, but saves 90%+ time
Fix: Budget accordingly. Use Sonnet ($2-4) for exploration, Opus ($6-10) only for final polish. Time saved > cost increase for most developers.
Time to fix: Awareness shift
#2: Agent spawning takes too long
Symptom: Stuck on "Spawning..." for >2 minutes
Causes: Large repo (>1GB), heavy deps, build failure
Fix: Check if build works locally. Verify API key has credits. Check agent logs in web interface.
Time to fix: 5-15 min investigation
#3: Preview not loading
Symptom: Preview button active but shows blank/error
Causes: App not actually running, wrong port, build failed silently
Fix: Check agent logs for build errors. Verify app's start command. Pull down to refresh preview.
Time to fix: 5 min
#4: Push notifications not arriving
Symptom: Miss completed agents, no alerts
Causes: iOS notifications disabled, Focus mode blocking
Fix: iOS Settings > Superconductor > Notifications > Enable All. Check Focus modes.
Time to fix: 2 min
#5: GitHub PR creation fails
Symptom: "Create PR" button errors
Causes: GitHub token expired, branch protection rules
Fix: Re-authorize GitHub in Superconductor settings. Check branch protection allows app.
Time to fix: 5 min
#6: API key invalid
Symptom: "Invalid API key" when launching agent
Causes: Key copied with whitespace, key expired/revoked
Fix: Re-copy key carefully from provider dashboard. Test key via curl.
Time to fix: 2 min
#7: API rate limited
Symptom: Agent fails mid-work with rate limit error
Causes: Provider limits hit (especially with multiple parallel agents)
Fix: Wait and retry. Consider provider plan upgrade. Stagger agent launches.
Time to fix: Wait time varies
#8: Build fails silently
Symptom: Agent proceeds but app doesn't work
Causes: Build errors not surfaced, agent didn't check build status
Fix: Check agent logs for hidden errors. Verify build commands in project config.
Time to fix: 10 min investigation

8 What's Hard

These are fundamental tensions in parallel mobile orchestration, not bugs to fix:

Cost Scales with Parallelism
The tension: More agents = more API costs, always. 4 agents = ~4x single agent cost. There's no free lunch.
Symptoms: API bills higher than expected. Single-agent users shocked at parallel costs.
Coping: Budget $15-25/feature. Use Sonnet for exploration, Opus for final. Accept that time saved is worth the premium. Or use fewer agents for simpler tasks.
Not All Tasks Benefit from Parallel
The tension: Simple, well-defined tasks don't need 4 different approaches. Parallel shines when you're genuinely unsure which approach is best.
Symptoms: 4 agents produce nearly identical output. Wasted cost on trivial changes.
Coping: Reserve parallel for genuinely uncertain tasks. Use single agent for "add button here" changes. Match agent count to uncertainty level.
Preview Requires Runnable App
The tension: Live browser previews only work for web-previewable apps. Backend-only, complex build pipelines, or non-web projects get less value.
Symptoms: Preview shows error or nothing for backend changes. CLI tool changes can't be previewed.
Coping: Check if your project has a preview-able frontend. Backend-only projects: rely on diff view instead. Accept that some project types fit better than others.
Managed Service Dependency
The tension: Superconductor requires their cloud infrastructure. No self-hosting option. You depend on Volition, Inc. for uptime.
Symptoms: Service outage = no agent orchestration. Enterprise security requirements may conflict.
Coping: Accept vendor dependency or use SSH+tmux alternative (001-mobile-claude-code-dabit3.md) for full control. Evaluate based on your security requirements.
iOS Ecosystem Lock-in
The tension: Apple devices only. No Android app yet. macOS requires Apple Silicon (M1+).
Symptoms: Android team members can't use. Intel Mac users excluded.
Coping: If you need cross-platform, use SSH+tmux approach instead. Watch for future Android app.
