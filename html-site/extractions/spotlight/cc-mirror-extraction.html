2 The Core Abstraction

"Task tool spawns a REAL Claude Code instance, not a weak subagent."
Task(subagent_type="general-purpose", description="...", prompt="...", run_in_background=True)
That's the primitive. Everything else is orchestration around it.

The key discovery: Claude Code's Task tool spawns subagents that have ALL the same capabilities as your main session. They can Read, Write, Edit, Bash, Grep, Glob - everything. This is NOT documented. Anthropic built it but disabled it. CC Mirror enables it.

This transforms you from a programmer to a conductor. You decompose work, spawn workers, coordinate merges. The workers do all the implementation. You orchestrate.

The Iron Law
YOU DO NOT WRITE CODE.
YOU DO NOT RUN COMMANDS.
YOU DO NOT EXPLORE CODEBASES.

You are the CONDUCTOR. Your agents play the instruments.

Violating the Iron Law causes: context pollution (orchestrator loses strategic view), verification bypass (same agent writes and approves), and uncontrolled recursion (workers spawning workers).

3 Design Decisions

WHY TASK TOOL (NOT CUSTOM ORCHESTRATOR)?
The Task tool is built-in, already optimized by Anthropic, requires zero infrastructure. Custom orchestrators add complexity, dependencies, and maintenance burden.
What this means for you
Zero setup. Install CC Mirror, run 3 commands, and you have multi-agent orchestration. No Docker, no custom APIs, no external services.
WHY IS THIS A "HIDDEN" PATTERN?
Anthropic built these capabilities but disabled them - likely for stability, cost control, and support burden reasons. The features exist in the minified JS. CC Mirror patches them to enable.
What this means for you
You're using a beta/disabled feature. It works, but may change without notice. Accept the risk, follow community updates, and have fallback plans.
WHY AGENTS.MD FOR COORDINATION?
Same mechanism as CLAUDE.md - a file that all agents read. Consistent pattern, easy to adopt. The orchestrator updates it after each coordination round to track state.
What this means for you
If you understand CLAUDE.md, you understand AGENTS.md. Same mental model, same file-based coordination, same debugging approach.
WHY GIT WORKTREES FOR PARALLELISM?
True file isolation. Each worktree is a complete codebase on a separate branch. Workers can't conflict because they're working in different directories.
What this means for you
Spawn 3 workers on 3 worktrees = 3 parallel feature branches with zero file conflicts. Same codebase, different branches, simultaneous development.
WHY "80% OF USE CASES"?
From @nummanali: "Simplicity covers 80% with zero effort - mass-market ease." Most work doesn't need factory-scale infrastructure. CC Mirror is the pragmatic middle ground.
What this means for you
Don't over-engineer. CC Mirror handles parallel features, exploration, and medium complexity. Only escalate to Gas Town when you actually need factory-scale.

6 Gotchas

Real problems you'll hit, with concrete fixes:

Workers editing same files
Symptom: Merge conflicts on completion
Time to detect: At merge time (too late)
Fix: Use worktrees, assign clear file boundaries BEFORE spawning. 10-30 min to resolve if you miss it.
Task output too long
Symptom: Lost or truncated worker results
Cause: Task output has size limits
Fix: Workers write to files, summarize in output. "See .worktrees/cart/RESULTS.md for details." 5 min fix.
Orchestrator forgets workers
Symptom: Lost coordination, duplicate work spawned
Cause: Context collapse in long sessions
Fix: Enable auto-compact beta feature. Track in AGENTS.md. Use /compact. 5 min to recover.
Too many parallel workers
Symptom: Overwhelmed orchestrator, lost tracking
Threshold: >5 workers = coordination overhead exceeds benefit
Fix: Max 3-5 workers per orchestrator. 2 min to apply (just spawn fewer).
Orchestrator writes code (breaks Iron Law)
Symptom: Context pollution, verification bypass
Cause: Habit from single-session work
Fix: Strict discipline. Review SKILL.md rules. If you catch yourself coding, spawn a worker instead. 5 min mental reset.
Workers spawning sub-workers
Symptom: Uncontrolled recursion, cost explosion
Cause: Worker preamble doesn't explicitly prohibit
Fix: Explicit prohibition in worker preamble: "Do NOT spawn sub-agents". 2 min to add.
Windows compatibility issues
Symptom: Installation fails on Windows
Workaround: Use WSL2, check community PRs for fixes. 30 min to set up WSL2.

7 What's Hard

These are fundamental tensions, not bugs to fix:

Coordination Overhead
The tension: Orchestrator must track all workers, context grows with each worker. More parallelism = more coordination cost.
Symptoms: Orchestrator loses track after 6+ workers. Duplicate work spawned. Merge order confusion.
Mitigation: Keep worker count low (3-5). Clear AGENTS.md updates after each round. Accept that some coordination overhead is the cost of parallelism.
No Shared Memory Between Workers
The tension: Workers can't see each other's work. If Worker B needs Worker A's output, must wait and coordinate through orchestrator.
Symptoms: Deadlock when dependencies unclear. Workers making incompatible assumptions.
Mitigation: Plan sequential dependencies BEFORE spawning. Use filesystem coordination (worker writes to agreed path). Accept that some work must be sequential.
Mental Model Shift
The tension: Must think like a PM/architect, not a programmer. Instinct says "just write the code" but you must delegate.
Symptoms: Breaking Iron Law repeatedly. Frustration with indirection. Slower than expected initially.
Mitigation: Practice with 2 workers first. Build intuition for decomposition. Expect initial slowdown before speedup.
Beta/Disabled Feature Risk
The tension: CC Mirror enables features Anthropic disabled. May change, break, or be officially blocked.
Symptoms: Updates break existing workflows. No official support. Community-dependent.
Mitigation: Accept the risk. Follow community updates. Have fallback plan (single session, Ralph). This is the price of cutting edge.

8 When to Use / When Not

USE CC MIRROR WHEN
You've mastered Ralph (Stage 4+) and understand orchestration
Task has 2-5 parallelizable streams with clear boundaries
Daily budget $20-50 (multiple agents but not extreme)
You want "team output from solo developer"
Zero setup is important (no infrastructure)
DON'T USE CC MIRROR WHEN
Sequential tasks (no parallelism benefit) -> Use Ralph
Factory-scale output (10+ agents regularly) -> Use Gas Town
Budget-constrained ($5-10/day) -> Single session + Ralph
Unclear boundaries (need to explore first) -> Single session
Need official Anthropic support -> Wait for GA
