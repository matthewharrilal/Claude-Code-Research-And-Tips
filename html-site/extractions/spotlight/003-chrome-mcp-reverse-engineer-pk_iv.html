2 The Core Abstraction

"I spent all of Christmas reverse engineering Claude Chrome so it would work with remote browsers."
- @pk_iv (Paul Klein IV), Founder of Browserbase

Claude Code has undocumented browser automation capabilities through a Chrome extension. The extension acts as an MCP server, exposing 19 tools (navigate, screenshot, click, get_text, etc.) that Claude can use to control your local browser.

The Protocol Format

Communication uses a binary protocol over Chrome's native messaging: 

// Message format:
[4 bytes: message length (little-endian)]
[JSON payload with tool call]

// Example tool call:
{
  "tool": "mcp__claude-in-chrome__navigate",
  "params": {
    "url": "https://status.github.com"
  }
}

The 19 Available Tools

All tools use the mcp__claude-in-chrome__* prefix:

  • navigate - Go to URL
  • screenshot - Capture visible page
  • click - Click element
  • get_text - Extract page text
  • fill - Fill form fields
  • submit - Submit forms
  • ...and more for scrolling, waiting, tab management
Cloud Architecture (The Solution) 
Claude Code (MCP Client)
        |
    Binary Protocol
        |
Intercepting Server (MCP -> CDP Translation)
        |
Browserbase Cloud Browsers
(Isolated, no personal data)

The intercepting server translates MCP tool calls to Chrome DevTools Protocol (CDP) commands. Claude believes it's using local Chrome - the same 19 tools work seamlessly - but actual execution happens in isolated cloud browser sessions.

3 Design Decisions

WHY BINARY PROTOCOL (NOT PURE JSON)?
Chrome's native messaging requires a specific format: 4-byte little-endian length prefix followed by the payload. This is Chrome's constraint, not Anthropic's choice.
What this means for you
You can't just send JSON over a pipe. The intercepting proxy must handle the binary framing correctly. This is low-level protocol work.
WHY LOCAL CHROME IS DANGEROUS?
"Only local Chrome; dangerous due to prompt injection -> potential data leak from personal browser." Your cookies, sessions, and history are exposed to whatever content Claude processes. If Claude navigates to a malicious page, prompt injection could exfiltrate your data.
What this means for you
NEVER use local Chrome MCP for untrusted sites or production automation. Your banking cookies, email sessions, and personal accounts are all at risk. This is a security emergency waiting to happen.
WHY MCP -> CDP TRANSLATION?
CDP (Chrome DevTools Protocol) is the standard for browser automation (Puppeteer, Playwright). By translating MCP to CDP, you get protocol compatibility with any CDP-speaking service - including cloud browsers like Browserbase.
What this means for you
The translation layer is where security happens. Claude's MCP calls are intercepted, translated, and executed remotely. Your local browser is never touched.
WHY THIS IS UNOFFICIAL?
Paul reverse-engineered this over Christmas. Anthropic hasn't documented or officially supported these capabilities. The protocol could change with any Claude Code update.
What this means for you
Pin your Claude Code version if you rely on this. Have a fallback plan. Monitor Claude Code releases for breaking changes.

5 Path of a Task

Trace "Check GitHub status and screenshot it" through both local and cloud execution paths:

Local Path (RISKY)

1
[T+0ms] Claude Code receives request "Check GitHub status"
2
[T+10ms] MCP tool call generated: mcp__claude-in-chrome__navigate
3
[T+20ms] Binary protocol encoding: 4-byte LE length + JSON payload
4
[T+50ms] Chrome extension receives via native messaging host
5
[T+500ms] Page loads in YOUR browser (YOUR cookies, sessions active)
6
[T+550ms] Screenshot tool called: mcp__claude-in-chrome__screenshot
7
[T+650ms] Claude analyzes screenshot: "GitHub status: All systems operational"

Cloud Path (SAFE)

1
[T+0ms] Claude Code receives same request
2
[T+10ms] Same MCP tool call generated (Claude doesn't know the difference)
3
[T+20ms] Intercepting proxy receives MCP call
4
[T+100ms] Proxy translates MCP -> CDP commands
5
[T+200ms] Browserbase cloud browser receives CDP
6
[T+600ms] Page loads in ISOLATED cloud browser (NO personal data)
7
[T+700ms] Screenshot captured via CDP
8
[T+750ms] Claude analyzes: Same result, but ISOLATED execution

Latency difference: ~100ms extra for cloud path. Worth it for security.

6 Gotchas

Security-focused warnings and practical problems:

#1 CRITICAL: PROMPT INJECTION
Symptom: AI tricked by malicious page content into exfiltrating your data
Risk: Local browser = your cookies, sessions, banking, email all exposed
Fix: NEVER use local Chrome for untrusted content. Cloud browsers ONLY for anything external.
#2 Extension Required
Symptom: Browser mode doesn't activate
Fix: Install "Claude for Chrome" extension. Run once manually first to complete OAuth flow.
#3 Anti-Bot Measures
Symptom: Captchas, blocks on cloud browsers
Cause: Sites detect headless/cloud browser fingerprints
Fix: Use residential proxies through Browserbase, or accept limitation for those sites.
#4 Local Proxy Still Required
Symptom: Cloud setup still needs local component running
Why: MCP -> CDP translation happens locally before sending to cloud
Fix: Run the proxy script before using browser features.
#5 Verbose Screenshots
Symptom: Claude takes screenshots instead of extracting text
Cause: Model prefers visual verification
Fix: Request specific data format: "Extract the text, don't screenshot"
#6 Unofficial / Could Break
Symptom: Protocol changes with Claude Code updates
Risk: Reverse-engineered, not documented by Anthropic
Fix: Pin Claude Code version. Monitor releases. Have fallback plan.
#7 Session State Isolation
Symptom: Local browser actions persist (logged in, cookies set)
Risk: Each task affects your actual browser state
Fix: Cloud browsers have fresh sessions. No state leakage.
#8 Cost of Cloud Browsers
Symptom: Browserbase usage costs money
Mitigation: Monitor usage, set budgets, batch operations where possible.

7 What's Hard

These are fundamental tensions, not bugs to fix:

Local Browser = Your Identity
The tension: Local Chrome is convenient (instant, no setup), but it's YOUR browser with YOUR identity.
Why no perfect solution: You can't have "local convenience" AND "identity isolation" simultaneously. Physics.
Mitigation: Accept the tradeoff. Cloud browsers for anything external. Local only for trusted, self-controlled sites (if ever).
Prompt Injection is Fundamental
The tension: AI must process web content to understand it. Web content can contain malicious instructions.
Why unsolved: This is an open research problem in AI safety. No production solution exists for trusting arbitrary web content.
Mitigation: Isolation. Never let AI process untrusted content in a context that has access to sensitive data.
Unofficial Protocol Status
The tension: This works because it was reverse-engineered. Anthropic hasn't blessed it.
Why risky: Any Claude Code update could change the protocol. No deprecation warnings.
Mitigation: Pin versions. Monitor releases. Have non-browser fallbacks for critical workflows.
Anti-Bot vs. Automation
The tension: Websites actively fight against automated browsers. Cloud browsers often get flagged.
Why hard: Cat-and-mouse game. Detection evolves. Evasion evolves. Repeat.
Mitigation: Accept some sites won't work. Use residential proxies. Don't rely on browser automation for critical paths.
