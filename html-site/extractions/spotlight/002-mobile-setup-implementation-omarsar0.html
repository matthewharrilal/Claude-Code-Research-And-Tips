3 Design Decisions

WHY PreToolUse (NOT PostToolUse)?
PreToolUse fires BEFORE the tool runs. For AskUserQuestion, this means you get notified BEFORE Claude waits for your input. PostToolUse would notify AFTER the question was asked and answered - useless for mobile.
What this means for you
Always use PreToolUse for notifications. PostToolUse is for logging/auditing after the fact.
WHY jq (NOT Python/Node)?
jq is purpose-built for JSON in shell scripts. It's lightweight (single binary), reliable, and standard on most systems. Python or Node would add dependencies and complexity for a simple JSON extraction.
What this means for you
Install jq: apt install jq. Use it for all JSON parsing in hooks.
WHY HASH-BASED PORTS (NOT Random)?
Hash-based ports are deterministic: same branch name always gets the same port. This means you can bookmark dev server URLs. Random ports would change every session, breaking your workflow.
What this means for you
Theoretical collision risk exists (~1% for 10 branches), but determinism is worth it. Override manually if collision occurs.
WHY CUSTOM SSH PORT 47892 (NOT 22)?
Security through obscurity as an addition to other measures (VPN, key auth). Port 22 is targeted by automated scanners. Custom port reduces noise in logs and adds a small barrier.
What this means for you
Choose any high port (1024-65535). Update SSH config: Port 47892 in /etc/ssh/sshd_config.

4 File Structure

Complete file locations with copy-paste ready code. Every path is absolute.

File Locations Summary

~/.claude/settings.json
Hook configuration - tells Claude when to run notification script
~/.claude/hooks/poke-notify.sh
Notification script - sends push to your phone
~/.zshrc (or ~/.bashrc)
tmux auto-attach - session persistence on login
~/Code/myproject/
Git worktrees - each branch in separate directory

~/.claude/settings.json

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "AskUserQuestion",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/hooks/poke-notify.sh question"
      }]
    }]
  }
}

~/.claude/hooks/poke-notify.sh

#!/bin/bash
# Mobile notification script for Claude Code
# Sends push notification when Claude needs user input

# Extract the question from Claude's event data
QUESTION=$(echo "$EVENT_DATA" | jq -r '.tool_input.questions[0].question')

# Your project name (set this per-project or use pwd)
PROJECT_NAME=$(basename "$(pwd)")

# Build the notification message
MESSAGE="$PROJECT_NAME: Claude needs input - $QUESTION"

# Send to Poke (replace with your API URL and token)
curl -s -X POST "https://api.pokepush.com/notify" \
  -H "Authorization: Bearer $POKE_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"$MESSAGE\"}"

After creating: chmod +x ~/.claude/hooks/poke-notify.sh

~/.zshrc - tmux Auto-Attach

# tmux auto-attach: join existing session or create new
if [[ -z "$TMUX" ]]; then
  tmux attach -t main 2>/dev/null || tmux new -s main
fi

Port Allocation Script

# port_for_branch.py - Deterministic port from branch name
import sys

branch_name = sys.argv[1] if len(sys.argv) > 1 else "main"
hash_val = sum(ord(c) for c in branch_name)
django_port = 8001 + (hash_val % 99)

print(django_port)

5 Path of a Notification: Follow the Signal

Like PostgreSQL's "follow the query," trace a mobile notification from Claude's need to your phone buzzing. See every decision point and what happens at each stage.

1
Claude Prepares to Ask
Claude is working autonomously. It encounters ambiguity and needs clarification. It prepares to call AskUserQuestion tool.
Internal state: Tool call queued, not yet executed
2
PreToolUse Hook Fires
BEFORE the tool executes, Claude Code checks ~/.claude/settings.json. It finds PreToolUse hook with matcher AskUserQuestion.
Decision Branch: Does matcher match? AskUserQuestion == AskUserQuestion? YES -> Execute hook
3
Hook Script Spawns
Claude Code executes ~/.claude/hooks/poke-notify.sh question. The script receives event data via $EVENT_DATA environment variable (JSON payload).
Failure Point: If script not executable -> "permission denied" -> silent failure, no notification
4
jq Extracts Question
Script runs: echo "$EVENT_DATA" | jq -r '.tool_input.questions[0].question'
Extracts the actual question text Claude wants to ask you.
Failure Point: If jq not installed -> empty QUESTION variable -> notification says "Claude needs input - "
5
curl Sends to Poke API
Script constructs message: "$PROJECT_NAME: Claude needs input - $QUESTION" and POSTs to Poke API with your token.
Decision Branch: API responds 200? YES -> Notification queued. NO (401) -> Token invalid, check dashboard.
6
Poke Server Delivers
Poke's infrastructure routes the push notification to your phone via APNs (iOS) or FCM (Android). Typically 1-2 seconds.
Observable: Phone buzzes, notification appears
7
Meanwhile: Claude Waits
Back on the server, Claude's AskUserQuestion tool has now executed. Claude is blocked, waiting for your response. The terminal shows the question prompt.
Key Insight: PreToolUse fired BEFORE Claude started waiting. You're notified the moment Claude needs you, not after.
8
You Connect via Mosh
Open Termius/Blink on phone. Run mosh --ssh="ssh -p 47892" user@100.x.x.x. Mosh handshakes, attaches to your server.
Decision Branch: Existing tmux session? YES -> Auto-attach. NO -> Create new session.
9
tmux Auto-Attaches
Your .zshrc runs tmux attach -t main 2>/dev/null || tmux new -s main. You're immediately in the same session where Claude is waiting.
Observable: You see Claude's question prompt exactly as you left it
10
You Respond, Claude Continues
Type your answer, press Enter. Claude receives input, resumes work. The loop continues autonomously until the next question.
Total latency: ~2-5 seconds from Claude's need to your phone buzz

6 Gotchas

Implementation-specific issues with concrete fixes and verification.

mosh doesn't forward SSH agent
Symptom: git push returns "Permission denied (publickey)"
Fix: Inside tmux, run ssh-add ~/.ssh/id_ed25519
Verify: ssh-add -l shows your key
settings.json invalid JSON
Symptom: Hooks don't fire, no errors shown
Fix: Validate with cat ~/.claude/settings.json | jq .
Verify: jq outputs formatted JSON without errors
Hook script not executable
Symptom: Hooks don't fire, "permission denied" in logs
Fix: chmod +x ~/.claude/hooks/poke-notify.sh
Verify: ls -la shows -rwxr-xr-x
$EVENT_DATA not available
Symptom: Script runs but QUESTION is empty
Fix: Ensure hook type is "command" (not "url")
Verify: Add echo "$EVENT_DATA" >> /tmp/debug.log to script
jq not installed
Symptom: Script fails silently, no notification
Fix: apt install jq or brew install jq
Verify: which jq returns path
Poke API token invalid
Symptom: No notifications, curl returns 401
Fix: Re-copy token from Poke dashboard, check for trailing spaces
Verify: curl -v ... returns 200
Port collision between branches
Symptom: "Address already in use" when starting dev server
Fix: Use different branch names, or manually set port
Verify: lsof -i :8047 shows which process
tmux session name conflict
Symptom: "duplicate session: main" error
Fix: Kill existing: tmux kill-session -t main
Verify: tmux ls shows sessions
Worktree on same branch
Symptom: "fatal: 'feature/x' is already checked out" error
Fix: Each worktree needs unique branch
Verify: git worktree list shows distinct branches
Relative path in hook command
Symptom: "command not found" error
Fix: Use absolute path: ~/.claude/hooks/poke-notify.sh
Verify: Run command manually in terminal

7 What's Hard

Fundamental implementation challenges, not simple fixes:

JSON Event Structure Changes
The tension: Claude updates may change the event JSON format. Your jq query that works today might break tomorrow.
Why hard: No versioned API contract for hook events.
Mitigation: Pin Claude version in production. Check release notes before updating. Add fallback paths in script.
Notification Service API Changes
The tension: Poke, Pushover, ntfy all have their own APIs. If they change, your script breaks.
Why hard: External dependency you don't control.
Mitigation: Abstract notification layer. Create notify() function that can swap backends.
Cross-Shell Compatibility
The tension: zsh and bash have subtle differences. Auto-attach that works in zsh might fail in bash.
Why hard: Shell scripting is not standardized.
Mitigation: Test in both. Use POSIX-compatible syntax where possible. Document shell-specific quirks.
Hook Execution Environment
The tension: Hooks run in a limited environment. Not all environment variables available. PATH may differ.
Why hard: Isolation is intentional but poorly documented.
Mitigation: Use absolute paths for all commands. Export required variables explicitly. Debug with logging.
Worktree Merge Complexity
The tension: Multiple worktrees = multiple active branches = complex merge situations.
Why hard: Git fundamentally assumes single working directory.
Mitigation: Merge frequently. Keep feature branches short-lived. Use clear naming conventions.
