3 Design Decisions

WHY LEARNINGS FOLDER? (Klaassen)
Knowledge compounds only if captured. Traditional development loses insights to tribal knowledge. The learnings/ folder creates institutional memory that persists across sessions and team members.
What this means for you
Each feature should contribute to institutional memory. After completing work, document: what patterns worked (patterns.md), what decisions were made (decisions.md), what mistakes to avoid (mistakes-to-avoid.md).
WHY 99 AGENTS? (Hobson)
Specialized agents force systematic task decomposition. Instead of "build a feature," you think: What does backend-architect need to decide? What does security-auditor need to review? The taxonomy is the discipline.
What this means for you
Don't load all 99 agents—load exactly what you need (3-5 per workflow). Match models to task complexity: Opus for strategic decisions, Sonnet for implementation, Haiku for routine tasks.
WHY RUST? (nwiizo)
Type-state patterns prevent invalid state transitions at compile time. If reliability matters—production systems, long-running automation—runtime errors aren't acceptable. Systems languages bring production rigor.
What this means for you
If you need quick prototyping, use Python. If you need production reliability, consider Rust. The learning curve is real but the payoff is compile-time guarantees.
WHY DATABASE-DRIVEN DISCOVERY? (Koylan)
AI hallucinates when asked to discover facts. By pre-populating a database with verified data (trends, research, communications), you ground the AI in reality. Its job becomes synthesis, not discovery.
What this means for you
Separate data collection from content creation. Use external agents or humans to populate your database first. The AI synthesizes unique angles from known data—creativity without confabulation.

6 Gotchas

Per-practitioner problems with specific fixes:

Compound Engineering

Learnings not being read
Symptom: Claude ignores patterns from .claude/learnings/, repeats documented mistakes
Cause: Files not referenced in CLAUDE.md or wrong directory structure
Fix: Add explicit reference: "Read .claude/learnings/ for past decisions before starting work." Time: 2 min
Knowledge not compounding
Symptom: Same mistakes repeated, no velocity improvement over time
Cause: Not documenting learnings after each feature (compound step skipped)
Fix: Add post-feature checklist to CLAUDE.md. Time: 5 min

99 Agents

Agent not found
Symptom: @backend-architect returns "unknown agent" error
Cause: Agent definition not in expected path
Fix: Verify .claude/agents/backend-architect/agent.md exists. Time: 5 min
Token bloat from too many agents
Symptom: Context window exceeded, slow startup, Claude forgetting earlier conversation
Cause: Loading all 99 agents when only 3-5 needed
Fix: Install only needed agents per project. Rule: 3-5 agents per workflow max. Time: 5 min

ccswarm

Cargo install fails
Symptom: Compilation errors during install
Cause: Missing Rust toolchain or outdated version (requires 1.70+)
Fix: Run rustup update stable and xcode-select --install (macOS). Time: 15 min
Session persistence not working
Symptom: Context lost after restart, 93% token savings not materializing
Cause: Persistence disabled in config or database corrupted
Fix: Verify persist = true in config.toml. If corrupted, delete .ccswarm/sessions/current.db. Time: 2 min

Ralph Marketer

Database empty, AI hallucinates
Symptom: AI invents facts, content not grounded in reality
Cause: Discovery step skipped—database not pre-populated
Fix: Populate trends, research tables BEFORE running synthesis. Time: 30 min

7 What's Hard

Cross-cutting tensions across all practitioners:

Learning vs Doing
The tension: Compound Engineering requires setup time. 99 Agents requires learning the taxonomy. ccswarm requires Rust. Each approach has upfront investment before payoff.
Symptoms: First feature takes 3x longer than "just prompting."
Mitigation: Start simple, expand gradually. Pick ONE approach, master it, then expand. Expect ROI at feature #5+, not feature #1.
Systems vs Scripting
The tension: Rust rigor is hard to learn. Python is fragile at scale. There's no language that's both easy AND reliable.
Symptoms: ccswarm installation friction vs LangChain runtime errors.
Mitigation: Match language choice to reliability requirements. Prototype in Python, productionize in Rust if needed.
Specialized vs General
The tension: Too specialized = overhead dominates. Too general = shallow output. 99 agents for simple task is waste; one agent for complex feature is insufficient.
Symptoms: Spending more time configuring agents than doing work, OR agents producing generic output.
Mitigation: Task-match specialization level. If task description > code change, you've over-decomposed.
Memory vs Tokens
The tension: Persistence costs context (files to read). Forgetting wastes work (repeated mistakes). No free lunch.
Symptoms: progress.txt grows unbounded OR late iterations repeat early mistakes.
Mitigation: External files, not conversation memory. Compact learnings regularly. Move permanent knowledge to AGENTS.md.
Adoption Curve
The tension: Each approach has a learning curve. Combining approaches multiplies the curve. Mastery takes months, not days.
Symptoms: Frustration when approach "doesn't work" after one attempt.
Mitigation: Commit to ONE approach for at least 10 features before judging. Re-read source material after real experience.
