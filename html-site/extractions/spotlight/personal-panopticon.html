Core Insight

Run Claude Code as a swarm of 8 parallel isolated instances - each owning a life domain - to achieve "self-legibility" and reverse the data asymmetry institutions have over you.

The Mental Model

You are surveilled by institutions who can see you but whom you cannot see back. Claude Code lets you build your own "watchtower" - a personal panopticon where YOU are the observer of your own life across all domains.

The Transformation

Before: Blind to your own patterns, cross-domain connections invisible.
After: Observer of all feeds, catching what you've unconsciously dropped, noticing patterns across domains you'd kept stubbornly separate.

The Architecture

GUI fallbackWhen no API exists, inject mouse/keystroke events to control desktop apps

Implementation Guide

Actionability: Deep Dive

This is a full life-operating-system architecture, not a quick tip. Expect significant setup time and iteration.

Getting Started

  1. Choose 3-5 life domains to start - Don't try all 8 at once
  2. Create isolated directories for each - Physical separation matters
  3. Set up basic cron jobs for data pulls - Start with read-only automation
  4. Build the simplest possible "brief" output first - Daily summary before actions
  5. Add GUI automation only when APIs fail - Most things have APIs
  6. Establish checkpointing/notification system - Human-in-the-loop is critical
  7. Log everything for recursive improvement - Your system should learn
