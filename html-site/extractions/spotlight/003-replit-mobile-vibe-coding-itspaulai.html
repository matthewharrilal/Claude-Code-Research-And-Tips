2 The Core Abstraction

"Vibe coding = loose, iterative, voice-driven creation where AI does 90%+ of the heavy lifting."
claude
That's the whole command. Everything else is the Replit environment.

Vibe coding inverts the traditional development model. You don't write code - you describe what you want. You don't debug - you iterate on prompts. The phone becomes a full development workstation because the heavy lifting happens in the cloud.

This works because Replit provides the full Linux environment, Claude Code provides the AI coding, and your phone is just the interface. The magic is in the feedback loop: prompt, watch it build, see live preview, iterate. Development becomes as casual as texting.

3 Design Decisions

WHY REPLIT (NOT CODESPACES)?
Replit's mobile app is purpose-built for phone development. Better touch interface, instant preview in split view, deployment built-in. GitHub Codespaces works but isn't optimized for mobile.
What this means for you
Less configuration, more friction-free flow. The preview updates instantly without manual refresh. You can see your code running while still prompting.
WHY VOICE DICTATION?
Phone keyboards are painful for long prompts. Voice dictation turns your natural speech into text 5-10x faster. Use meta-prompting: speak loosely, then ask Claude to clean it up.
What this means for you
Tap the mic, speak naturally: "Create a booking form with date picker and time slots, clean minimal style with rounded corners." Done in 5 seconds vs 60 seconds typing.
WHY CLAUDE CODE CLI (NOT REPLIT AGENT)?
Replit Agent is great for scaffolding - it creates project structure fast. Claude Code is better for detailed, iterative work. Use both: Agent for structure, Claude for details.
What this means for you
Critical cost difference: Replit Agent uses Replit credits. Claude Code CLI uses your Anthropic API key directly. Claude Code doesn't consume Replit credits.

4 The Stack

Unlike VM-based approaches, everything lives in Replit's cloud. No SSH, no local files, no configuration. The "file structure" is just Replit's standard project layout. 

my-project.repl/
├── .replit                   # Replit configuration
├── replit.nix                # Nix packages (if needed)
├── package.json              # Node dependencies
├── (your framework files)    # Next.js, React, etc.
│
├── [Replit Shell]            # Access via Shell icon
│   └── $ claude              # Run Claude Code here
│
└── [Replit Secrets]          # Environment variables
    └── ANTHROPIC_API_KEY     # Set your API key here

6 Gotchas

Mobile-specific problems you'll hit, with concrete fixes. These are patterns observed across hundreds of vibe coding sessions documented by Paul and the community:

"Extremely addictive" (Paul's words)
Symptom: You're building apps at 2am when you should be sleeping
Cause: The feedback loop is too satisfying - prompt, see result, iterate
Fix: Set time limits. Use Screen Time. Acknowledge this is a real risk.
Mobile typing is painful
Symptom: Prompts take forever to type, full of typos
Cause: Phone keyboards aren't designed for code
Fix: Use voice dictation. Tap mic, speak naturally. 10x faster.
Preview looks good on phone, broken on desktop
Symptom: App looks perfect in Replit preview, breaks on larger screens
Cause: You're only testing mobile viewport
Fix: Open preview URL in desktop browser before shipping. Add "test on desktop" to workflow.
Replit Agent vs Claude Code confusion
Symptom: Unexpected charges on Replit bill
Cause: Used Replit Agent (uses credits) instead of Claude Code CLI (uses Anthropic API)
Fix: Know the difference. Replit Agent = Replit credits. Claude Code = Anthropic API key.
Voice dictation misheard technical terms
Symptom: "React" becomes "react", "TypeScript" becomes "type script"
Cause: iOS/Android dictation isn't trained on programming vocabulary
Fix: Review prompt before sending. Spell out ambiguous terms. Use meta-prompting to clean up.
Split view too cramped
Symptom: Can't see enough of terminal OR preview
Cause: Phone screens are small
Fix: Rotate to landscape. Use tablet if available. Accept the limitation.
No offline mode
Symptom: Can't code on airplane without WiFi
Cause: Replit is cloud-only, Claude API requires internet
Fix: Accept the limitation. This is a cloud-first approach by design.
Replit container sleep
Symptom: Project takes 10-30 seconds to load after being idle
Cause: Free Replit containers sleep after inactivity to save resources
Fix: Wait for container to wake. Paid plans have "Always On" option for faster startups.
Large files crash preview
Symptom: Preview stops updating, app becomes unresponsive
Cause: Loading large assets (images, fonts, datasets) in mobile preview
Fix: Optimize assets. Use lazy loading. Test large files on desktop first.

7 What's Hard

Fundamental tensions in vibe coding, not bugs to fix:

Vibe Coding != Production Quality
The tension: Fast iteration optimizes for speed, not rigor. "Vibe" and "production-ready" are different goals.
Symptoms: App works but has no tests, no error handling, no accessibility, no security review.
Coping strategy: Use vibe coding for prototypes. Add rigor later. Don't ship vibed code to production without review.
Phone Screen Size
The tension: You can't see full context on a 6-inch screen. Complex debugging is painful.
Symptoms: Lose track of file structure, miss errors scrolled off-screen, can't compare two files.
Coping strategy: Accept the limitation. Use phone for simple tasks. Graduate to laptop for complex debugging.
Voice Dictation Accuracy for Technical Content
The tension: Natural speech doesn't map cleanly to code. "useState" might become "use state" or "you state".
Symptoms: Claude misinterprets prompts, generates wrong code, you spend time fixing dictation errors.
Coping strategy: Learn to spell out ambiguous terms. "useState - u-s-e-S-t-a-t-e". Review before sending.
Distraction Risk
The tension: Your phone is also your distraction device. Notifications, social media, messages.
Symptoms: Start coding, get notification, 30 minutes later you're on Twitter.
Coping strategy: Enable Do Not Disturb. Use Focus modes. Close other apps before starting.
Context Loss on Complex Projects
The tension: Claude Code has context limits. On phone, you can't easily reference multiple files.
Symptoms: Claude forgets earlier decisions. Inconsistent code patterns emerge.
Coping strategy: Use CLAUDE.md file to persist project context. Keep sessions short and focused.
Battery Drain
The tension: Constant screen, network activity, and processing drains battery fast.
Symptoms: Phone dies mid-session. Thermal throttling slows everything.
Coping strategy: Keep charger nearby. Take breaks. Use iPad if available (bigger battery).

Mobile Workflow Optimization

To mitigate these challenges, structure your mobile sessions intentionally:

1
Pre-session: Clear notifications, enable Do Not Disturb, close other apps
2
Scope: Define ONE clear goal before opening Replit ("add booking form" not "work on app")
3
Time-box: Set 25-minute timer (Pomodoro). Stop when it rings.
4
Capture: Note any complex issues for laptop follow-up
5
Commit: Always commit working code before ending session
