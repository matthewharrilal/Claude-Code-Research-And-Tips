3 Design Decisions

WHY REASONINGBANK?
"Context lost between sessions - each new session starts from zero, repeating expensive learning." ReasoningBank solves the persistent memory problem.
What this means for you
Patterns persist across sessions. Learning transfers. Your agents genuinely improve over time instead of starting from scratch each run.
WHY QUIC TRANSPORT?
"TCP latency kills multi-agent coordination." At scale, traditional HTTP/TCP adds 1-3 RTT overhead for each connection. QUIC provides 0-RTT reconnection.
What this means for you
Sub-millisecond RTT for agent coordination. Stream multiplexing eliminates head-of-line blocking. Connection migration survives WiFi-to-cellular transitions.
WHY SWARM TOPOLOGIES?
"No single topology fits all use cases." Hierarchical scales to ~100 agents with clear chain of command. Mesh provides full redundancy. Ring enables ordered pipelines.
What this means for you
Choose topology based on workload: Hierarchical for codebase analysis, Mesh for consensus-heavy tasks, Ring for data processing pipelines.
WHY MULTI-MODEL ROUTING?
"No single model is best for everything." Different tasks have different priorities: quality, cost, speed, or privacy.
What this means for you
Route by priority: quality uses Claude 3.5 Sonnet/GPT-4. cost uses OpenRouter/Gemini Flash. privacy uses local ONNX models.
WHY FLASH ATTENTION?
"Memory efficiency at scale." Standard attention is O(n^2) in memory. Flash Attention provides 2.49x-7.47x speedup with 50-75% memory reduction.
What this means for you
Run larger models on smaller hardware. Process longer sequences without OOM. Essential for production workloads.
WHY SONA?
"Self-learning infrastructure - agents genuinely improve over time." Traditional agents reset every session. SONA enables continual learning.
What this means for you
<1ms overhead for adaptation. 99% parameter reduction with LoRA. EWC++ prevents catastrophic forgetting of previous knowledge.

Deployment Patterns

Pattern 1: Single Container (Development)

# docker-compose.yml
version: '3.8'

services:
  agentic-flow:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - agentdb-data:/data/agentdb
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - SONA_PROFILE=balanced
      - ATTENTION_TYPE=flash
    restart: unless-stopped

volumes:
  agentdb-data:

Pattern 2: Multi-Container Swarm (Production)

# docker-compose.swarm.yml
services:
  coordinator:
    image: agentic-flow:latest
    command: ["npx", "agentic-flow", "swarm", "coordinator"]
    ports:
      - "3000:3000"
      - "4433:4433"  # QUIC
    environment:
      - ROLE=coordinator
      - SWARM_TOPOLOGY=hierarchical
      - QUIC_ENABLED=true
    deploy:
      replicas: 1
      resources:
        limits:
          cpus: '2'
          memory: 4G

  worker:
    image: agentic-flow:latest
    command: ["npx", "agentic-flow", "swarm", "worker"]
    environment:
      - ROLE=worker
      - COORDINATOR_URL=http://coordinator:3000
    deploy:
      replicas: 4
      resources:
        limits:
          cpus: '1'
          memory: 2G
    depends_on:
      - coordinator

6 Gotchas

Real problems in v2.0.0-alpha with concrete numbers and fixes:

HNSW index catastrophically slow on import
Symptom: 183-516ms per vector insertion
Time to fix: 30 min
Fix: Use batch insertion with db.batchStore(vectors, { batchSize: 100 }). Enable WASM acceleration. Pre-build index offline.
"__dirname is not defined" error
Symptom: ESM/CommonJS context mismatch in agentdb-cli.js
Time to fix: 5 min
Fix: Use npx agentic-flow-esm agentdb or add "type": "module" to package.json
Port 3000 conflicts with Docker
Symptom: Source and container using same port
Time to fix: 2 min
Fix: Map to different host port: ports: - "3001:3000"
Missing dist files causing package load failures
Symptom: Import errors, module not found
Time to fix: 15 min
Fix: Clone and build from source: git clone && npm install && npm run build
Workers not connecting to coordinator
Symptom: Workers stuck in "connecting" state
Time to fix: 10 min
Fix: Verify COORDINATOR_URL environment variable. Check network connectivity. Ensure QUIC port 4433 open if using QUIC.
ReasoningBank empty results
Symptom: No patterns returned from queries
Time to fix: 5 min
Fix: Lower minConfidence from 0.8 to 0.5. Check namespace spelling (case-sensitive). Verify patterns exist with npx agentic-flow reasoningbank list
Out of memory errors
Symptom: OOM crashes during large workloads
Time to fix: 10 min
Fix: Enable Flash Attention (type: 'flash'). Add gradient_checkpointing: true. Use mixed_precision: 'fp16'.
Latency too high for coordination
Symptom: Multi-agent operations slow
Time to fix: 15 min
Fix: Switch to QUIC transport: transport: 'quic', quicConfig: { zeroRtt: true }
Low throughput
Symptom: Low ops/sec despite good hardware
Time to fix: 10 min
Fix: Use batch SONA profile: sonaProfile: 'batch' for high throughput workloads
Alpha stability issues
Symptom: Various unexpected behaviors
Time to fix: Varies
Fix: Pin to specific version. Have fallback plan. Test thoroughly before production.

7 What's Hard

These are fundamental tensions, not bugs to fix:

Context Loss Between Sessions
The tension: "Each new session starts from zero, repeating expensive learning."
Why it's hard: LLMs have no built-in persistence. Every restart means full re-learning.
Coping strategy: ReasoningBank provides persistent pattern memory. But you still pay the cost of loading patterns each session.
Scaling Ceiling
The tension: "Can't move beyond single-agent workflows" without significant infrastructure.
Why it's hard: Multi-agent coordination requires consensus, fault tolerance, and monitoring. Each adds complexity.
Coping strategy: Swarm topologies abstract coordination. But choosing the right topology requires understanding your workload.
Reliability Gaps
The tension: "No fault tolerance, recovery, or monitoring" in basic setups.
Why it's hard: Production reliability requires Byzantine/Raft consensus, health checks, and observability.
Coping strategy: Use the full Gas Town deployment pattern. Accept that reliability = complexity.
Cost at Scale
The tension: "$50-200+/day at scale" for production workloads.
Why it's hard: Every token costs money. Multi-agent = multiply tokens. Learning = more tokens.
Coping strategy: Multi-model routing with cost priority. Use Haiku/GPT-3.5 for low-stakes tasks. Monitor and optimize.
Alpha Status
The tension: "v2.0.0-alpha has documented issues" with ESM, dist files, HNSW performance.
Why it's hard: Active development means breaking changes. Documentation lags implementation.
Coping strategy: Pin versions. Read GitHub issues. Have fallback plan. Test thoroughly.
Documentation Complexity
The tension: "241+ documentation files can be overwhelming."
Why it's hard: Comprehensive documentation = more to read. Feature-rich = more to learn.
Coping strategy: Start with minimal patterns. Add features as needed. Follow this guide's decision criteria.
