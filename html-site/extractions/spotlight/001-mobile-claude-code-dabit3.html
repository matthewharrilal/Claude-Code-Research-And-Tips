2 The Core Abstraction

"Instead of long periods of intense focus, software development can now just fit into the gaps of your day."
Development becomes async. Start agents, live life, phone buzzes, respond, repeat.

The paradigm shift: software development as an asynchronous, agent-orchestrated process that interleaves with life, rather than a synchronous, focused activity requiring dedicated hardware and time.

The Stack Explained

Persistence enables handoffs: tmux keeps sessions alive. mosh survives network transitions. You can disconnect, put phone in pocket, reconnect hours later. The agent state persists.

Claude comes to you: PreToolUse hook triggers on AskUserQuestion. Script extracts the question, sends to Poke API via curl. Phone buzzes. You respond. Agent continues.

3 Design Decisions

WHY CLOUD VM (NOT HOME MAC)?
Cloud VM is always available, isolated from personal data, and scalable to parallel agents. Home Mac requires always-on hardware, exposes personal files to Claude, and can't scale beyond one machine.
What this means for you
Trade ~$7/day for independence from hardware. Stop VM when not using to avoid $210/month if always-on. Use hourly billing to pay only for active development time.
WHY TAILSCALE (NOT RAW SSH)?
Tailscale provides zero-config VPN with no public SSH exposure. Your VM has no public ports. All traffic encrypted through Tailscale's mesh network. Setup takes 5 minutes.
What this means for you
No SSH key management headaches. No exposed ports for attackers. Install Tailscale on phone + VM, sign in, they find each other automatically.
WHY MOSH (NOT SSH)?
mosh uses UDP, not TCP. It survives WiFi to cellular transitions, device sleep, walking into elevator, changing networks. SSH would disconnect and lose your session.
What this means for you
Start a session at home on WiFi. Walk to car. Drive to work on cellular. Connection persists. No "Connection reset by peer" frustration.
WHY POKE (NOT SMS)?
Poke provides instant push notifications with webhook integration. SMS requires carrier setup, has delivery delays, and costs money per message. Poke is free and instant.
What this means for you
Hook triggers, Poke buzzes in 1-3 seconds. You see Claude's question immediately. No Twilio setup, no per-message costs.
WHY GIT WORKTREES (NOT BRANCHES)?
Each worktree is a separate directory with full isolation. Multiple agents can work on different features simultaneously without git switching conflicts.
What this means for you
Run 6 agents on 6 features. Each in its own tmux window, its own directory, its own port (hash-based allocation). True parallelism.

Notification Hook

# ~/.claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "AskUserQuestion",
        "command": "~/.claude/hooks/poke-notify.sh"
      }
    ]
  }
}
#!/bin/bash
# ~/.claude/hooks/poke-notify.sh

POKE_WEBHOOK="https://poke.dev/api/webhook/YOUR_WEBHOOK_ID"
MESSAGE="Claude needs input: $(echo $HOOK_INPUT | jq -r '.question')"

curl -X POST $POKE_WEBHOOK \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"$MESSAGE\"}"

6 Gotchas

10 real problems with numbers and fixes:

#1: VM costs if not stopped
Symptom: $210/month bill when VM runs 24/7
Fix: Use vm-stop script religiously. Set iOS reminder. Use hourly billing.
Time to prevent: 5 seconds (tap Shortcut)
#2: mosh doesn't forward SSH agent
Symptom: git push fails with auth error
Cause: mosh can't forward SSH agent like SSH can
Fix: ssh-add ~/.ssh/id_ed25519 inside tmux after connecting
Time to fix: 30 seconds
#3: Notifications disrupt off-time
Symptom: Phone buzzing at dinner, weekends, 2AM
Cause: Agents don't know your schedule
Fix: Configure Poke quiet hours. Stop VM when truly offline.
Time to fix: 2 minutes
#4: Spotty cellular connection
Symptom: Commands lag or timeout on SSH
Cause: SSH uses TCP, drops on network changes
Fix: Use mosh instead of SSH. It handles bad connections gracefully.
Time to fix: 5 min setup
#5: Phone keyboard pain
Symptom: Typos, slow input, frustration
Fix: Use voice dictation for prompts. Keep responses short. Use external keyboard for longer sessions.
Time to fix: Learning curve
#6: Hash collision on ports
Symptom: Two branches get same port, dev server conflict
Cause: Hash-based port allocation isn't collision-free
Fix: Rename branch or manually specify port. Rare issue.
Time to fix: 1 minute
#7: Permissive mode security
Symptom: Claude has full system access
Fix: ONLY use permissive mode on isolated VMs. Never on personal machines.
Time to fix: N/A - prevention only
#8: tmux session lost on reboot
Symptom: All agents gone after VM restart
Cause: tmux is memory-resident, not persistent
Fix: Re-run startup script. Keep work in git, not in session.
Time to fix: 5 minutes
#9: Tailscale disconnects
Symptom: Can't reach VM, connection hangs
Fix: Re-authenticate in Tailscale app. Check VPN status.
Time to fix: 30 seconds
#10: Poke webhook misconfigured
Symptom: No notifications, agents stuck waiting
Fix: Test with curl first. Check webhook URL in settings.json.
Time to fix: 5 minutes

7 What's Hard

Fundamental tensions, not bugs to fix:

Small Screen vs. Code Density
The tension: Phone screens can't show full context. You see 20 lines at a time. Scrolling is constant.
Why it's fundamental: Physics. Screen size won't magically increase.
Coping strategy: Use tablet for review sessions. Accept limitation. Use for oversight/direction, not deep debugging.
Async Notification Fatigue
The tension: Too many notifications = burnout. Too few = stuck agents.
Why it's fundamental: You can't be responsive AND disconnected.
Coping strategy: Smart filtering. Quiet hours. Batch notifications. Accept that async has limits.
Cost vs. Availability
The tension: VM always on = always ready but expensive. VM off = cheap but boot delay.
Why it's fundamental: Cloud resources cost money. Uptime costs money.
Coping strategy: Hourly billing. Stop when done. Use home Mac if zero-cost is priority.
Mental Context Switching
The tension: Gap development means constant context switches. Deep focus is impossible.
Why it's fundamental: Can't be deeply focused AND interleaved with life.
Coping strategy: Use mobile for oversight/direction, laptop for complex debugging. Accept the paradigm difference.
Network Dependency
The tension: No connection = no development. Dead zones exist.
Why it's fundamental: Cloud requires network. There's no offline mode.
Coping strategy: Accept limitation. Use airplane time for other things. Ollama if truly offline-first needed.
Typing on Phone
The tension: Slow, error-prone, frustrating for long inputs.
Why it's fundamental: Touch keyboards have physical limits.
Coping strategy: Voice dictation. Short responses. External keyboard. Pre-written prompts.
