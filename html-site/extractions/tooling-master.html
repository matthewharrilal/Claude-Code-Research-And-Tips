rpai - Multi-Agent Monitor

TUI for Agent Orchestration
Author: @radoslav11 (Rado)

TUI tool for tmux that makes monitoring and switching between multiple parallel AI agents actually usable.

The Problem

"UX around parallel agents sucks right now."

When running 3+ agents in tmux panes, you're constantly tabbing between panes, losing track of which agent is where, missing outputs from background agents.

Installation

cargo install rpai

# Run in any tmux session
rpai

What It Does

┌─────────────────────────────────────────────────┐ │ RPAI - Agent Monitor │ ├─────────────────────────────────────────────────┤ │ [1] CLAUDE AGENT ████████ running │ │ [2] CURSOR AGENT ████░░░░ running │ │ [3] CODEX AGENT ░░░░░░░░ idle │ ├─────────────────────────────────────────────────┤ │ Preview: Claude is executing Python... │ │ > import pandas as pd │ │ > df = pd.read_csv('data.csv') │ └─────────────────────────────────────────────────┘

Features

  • Scans tmux panes for running AI agents
  • Shows status (running/idle/waiting)
  • Output preview for each agent
  • One-key jump to any agent pane
  • Mouse support

Detected Agents

Claude, Cursor, Codex, Opencode (pattern-matched by process name)

Quick Win

Install and running in under 5 minutes. Immediate improvement to multi-agent workflows.

claude-mem - Persistent Memory

Infinite Memory Plugin
Author: @thedotmack

Open-source plugin that gives Claude Code "infinite memory" by automatically capturing sessions, compressing to semantic summaries, and injecting relevant context into future sessions.

The Problem Solved

  • Context resets on restart
  • Must re-explain project history every session
  • Token waste from repetition
  • Can't maintain long-term project continuity

Key Stats

Metric Improvement
Token usageUp to 95% fewer per session
Tool calls before limit~20x more
StorageLocal SQLite only

How It Works

┌─────────────────────────────────────────────────┐ │ Claude Code Session │ │ └─ Lifecycle Hooks │ │ └─ Capture events │ └─────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────┐ │ Worker Service (port 37777) │ │ ├─ SQLite DB (raw data) │ │ ├─ Chroma Vector DB (embeddings) │ │ └─ Claude Agent SDK (compression) │ └─────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────┐ │ Future Session │ │ └─ Auto-inject relevant context │ └─────────────────────────────────────────────────┘

Installation

/plugin marketplace add thedotmack/claude-mem
/plugin install claude-mem

Claude HUD - Session Dashboard

Real-time Session Monitoring
Author: @jarrodwatts

A heads-up display for Claude Code sessions showing token usage, cost, and session metrics.

What It Shows

  • Token usage (input/output)
  • Cost tracking ($)
  • Session duration
  • Tool call count
  • Context window usage %

When to Use

Essential for cost-conscious usage, especially with Ralph loops or long sessions. Know when you're approaching limits before you hit them.

Key Insight

Users should avoid using the final 20% of their context window for complex tasks - performance degrades significantly when approaching limits.

Claude Agent SDK

Build Your Own Agents
Tutorial: @dabit3 (Nader Dabit)

Official SDK for building custom Claude-powered agents with tool use, memory, and orchestration capabilities.

Key Components

Agent
Core agent class with conversation management and tool execution.
Tools
Define custom tools the agent can use - functions with typed parameters.
Memory
Conversation history and state management across turns.
Streaming
Real-time response streaming for interactive applications.

Basic Example

import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

const tools = [{
  name: "get_weather",
  description: "Get current weather for a location",
  input_schema: {
    type: "object",
    properties: {
      location: { type: "string" }
    },
    required: ["location"]
  }
}];

async function runAgent(prompt) {
  const response = await client.messages.create({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1024,
    tools: tools,
    messages: [{ role: "user", content: prompt }]
  });

  // Handle tool use...
}

Use Cases

  • Custom orchestration systems (Gas Town-style)
  • Specialized domain agents
  • Integration with existing systems
  • Production agent deployments

tmux Patterns

Session Persistence for Agents

tmux is essential infrastructure for mobile workflows and long-running agents. Sessions persist across disconnects.

Auto-Attach Configuration

# ~/.zshrc or ~/.bashrc
if [[ -z "$TMUX" ]] && [[ -n "$SSH_CONNECTION" ]]; then
  tmux attach -t main 2>/dev/null || tmux new -s main
fi

Mobile-Optimized Config

# ~/.tmux.conf
set -g default-terminal "screen-256color"
set -g history-limit 100000
set -g mouse on
set -s escape-time 0

# Start at 1 (easier on phone)
set -g base-index 1
setw -g pane-base-index 1

# Status bar with git branch
set -g status-right '#[fg=yellow]#(git branch --show-current) #[fg=cyan]%H:%M'

# Quick window switching
bind -n M-1 select-window -t 1
bind -n M-2 select-window -t 2
bind -n M-3 select-window -t 3

# Session persistence
set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'
set -g @continuum-restore 'on'

Multi-Agent Layout

#!/bin/bash
# create-agent-layout.sh
SESSION="agents"

tmux new-session -d -s $SESSION -n "agent-1"
for i in {2..6}; do
  tmux new-window -t $SESSION -n "agent-$i"
done

tmux attach -t $SESSION
