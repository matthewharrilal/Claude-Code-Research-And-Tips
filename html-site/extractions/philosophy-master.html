Boris Cherny's Workflow

The Creator's Philosophy
Boris Cherny - Claude Code creator, former Facebook engineer
"What we're building is a new programming paradigm - you give Claude context, it reasons about your code, and you iterate together."

Core Principles

Verification Loops
The single biggest quality multiplier. Give Claude ways to test its own work - tests, type checks, linting.
Vanilla Claude
Start with built-in capabilities before adding extensions. Most productivity comes from mastering fundamentals.
Context as Constraint
CLAUDE.md files provide structure, not just information. They constrain Claude's solution space productively.
Compounding Knowledge
Update CLAUDE.md after learning sessions. Over time, the AI gets better at YOUR projects specifically.

Workflow Pattern

# Boris's typical flow
1. Write CLAUDE.md with project context
2. Start with clear, specific task
3. Let Claude propose approach
4. Review, provide feedback
5. Iterate on implementation
6. Verify with tests/typecheck
7. Update CLAUDE.md with learnings

Key Quotes

"Claude's tendency to mark features complete without proper testing... did much better at verifying end-to-end once explicitly prompted to use browser automation tools."
"This creates evolving 'memory' that compounds over time."
The Verification Insight

Claude will enthusiastically mark features "complete" without proper testing unless explicitly prompted to verify. Build verification into the workflow, not as an afterthought.

Vibe Coding Philosophy

Give In to the Vibes
Origin: Andrej Karpathy
"Vibe coding" is a new paradigm where you fully embrace AI assistance and "give in to the vibes." Instead of carefully controlling every line, you describe what you want and let the AI figure out implementation details.

The Core Shift

You don't need to understand every line of code anymore. You focus on WHAT should happen, the AI handles HOW. This is terrifying for traditional developers but liberating once embraced.

When to Vibe Code

Good For Not Good For
PrototypesCritical production systems
Internal toolsSecurity-sensitive code
Exploratory codePerformance-critical paths
Personal projectsHighly regulated domains
MVPsLong-term maintainability

The Unlock

  • 10-100x velocity on appropriate tasks
  • Freedom from framework memorization - Claude knows it
  • Ship ideas that would otherwise die in your notebook
  • Mental shift from "implementer" to "director"
The key question: What level of quality is actually needed here? For many use cases, "good enough" is the goal. Perfectionism is the enemy of shipping.

Steve Yegge's Vision

The Idea Compiler
Steve Yegge - Gas Town creator, ex-Google/Amazon
"Work becomes fluid, an uncountable that you sling around freely, like slopping shiny fish into wooden barrels at the docks."

Key Concepts

Human as PM, AI as Coder

Your role fundamentally shifts from "person who writes code" to "product manager who directs agents." You think about features, requirements, acceptance criteria, and user value. The AI thinks about implementation, syntax, and technical details.

Throughput Over Perfection

Optimize for THROUGHPUT, not perfection. Some bugs get fixed 2-3 times. Some fixes get lost. That's fine. The goal is shipping features, not writing pristine code.

Boring Technology Advantage

For AI-generated code, "boring" languages outperform exciting ones:

Language Issue
TypeScript1/3 to 1/2 of diffs are type wrestling
Python"Fine" but feels like scripts
Go"Boring" = evolutionary advantage. Diffs are understandable.

Colonies Over Individuals

"Claude Code is the world's biggest fuckin' ant. Everyone is focused on making their ant run longer... when nature prefers colonies."

Desire Paths

"Desire paths" in UX: watch where people walk across grass, then pave those paths. Applied to AI: watch what agents try to do, then make those things real. Don't design AI systems top-down. Observe emergent agent behavior and formalize what works.

Markdown as Universal Interface

The .md Superpower
Source: Greg Isenberg viral thread
"You're a few .md files from being able to outperform 99% of people on this planet."

The Insight

Markdown files are the "API" between you and AI systems. A few well-structured .md files containing your context, preferences, and structured knowledge can outperform 99% of unstructured interactions.

Structured Context

"Structured context is the highest leverage asset you can own right now." - @advait_jayant

The gap between those who curate personal data and those who don't is widening exponentially.

What to Include

brand-voice.md
Your tone, style, values. How you communicate.
project-context.md
Architecture, conventions, common patterns.
CLAUDE.md
Project rules, constraints, preferences for Claude.
skill libraries
Reusable instruction sets for common tasks.

The Benefits

  • 80% better first-try outputs
  • No re-explaining context each session
  • Composable automation (pipe outputs)
  • "Second brain" that AI can read and augment
Quick Win

Create one .md file today with your project context. Upload to Claude. Notice the difference immediately.

Honest Assessment

Most developers are at Stage 2-3. Most overestimate their stage. Focus on mastering current stage before advancing.

Stage Transitions

  • Stage 4-5 requires trust-building: Many get stuck here. Need to see agents succeed repeatedly.
  • Stage 6+ requires infrastructure: Git worktrees, multiple API keys, tmux mastery
  • Stage 8 is advanced: "If you have any doubt whatsoever about running Gas Town, then you can't use it."

Key Mental Models

Fresh Context Per Iteration

Long AI sessions degrade. Context fills up, summaries lose detail, quality drops. The solution: spawn fresh agent instances per task, using external state (git, files) for continuity.

Each iteration starts with zero context pollution. The agent reads current state from files and works with a clean slate.

External Memory Architecture

AI assistants forget between sessions. The solution isn't persistent AI memory—it's external memory systems that the AI reads on startup.

  • progress.txt - Append-only learning log
  • prd.json - Task state with passes: true/false
  • CLAUDE.md - Project context and rules
  • Git history - What actually changed
The Contractor Mental Model

Think of CLAUDE.md as onboarding documentation for an extremely competent but context-free contractor. This contractor:

  • Is brilliant at execution
  • Has no institutional memory
  • Needs explicit instruction on conventions
  • Won't "know what you mean" without documentation
The Amplification Thesis

AI tools don't democratize coding ability—they amplify existing differentials. Good developers become great. Mediocre developers produce mediocre code faster. The gap widens, not narrows.

This explains why some find AI "useless" while others are 10x.

Breadth-Depth Tradeoff

AI is better at breadth, humans are better at depth:

  • AI: Explore 10 approaches, cover all edge cases, generate options
  • Human: Choose the right approach, deep analysis, judgment calls
The Activation Energy Bottleneck
"The bottleneck is no longer ability. The bottleneck is activation energy: who has the nerve to try, and the stubbornness to finish." - Molly Cantillon

AI has made capability abundant. What differentiates outcomes is the willingness to try ambitious things and the persistence to iterate through failures.

Anti-Patterns to Avoid

The Perfectionist Trap

Treating AI output like production code that needs extensive review. For many use cases, "good enough" is the goal.

The Micro-Manager

Specifying HOW instead of WHAT. Let the AI figure out implementation.

Context Maximalism

"More context is always better." Actually, context rot means lean context often produces better results.

The Trust Binary

Either trusting AI completely or not at all. Build graduated trust through verification.

Tool Obsession

Chasing the newest orchestration tool when fundamentals aren't mastered. Stage 3 developers trying Stage 8 tools.

Prompt Magic Thinking

Believing the "right prompt" will fix everything. Structured context beats clever prompts.

Human Loop Addiction

Requiring human approval for every action. Build verification systems that let agents run autonomously.
