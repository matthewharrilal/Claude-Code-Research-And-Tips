Mobile Workflows

Development From Anywhere

Nader Dabit pioneered mobile-first Claude Code workflows, enabling development from phone or tablet.

The Setup

  1. Cloud VM running Claude Code (always-on)
  2. SSH client on mobile device
  3. tmux for session persistence
  4. Push notifications on AskUserQuestion

Key Insight

"Development fits into the gaps of your day. Waiting for coffee? Review what the agent did. Commuting? Approve the next task."

Replit Integration

Browser-Based Development

Replit provides a full development environment accessible from any device with a browser.

Advantages

  • No local setup required
  • Works on any device
  • Built-in collaboration
  • Instant deployment

Limitations

  • Resource constraints on free tier
  • Network dependency
  • Limited Claude Code integration (improving)

Session Teleportation

Move Sessions Between Devices

Session teleportation lets you start work on one device and continue seamlessly on another.

How It Works

  1. All state in external files (progress.txt, prd.json)
  2. Git sync between devices
  3. Fresh Claude Code session reads state
  4. Continue exactly where you left off

Required Files

project/
├── CLAUDE.md      # Project context (synced)
├── prd.json       # Current task (synced)
├── progress.txt   # Current state (synced)
└── .git/          # Git for sync
