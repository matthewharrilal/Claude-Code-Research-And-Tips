The Paradigm Shift

Traditional development requires dedicated laptop time - opening your machine, focusing for hours, then closing it. Life interrupts this constantly: meetings, commutes, kids, errands.
Traditional Development: [Open laptop] --> [Focus 2-4 hours] --> [Close laptop] Mobile-First Development: [Start agents] --> [Go about day] --> [Phone buzzes] --> [Quick response] --> [Agents continue] --> [Review later]

The Key Insight

Async Agent Orchestration

Software development becomes an asynchronous, agent-orchestrated process that interleaves with life, rather than a synchronous activity requiring dedicated hardware and time blocks.

Three Mobile Architectures

Architecture A: Cloud VM
Phone -> Tailscale -> Cloud VM -> tmux -> Claude Code
  • Cost: $5-20/day depending on specs
  • Best for: Heavy parallel workloads, team setups
  • Providers: Hetzner, Vultr, AWS, DigitalOcean
Architecture B: Home Mac
Phone -> Tailscale -> Home Mac -> tmux -> Claude Code
  • Cost: $0 (electricity only)
  • Best for: Personal use, always-on home setup
  • Caveat: Must keep machine running
Architecture C: Hybrid
Phone -> Tailscale -> [Home Mac | Cloud VM]
  • Cost: $0-7/day depending on usage
  • Best for: Cost-conscious power users
  • Strategy: Home for light, cloud for heavy

tmux Configuration

Auto-Attach on SSH Login

Add to ~/.zshrc or ~/.bashrc:

# Auto-attach to tmux on SSH login
if [[ -z "$TMUX" ]] && [[ -n "$SSH_CONNECTION" ]]; then
  tmux attach -t main 2>/dev/null || tmux new -s main
fi

Mobile-Optimized tmux.conf

# ~/.tmux.conf - Mobile-optimized configuration

# Basics
set -g default-terminal "screen-256color"
set -g mouse on
set -s escape-time 0

# Start at 1 (easier on phone keyboards)
set -g base-index 1
setw -g pane-base-index 1

# Large scrollback for catching up on agent work
set -g history-limit 100000

# Status bar showing git branch and time
set -g status-right-length 60
set -g status-right '#[fg=yellow]#(cd #{pane_current_path} && git branch --show-current) #[fg=white]| #[fg=cyan]%H:%M'

# Easy window switching (works on mobile keyboards)
bind -n M-1 select-window -t 1
bind -n M-2 select-window -t 2
bind -n M-3 select-window -t 3

# Session persistence (survives reboot)
set -g @plugin 'tmux-plugins/tmux-resurrect'
set -g @plugin 'tmux-plugins/tmux-continuum'
set -g @continuum-restore 'on'

Multi-Agent Layout Script

#!/bin/bash
# create-agent-layout.sh - Set up 4-agent layout

SESSION="agents"

# Create session with first window
tmux new-session -d -s $SESSION -n "agent-1"

# Create additional windows for each agent
for i in {2..4}; do
  tmux new-window -t $SESSION -n "agent-$i"
done

# Create overview window with all agents visible
tmux new-window -t $SESSION -n "overview"
tmux split-window -h -t $SESSION:overview
tmux split-window -v -t $SESSION:overview.1
tmux split-window -v -t $SESSION:overview.2

# Attach
tmux attach -t $SESSION

Claude Code Hook for Notifications

Configure in ~/.claude/settings.json:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "AskUserQuestion",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/notify-mobile.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Task",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/task-complete.sh"
          }
        ]
      }
    ]
  }
}

Production Workflow Patterns

Pattern 1: Morning Kickoff

Daily Workflow Start Day with Agents
  1. Wake up, coffee
  2. SSH into VM from phone
  3. Review overnight agent progress in tmux
  4. Kick off new tasks for the day
  5. Go to work/life

Pattern 2: Gap Development

Micro Workflow 10-Minute Breaks
  1. 10-minute break between meetings
  2. Pull out phone, check tmux
  3. Agent asking question -> respond
  4. Agent needs review -> quick approval
  5. Back to meeting

Pattern 3: Evening Review

End of Day Wrap Up and Queue Overnight
  1. SSH in after dinner
  2. Review all agent outputs
  3. Merge successful branches
  4. Queue up overnight tasks
  5. Sleep while agents work

Teleportation Patterns

Teleportation = seamlessly moving sessions/context between devices without losing state.

Native Claude Code Teleportation

# In terminal Claude session
&
# or
--teleport

# Outputs URL like: https://claude.ai/code/session/abc123
# Open URL on any device to continue

tmux Session Teleportation

# On laptop
tmux attach -t work
# Work for a while
# Close laptop lid (connection drops)

# On phone (via Termius)
ssh devuser@vm
# Auto-attaches to existing tmux session
# Continue exactly where you left off
tmux is the Key

tmux sessions persist on the server. You can detach anywhere, attach from anywhere. Your Claude session, full scrollback history, all running processes - everything survives the device switch.

Troubleshooting

Connection drops frequently

Solution: Use mosh instead of SSH. Mosh handles network transitions gracefully.

apt install mosh
mosh devuser@100.x.x.x
tmux session not auto-attaching

Verify your shell config:

cat ~/.zshrc | grep tmux
# Should see the auto-attach snippet

Make sure it's at the end of the file and not overridden.

Notifications not arriving
  1. Test the notification service directly: curl -d "test" ntfy.sh/your-topic
  2. Check hook is executable: chmod +x ~/.claude/hooks/notify-mobile.sh
  3. Verify environment variables are set in your shell profile
  4. Check phone notification settings for the app
Tailscale not connecting
  1. Check Tailscale status: tailscale status
  2. Verify same account on all devices
  3. Restart Tailscale: sudo tailscale down && sudo tailscale up
  4. Check admin console for device authorization
Battery draining fast on phone

Long SSH sessions drain battery. Strategies:

  • Reduce screen brightness
  • Enable dark mode (OLED screens)
  • Use WiFi over cellular
  • Enable Low Power Mode when polling
  • Consider MagSafe battery pack
