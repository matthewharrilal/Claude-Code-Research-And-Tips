Mental Models Journey

You Are Here

What this journey covers: How expert practitioners think about Claude Code. These are the cognitive frameworks that separate power users from beginners - the mental models that make architecture decisions obvious and anti-patterns recognizable.

Who this is for: Anyone using Claude Code who wants to understand the "why" behind the patterns. Essential before diving into architecture or implementation - these models inform every decision you'll make.

What you'll be able to do:

  • Recognize when context is becoming a constraint (and what to do about it)
  • Design systems that use external state instead of relying on memory
  • Know when to start fresh vs. extend a session
  • Apply heuristics that prevent common failure modes

The 8 Core Principles (Preview)

1. CONTEXT AS CONSTRAINT
   Context windows are finite. Design around this limit, not against it.

2. EXTERNAL STATE > INTERNAL MEMORY
   Files, git, databases outlive sessions. Memory doesn't.

3. FRESH CONTEXT > EXTENDED SESSIONS
   New sessions with clean state beat long sessions with accumulated cruft.

4. VERIFICATION LOOPS
   Trust but verify. Every change should prove it works.

5. SPECIFICATION OVER CONVERSATION
   PRDs beat chat history. Write it down.

6. COMPOSITION OVER COMPLEXITY
   Simple patterns combined beat complex patterns alone.

7. APPROPRIATE SCALE
   Match pattern complexity to problem complexity. No more, no less.

8. HUMAN IN THE RIGHT LOOP
   Some decisions need humans. Know which ones.
