Quick Selection Guide

QUICK FRAMEWORK SELECTION "I need self-learning agents that improve over time" -> Claude-Flow (SONA architecture, EWC++ retention) "I need 100+ agents with multiple LLM providers" -> Swarms (14+ topologies, OpenAI/Anthropic/Groq/Ollama) "I need structured SDLC with role-specific agents" -> BMAD (9 personas across 4 phases, workflow-driven) "I need Claude Code native integration" -> BMAD or Claude-Flow (both MCP-enabled) "I need Python-based orchestration" -> Swarms (pure Python, pip install) "I'm building a product with AI-driven development" -> BMAD (Analysis -> Planning -> Solutioning -> Implementation)

Swarms: Scale-Dependent Optimization

Swarms cost optimization is manual through model and topology selection:

# Strategy 1: Model Selection Per Agent
coordinator = Agent(model_name="gpt-4o")      # $0.01/1K
workers = [
    Agent(model_name="gpt-4o-mini")           # $0.00015/1K
    for _ in range(10)
]

# Strategy 2: Fallback Models
agent = Agent(
    model_name="gpt-4o",
    fallback_models=["gpt-4o-mini", "gpt-3.5-turbo"]
)

# Strategy 3: Local Models
sensitive_agent = Agent(model_name="ollama/llama-3.3-70b")
# No API costs, runs locally

Anti-Patterns: When NOT to Use

Framework Anti-Patterns

DON'T USE CLAUDE-FLOW WHEN:

  • Need production stability (it's Alpha)
  • Team has no MCP/Node.js experience
  • Simple tasks that don't benefit from learning

DON'T USE SWARMS WHEN:

  • Need self-improving system (manual tuning only)
  • Working exclusively with Claude Code
  • Need Byzantine fault tolerance

DON'T USE BMAD WHEN:

  • Need more than 9 agent types
  • Not doing software development
  • Scale beyond single-user workflows

Troubleshooting

Claude-Flow: Alpha Stability Issues

Symptom: Unexpected crashes, inconsistent behavior, or breaking changes between updates.

Cause: Claude-Flow is in alpha (v3.0.0-alpha.104).

Fix: Pin to specific version, monitor GitHub issues, have fallback plan. Consider waiting for stable release for production use.

Swarms: Agent Coordination Overhead

Symptom: Large swarms become slow, excessive inter-agent communication.

Cause: Wrong topology for agent count, or too many dependencies.

Fix:

# Use ForestSwarm for semantic routing at scale
from swarms import ForestSwarm

# Use SpreadSheetSwarm for mass parallel execution
from swarms.structs.spreadsheet_swarm import SpreadSheetSwarm

# Reduce dependencies with ConcurrentWorkflow
from swarms import ConcurrentWorkflow
BMAD: Persona Confusion

Symptom: Wrong agent activated, outputs don't match expected persona.

Cause: Incorrect workflow invocation or persona selection.

Fix: Use explicit workflow names (e.g., *create-prd for John PM). Check bmad-agents/ for persona definitions. Use /bmad to see available agents and workflows.

Multi-Framework Integration Issues

Symptom: Hybrid deployment fails, frameworks don't communicate.

Cause: No built-in integration between frameworks.

Fix: Use file-based handoffs: BMAD outputs documents, Swarms reads them. Export Claude-Flow patterns as JSON for Swarms. Use shared git repos for state synchronization.
