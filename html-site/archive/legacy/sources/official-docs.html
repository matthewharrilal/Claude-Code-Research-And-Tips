CLAUDE.md Configuration

Essential

The single most important configuration file. Provides persistent context Claude loads every session. Think of it as your project's README for Claude.

File locations (priority order):

  1. ~/.claude/CLAUDE.md - Global user preferences
  2. /project/CLAUDE.md - Project-level config
  3. /project/.claude/CLAUDE.md - Alternative location

Non-Interactive Mode

Advanced

Running Claude Code in scripts, CI/CD pipelines, and automation. Headless execution, output parsing, exit codes.

# Single prompt, immediate exit
claude --print "Explain this function"

# Pipe input
git diff HEAD~1 | claude --print "Review these changes"

# JSON output for parsing
claude --print --output-format json "List all functions"

# CI/CD integration
git diff $BEFORE $AFTER | claude --print --dangerously-skip-permissions \
  "Review for security issues"

MCP in Claude Code

Important

How to configure MCP servers in Claude Code. Server setup, environment variables, connection management.

// In .claude/settings.json
{
  "mcp": {
    "servers": {
      "github": {
        "command": "npx",
        "args": ["@anthropic-ai/mcp-server-github"],
        "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" }
      },
      "postgres": {
        "command": "npx",
        "args": ["@anthropic-ai/mcp-server-postgres"],
        "env": { "DATABASE_URL": "${DATABASE_URL}" }
      }
    }
  }
}

Hooks & Permissions

Automation and security configuration.

Hooks System

Important

Event-driven automation that runs shell commands at lifecycle points. Important: Hooks run shell commands, NOT AI prompts.

EventWhen it Fires
PreToolUseBefore any tool executes
PostToolUseAfter any tool completes
SessionStartWhen a session begins
SessionEndWhen a session ends
// Auto-format on edit
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "filePattern": "*.ts",
        "command": "prettier --write $FILE"
      }
    ]
  }
}
docs.anthropic.com/en/docs/claude-code/hooks →

Permissions System

Essential

Control what tools and commands Claude can use. Permission levels: ask, allow, allow-always, deny, deny-always.

Security Warning

Never whitelist: rm -rf, git push --force, git reset --hard, or commands modifying system files.

{
  "permissions": {
    "tools": {
      "Bash": "ask",
      "Read": "allow",
      "Edit": "allow",
      "Write": "ask"
    },
    "commands": {
      "git status": "allow-always",
      "npm test": "allow-always"
    },
    "paths": {
      "deny": [".env*", "*.pem", "node_modules/**"]
    }
  }
}
docs.anthropic.com/en/docs/claude-code/permissions →

Troubleshooting

When things go wrong, start here.

Common Issues

Essential

Authentication issues, performance problems, context management, permission errors.

# Authentication issues
claude auth logout && claude auth login
claude auth status
rm -rf ~/.claude/credentials

# Performance issues
rm -rf ~/.claude/cache
claude config reset

# Context issues
/compact              # Summarize and clear
/reset                # Full reset

# Debug mode
CLAUDE_CODE_DEBUG=1 claude
docs.anthropic.com/en/docs/claude-code/troubleshooting →

Debug Mode

Advanced

Deep debugging with specific categories: API, tools, MCP, hooks, auth.

export CLAUDE_CODE_DEBUG_TOOLS=1
export CLAUDE_CODE_DEBUG_MCP=1

# Log to file
export CLAUDE_CODE_LOG_FILE="/tmp/claude-debug.log"
