Chapter 3

Orchestration Mastery

Multi-Agent Coordination Patterns (Levels 5-7)

12 pages ~3-4 hours Flagship: Gas Town

You Are Here

What you'll learn:

  • Multi-agent coordination patterns (hub-spoke, pipeline, peer-to-peer)

  • Worktree isolation and parallel agent workflows

The Orchestration Shift

From single agent endurance to multi-agent coordination. You're no longer running one assistant longer - you're managing a colony.

"Claude Code is the world's biggest fuckin' ant. Everyone is focused on making their ant run longer... when nature prefers colonies."
- Steve Yegge

Core Concepts

Gas Town

Steve Yegge's factory-scale architecture with 8 specialized roles: Mayor, Deacon, Dogs, Refinery, Polecat, Witness, Crew, and you as Overseer.

Multi-Agent Patterns

Hub-spoke, pipeline, peer-to-peer architectures. Git worktrees for isolation. File-based handoffs for coordination.

Context Management

Token budgets, subagent isolation, fresh context per iteration. Fighting context rot at scale.

The Key Insight

Single agents hit context limits. Parallel agents cause merge conflicts. Orchestrated agents with proper isolation, handoffs, and memory systems can achieve team-scale output as a solo developer.

Chapter Complete When...

  • Understand Gas Town's 8 roles and when to use factory-scale
  • Can choose between hub-spoke, pipeline, and peer-to-peer architectures
  • Know how to use git worktrees for parallel agent isolation
  • Understand context management strategies at multi-agent scale
  • Can implement file-based handoffs between agents
  • Know when Ralph loops are enough vs. when to escalate
After this chapter: You'll orchestrate multiple agents like a colony, not just run one agent longer.
