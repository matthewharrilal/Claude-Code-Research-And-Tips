Orchestration Patterns

Ralph Pattern: Autonomous Development Loops
Fresh context, external state, overnight execution
The persistence pattern for AI coding agents. At its core: while :; do cat PROMPT.md | claude ; done. Ralph spawns fresh Claude instances repeatedly, using external files (prd.json, progress.txt) for memory. Each iteration reads state, picks one task, implements it, commits, and exits. Enables "ship while you sleep" autonomous development.
Level 3-4 5 min setup $10-50/run

Enables Claude Code's disabled native multi-agent orchestration using primitives Anthropic already built. Central orchestrator delegates to specialized workers via TaskCreate/TaskUpdate/TaskGet APIs. Quick Reference
Pattern Selection
Which pattern should I use?
Single features, overnight: Ralph
Parallel tasks, native tools: CC Mirror
Factory scale, 20+ agents: Gas Town

Start with Ralph. Move to CC Mirror when you need parallelism. Gas Town only for Stage 7+ scale.
The Iron Law
Orchestrator vs Worker
Orchestrator: Decompose, assign, track, synthesize
Worker: Execute specific tasks, use tools, report results

Never mix these roles. Violating this causes recursion and context explosion.
Context Rot Warning
When quality degrades
0-50%: Full capability
50-70%: Slight degradation
70-85%: Noticeable loss
85%+: Failure mode

Fresh context beats extended sessions. Always.
