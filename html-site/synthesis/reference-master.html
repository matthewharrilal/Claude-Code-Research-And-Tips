Reference Section
Authoritative data tables, pricing guides, and quantitative benchmarks for Claude Code economics.

The Reference section contains lookup-oriented documentation. Unlike conceptual guides that explain "why" and "how", reference documents provide concrete numbers, formulas, and data you can use directly in planning and decision-making.

When to use Reference docs: Budgeting a project, comparing API vs subscription costs, setting rate limits, measuring ROI, or any time you need specific numbers rather than concepts.

Model Selection Dilemma
Haiku: 80% cheaper, but loses track in long sessions

Inflection Point
When subscription beats API
Below $20/month usage: API wins
$20-100/month: Pro subscription wins
$100-200/month: Max 5x wins
Above $200/month: Max 20x or API depending on predictability

Key variable: Is your usage predictable or bursty?
War Story
YC Hackathon: $297 for 6 repos
Teams ran Ralph loops overnight across 6+ repositories for under $50 each. Key insight: Task decomposition quality drives cost more than raw iteration count.
Failure Gradient
How costs spiral
$5 session: Focused, well-planned
$15 session: Scope creep, context growing
$50 session: Debugging in circles, should have reset
$150+ session: Runaway Ralph loop, no iteration limits

Prevention: Set MAX_ITERATIONS before every Ralph run.
