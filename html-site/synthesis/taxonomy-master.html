These three taxonomies form the foundation of Claude Code understanding. They classify how agents are structured, how they coordinate, and how work flows through them.

Comprehensive classification of agent structures from simple interactive sessions to factory-scale swarm architectures. Includes comparison matrices, the Iron Law of Worker Separation, and production deployment patterns.

Orchestration Pattern Taxonomy

2 Infrastructure Taxonomies

These taxonomies address the "where" and "how" of running Claude Code - deployment environments, context management, and memory persistence strategies.

Minimal Pattern
The irreducible taxonomy
3 core files cover 80%:
- taxonomy-architectures (how to structure)
- taxonomy-orchestration (how to coordinate)
- taxonomy-tools (how to extend)

Start with these three. Add others as needed.
Analogical Bridge
Taxonomies = Periodic Table
Just as the periodic table organizes elements by properties, these taxonomies organize patterns by function.

Architectures = Element families
Orchestration = Bonding rules
Tools = Lab equipment
Constraint Chain
Why these three are "core"
ROOT: Every Claude Code system must answer:
1. How is it structured? (Architecture)
2. How does it coordinate? (Orchestration)
3. How does work flow? (Workflow)

Everything else builds on these foundations.

Trade-Off Tension
Local vs. Cloud deployment
Local: Fast iteration, full control, no cost per token
Cloud: Scale, always-on, team access
Hybrid: Best of both but operational complexity

See deployment taxonomy decision tree for guidance.
