Pattern Grammar Master Reference

This page consolidates the vocabulary and syntax of Claude Code patterns - the language we use to describe and compose patterns.

Core Vocabulary

Agent Terms

  • Orchestrator: The agent that coordinates others
  • Worker: An agent that executes tasks
  • Swarm: Multiple workers in parallel
  • Colony: Multiple orchestrators coordinating

State Terms

  • Context: The current conversation state
  • Memory: Persistent state across sessions
  • State file: External file holding agent state

Loop Terms

  • Iteration: One pass through a loop
  • Fresh context: Starting a new session
  • Compounding: Each iteration builds on previous

Pattern Syntax

Patterns are described using composition operators:

Pattern = Primitive | Pattern + Pattern | Pattern * Pattern

Examples:
- Ralph = Loop + StateFile + PRD
- CC Mirror = Orchestrator + Worker* + TaskAPI
- Gas Town = Mayor + Deacon + Dogs* + Refinery

Composition Rules

  • Sequential (+): One pattern after another
  • Parallel (*): Multiple patterns simultaneously
  • Nested (→): One pattern contains another
  • Conditional (?): Pattern based on condition
