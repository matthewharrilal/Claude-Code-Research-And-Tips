Part 1: Foundations

Everything builds on these principles. Understand them first, and you can derive any pattern yourself.

The Eight Core Principles

Principle 1

Context Is The Primary Constraint

Claude's context window is finite (~200K tokens). Quality degrades as context fills (observable at 60-70%). Long sessions accumulate noise, contradictions, and cruft.

Every pattern in this playbook ultimately optimizes around context limits. This is THE constraint that shapes everything.

Principle 2

External State > Internal Memory

Claude's "memory" lives only in the context window. When the session ends, everything is lost.

Store state in files (prd.json, progress.txt), in git (commits as memory), and in external tools.

Principle 3

Fresh Context > Extended Sessions

Long sessions degrade in quality. Context becomes polluted with false starts, corrections, tangents.

Quality of 20 five-minute sessions > Quality of one 100-minute session
Principle 4

Orchestrator/Worker Separation

The Iron Law: Orchestrators coordinate. Workers execute. Never mix.

  • Orchestrator: Creates tasks, monitors, decides. NEVER writes code.
  • Worker: Executes tasks, reports results. NEVER spawns other workers.
Principle 5

Atomic, Verifiable Tasks

"Build the auth system" is too big. "Make it work better" can't be verified.

Break into smallest testable units. Each task fits in one context window. Each has explicit acceptance criteria.

Principle 6

Quality Gates Before Commit

Never mark work as "done" without verification.

npm run typecheck && npm run test && npm run lint && git commit
Principle 7

Isolation For Parallel Work

Multiple agents editing the same files = merge conflicts. Shared state = race conditions.

Use git worktrees for file isolation, separate branches per feature, domain-specific Claude instances.

Principle 8

Explicit Communication Channels

Agents can't read minds. Assumptions lead to wrong work.

Use progress files, structured task definitions (JSON), clear acceptance criteria, and hooks for notifications.

Checkpoint: Principles

You should now understand WHY patterns work. The key insight: context is the constraint, external state is the solution, fresh starts beat long sessions.

Building Blocks: The Primitives

Patterns are built from these atomic components. Understand the primitives to understand any pattern.

Loop Primitives
# Bash While Loop - indefinite iteration
while [ condition ]; do
  # agent iteration
done

# For Loop with Counter - bounded iteration
for i in $(seq 1 $MAX_ITERATIONS); do
  # iteration $i
done

# Stop Condition Check
if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
  exit 0
fi
State Primitives

JSON Task List (prd.json):

{
  "userStories": [
    {"id": "US-001", "title": "...", "passes": false}
  ]
}

Append-Only Log (progress.txt):

## 2026-01-09 - US-001
- Implemented: login form
- Learnings: Use Zod for validation

Git As Memory: Commits preserve state. History is searchable. Can revert to known-good states.

What Fails & Why: Anti-Patterns

Context Rot

Single long sessions where quality degrades over time. Claude "forgets" earlier decisions, contradicts itself, makes errors it wouldn't make fresh.

Fix: Fresh context per iteration + external state.

Orchestrators That Write Code

An orchestrator agent that also tries to implement. Context fills with implementation details, loses big-picture view.

Fix: Iron Law: Orchestrators coordinate, workers execute.

Workers Spawning Workers

Worker agents that spawn their own sub-workers. Leads to infinite recursion, loss of control, context explosion.

Fix: Workers report back to orchestrator. Only orchestrator spawns.

Vague Tasks

Tasks like "make it better" or "fix the issues." No clear completion criteria, agent guesses what you want.

Fix: Explicit acceptance criteria. Testable conditions.

Part 2: Your Capability Progression

Your journey from single sessions to factory-scale development. Each level builds on the previous.

Level 0: Where You Are Now

0

Single Session

Manual approval, context decay, 90-minute limit

The Situation:

  • All work happens in one continuous conversation
  • Manual approval for every operation
  • Context degrades after 60% fill
  • 90 minutes leads to session death or burnout

The Pain Points:

  • Re-explaining project context every session
  • Quality drops as you approach token limits
  • Session death means lost progress
  • You must be present the entire time
Example: Building Auth at Level 0

11:00 PM - Start session, explain project
11:15 PM - Ask for login form, approve changes
11:45 PM - Context at 55%, Claude getting confused
12:15 AM - Context at 75%, quality noticeably worse
12:30 AM - Give up, feature incomplete

Result: 90 minutes invested, feature not done, frustrated.

Levels 1-3: Quick Wins to Autonomy

1

First Quick Wins

CLAUDE.md + Slash Commands (Week 1)

What You Add: CLAUDE.md for project context that persists, 3-5 slash commands for common workflows.

Before

"I need you to review this code. We're using Next.js 14 with TypeScript, server actions, Prisma ORM..." [5 minutes of context every session]

After

/review
[Claude already knows your stack from CLAUDE.md]

What You Gain:

  • Token savings: ~20% per session
  • Consistency: Claude behaves the same across sessions
  • Speed: Common workflows become one-word commands
2

Specialization

Subagents + Hooks (Week 2)

What You Add: Specialized AI workers (subagents), automated responses to events (hooks).

Before

One Claude doing everything, context polluted with unrelated work.

After

Specialized agents with clean, focused contexts: security reviewer, test writer, performance auditor.

What You Gain: Isolated contexts, automatic formatting, specialized quality.

3

Autonomy

Ralph Loop + External Memory (Week 3-4)

What You Add: Ralph Loop for autonomous iteration, external memory (progress.txt, prd.json).

The Ralph Pattern
./ralph.sh 25  # Run up to 25 iterations
# Go to sleep
# Wake up to completed feature

What Happens:

  • Each iteration: fresh Claude instance (0% context)
  • Reads state from files (prd.json, progress.txt)
  • Picks highest priority incomplete task
  • Implements, verifies, commits
  • Appends learnings, exits
  • Next iteration: fresh context, updated files

What You Gain:

  • Human time: 90 min down to 10 min (setup only)
  • Fresh context: Every iteration starts clean
  • Sleep shipping: Features complete overnight
Checkpoint: Levels 0-3

If you implement Levels 1-3, you've already transformed your workflow. Human time drops from 90 minutes to 10 minutes. Features ship while you sleep.

Levels 4-7: Reliability to Factory Scale

4

Reliability

PRD-driven Ralph (Month 1)

Add explicit acceptance criteria, progress tracking, learning accumulation. Success rate goes from 60% to 95%.

5

Parallelism

CC Mirror / Hub-and-Spoke (Month 2)

Orchestrator coordinates multiple workers in parallel. Model tiering: Haiku for lookups, Sonnet for coding, Opus for architecture. 2-3x faster.

6

True Parallelization

Git Worktrees + Parallel Ralph (Month 3)

Complete file isolation with git worktrees. 3 features in parallel = 1 hour instead of 3 sequential hours.

7

Factory Scale

Gas Town Architecture (Month 6+)

Persistent named workers (Polecats), self-improving systems, automatic task decomposition. Team equivalent: 3-5 mid-level developers. 24/7 operation.

Prerequisites

100+ hours multi-agent experience. $50-200/day budget tolerance. Comfort with emergence and chaos.

Part 3: Deep Pattern Mastery

Context Management

Context Is THE Primary Constraint

Context Level Quality Impact

50-70%Slight degradation possible
70-85%Noticeable quality loss
85-95%Significant degradation
95%+Failure mode
The Ralph Loop

Complete Implementation

"Think of the AI agent as an eager but somewhat dim junior developer (Ralph) who never sleeps, happily works on one tiny task at a time, commits often, and tells you when it is truly finished."

Task Sizing - The Most Critical Factor:

If you can't describe it in 2-3 sentences, it's too big.

Right-Sized Tasks
  • Add a database column + migration
  • Add a UI component to existing page
  • Write tests for one module
  • Implement one API endpoint
Too Big (Split These)
  • "Build the entire dashboard"
  • "Add authentication"
  • "Create e-commerce flow"
Checkpoint: Deep Patterns

The key insight: context management + fresh iterations + right-sized tasks = reliable autonomous development.
