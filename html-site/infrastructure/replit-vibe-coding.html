Replit + Claude Code - Mobile Vibe Coding

Full-stack development from your phone with instant preview and deployment

You Are Here

Context: This is the phone-first approach to mobile Claude Code development. Unlike the remote Mac approach (which remotes into an always-on machine), this uses Replit as your cloud IDE - no personal machine required.

Where this fits: Perfect when you want to "vibe code from bed" - loose, iterative, voice-driven development where AI does 90%+ of the work and you steer direction.

Alternative to:

  • System (Remote Mac) - requires always-on Mac
  • Cloud VMs with SSH - more setup friction
  • GitHub Codespaces - less mobile-optimized

Setup Guide

Step 1: Install Replit Mobile App

Download from iOS App Store or Google Play Store.

Checkpoint: Replit App Installed

You should see:

  • Replit app icon on your phone
  • Able to sign in with GitHub/Google/email
  • "My Repls" screen after login
App won't install or crashes on launch
  • iOS: Requires iOS 15+. Check Settings → General → Software Update
  • Android: Requires Android 8+. Clear Play Store cache if stuck
  • Storage: Need ~200MB free space

Step 2: Create New Repl/Project

My Repls:

  dentist-booking-app
  portfolio-site
  api-backend

  [+ Create Repl]  ← TAP THIS

Tap [+ Create Repl], then:

  1. Select a template (Next.js, Python, etc.)
  2. Name your project
  3. Tap [Create Repl]
Timing: Project creation takes 30-60 seconds (framework scaffolded, dependencies installed, dev server started)

Checkpoint: Project Created

You should see:

  • File explorer with project files
  • Code editor pane
  • Shell icon (for terminal)
  • Webview pane (live preview)
Repl stuck at "Creating..."
  • Network: Ensure stable WiFi/cellular connection
  • Timeout: Wait up to 2 minutes. If still stuck, force close app and retry
  • Free tier limits: Free accounts limited to 3 active Repls. Delete unused ones

Step 3: Install Claude Code CLI

Tap the Shell icon to open terminal, then run:

npm install -g @anthropic/claude-code-cli
Timing: Installation takes 10-20 seconds

Checkpoint: Claude Code CLI Installed

You should see:

added 47 packages in 12s

Claude Code CLI installed

~/your-project$ _
npm install fails with permission error

Replit should handle this automatically. If not:

npm install -g @anthropic/claude-code-cli --prefix ~/.local

Then add to PATH: export PATH="$HOME/.local/bin:$PATH"

Command not found after install
  • Run hash -r to refresh shell
  • Or start a new terminal session
  • Verify with which claude

Step 4: Authenticate

Set your Anthropic API key as an environment variable:

export ANTHROPIC_API_KEY="sk-ant-your-key-here"

Better approach: Add to Replit Secrets (padlock icon) so it persists:

  1. Tap padlock icon in Replit
  2. Add key: ANTHROPIC_API_KEY
  3. Add value: sk-ant-your-key-here

Step 5: Start Claude Code

claude

Checkpoint: Claude Code Running

You should see:

CLAUDE CODE
Replit Environment

Model: claude-sonnet-4.5
Project: /home/runner/your-project
Preview: https://your-project.username.repl.co

claude> _
API key not recognized
  • Verify key starts with sk-ant-
  • Check Anthropic console for key validity
  • Ensure no extra spaces in key
  • Try: echo $ANTHROPIC_API_KEY to verify it's set

Vibe Coding Workflow

Pro Tip: Use Voice Dictation

Typing long prompts on mobile is painful. Use your phone's voice dictation:

  • iOS: Tap microphone on keyboard
  • Android: Tap microphone on Gboard

Power Combo: Replit Agent + Claude Code

Replit Agent

Use for:

  • Big structural changes
  • Scaffolding new features
  • Auth, database setup

Uses Replit credits

Claude Code

Use for:

  • Detailed/iterative work
  • Fine-grained edits
  • Polish and refinements

Uses Anthropic API

Example Workflow

  1. Replit Agent: "Add user authentication with email/password login, signup, and password reset"
  2. Wait for scaffolding to complete
  3. Claude Code: "Improve the login form validation and add better error messages"
Key Insight

Claude Code doesn't consume Replit credits - only Replit Agent does. This makes Claude Code the economical choice for iterative refinement.

Deploying from Phone

claude> Deploy this to production

Claude will run the build and Replit auto-deploys:

[Claude running...]

$ npm run build
  Build successful

Replit auto-deploys on build!
  Live at: https://your-project.username.repl.co

Share this URL with anyone.

Checkpoint: First Deployment

You should see:

  • Build completes without errors
  • Live URL displayed
  • URL accessible from any device/browser
Build fails with errors
  • Ask Claude: "Fix the build errors and try again"
  • Check for TypeScript errors: npx tsc --noEmit
  • Check for missing dependencies: npm install
Preview works but deployed site shows errors
  • Development vs production differences - check environment variables
  • Verify all API keys are set in Replit Secrets
  • Check browser console for specific errors
You've shipped from your phone!

Caveats & Gotchas

Issue Notes
Extremely addictive Risk of too much phone time - set boundaries!
Mobile typing pain Use voice dictation religiously
Preview limitations Mobile view ≠ full desktop. Test in multiple views
API costs Paid Anthropic usage - monitor your spend
Voice dictation misunderstands technical terms

Use meta-prompting to clean up:

"I'm going to dictate a prompt. Clean it up and interpret what I mean: [voice dictation here]"

Replit session disconnects frequently
  • Replit free tier has idle timeouts
  • Keep terminal active with commands
  • Consider Replit paid plan for longer sessions
Claude Code seems slow on mobile
  • Network latency adds to response time
  • Large file operations are slower over mobile data
  • Use WiFi when possible for better experience
