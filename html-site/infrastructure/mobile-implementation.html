Step 1: tmux Auto-Attach

When you SSH into your VM, you want to immediately be inside tmux. No extra commands. This snippet in ~/.zshrc handles it.

Add to ~/.zshrc

if [[ -z "$TMUX" ]]; then
  tmux attach -t main 2>/dev/null || tmux new -s main
fi

What it does:

  • If not already in tmux, try to attach to session named "main"
  • If "main" doesn't exist, create it
  • Result: every SSH login drops you into tmux automatically
$ ssh user@100.x.x.x # Connect to VM [Auto-attaching to tmux...] # The magic! Either: - Attaching to existing 'main' session (if exists) - Creating new 'main' session (if doesn't exist) +-----------------------------------------------+ | tmux: session 'main' | +-----------------------------------------------+ user@vm:~$ _ # Already in tmux!

Checkpoint: tmux Auto-Attach Working

After adding to ~/.zshrc and reconnecting:

  • You should see tmux status bar at bottom of terminal
  • Running tmux ls should show "main" session
  • Disconnecting and reconnecting resumes same session

Step 2: Notification Hook

When Claude needs input (asks you a question), you want a push notification on your phone. This uses Claude's hook system.

2a. Create hook directory

mkdir -p ~/.claude/hooks

2b. Create notification script

Save this as ~/.claude/hooks/poke-notify.sh:

#!/bin/bash
# Extract question from Claude's event data
QUESTION=$(echo "$EVENT_DATA" | jq -r '.tool_input.questions[0].question')

# Your project name for context
PROJECT_NAME="my-feature-branch"

# Build message
MESSAGE="$PROJECT_NAME: Claude needs input - $QUESTION"

# Send to Poke (or your notification service)
curl -X POST "https://api.pokepush.com/notify" \
  -H "Authorization: Bearer $POKE_TOKEN" \
  -d "{\"message\": \"$MESSAGE\"}"

2c. Make it executable

chmod +x ~/.claude/hooks/poke-notify.sh

2d. Configure Claude settings

Save this as ~/.claude/settings.json:

{
  "hooks": {
    "PreToolUse": [{
      "matcher": "AskUserQuestion",
      "hooks": [{
        "type": "command",
        "command": "~/.claude/hooks/poke-notify.sh question"
      }]
    }]
  }
}

Checkpoint: Notification Hook Working

Test by starting Claude and giving it a task that requires a question:

claude
> Build a login form and ask me about the design

You should see:

  • Claude creates the form and asks a styling question
  • Within 2-3 seconds, your phone gets a push notification
  • Notification contains project name + question text
Important: Set your POKE_TOKEN environment variable before testing. Add to ~/.zshrc: export POKE_TOKEN="your-token-here"

Step 3: Git Worktrees for Parallel Agents

Each feature branch gets its own directory, its own tmux window, and its own Claude instance. They don't interfere with each other.

Create worktrees for each feature branch

cd ~/Code/myproject

# Create worktrees for parallel development
git worktree add ../myproject-sidebar feature/sidebar
git worktree add ../myproject-dark-mode feature/dark-mode

Result: Three independent working directories

~/Code/ ├── myproject/ # main branch (original repo) ├── myproject-sidebar/ # feature/sidebar (worktree) └── myproject-dark-mode/ # feature/dark-mode (worktree)

Each gets its own tmux window

+-----------------------------------------------+ | tmux windows: | |------------------------------------------------| | 0: main ~/Code/myproject/ | | 1: sidebar ~/Code/myproject-sidebar/ claude running | 2: dark-mode ~/Code/myproject-dark-mode/ claude running +-----------------------------------------------+

Create tmux windows:

# Create named windows for each worktree
tmux new-window -n sidebar -c ~/Code/myproject-sidebar
tmux new-window -n dark-mode -c ~/Code/myproject-dark-mode

# Switch between them
tmux select-window -t sidebar    # Ctrl+b, then window number also works

Checkpoint: Worktrees Set Up

  • git worktree list shows all three worktrees
  • Each directory has its own branch checked out
  • Changes in one worktree don't affect others
  • You can run claude in each window independently

Step 4: Port Allocation

When running dev servers in multiple worktrees, they fight over ports. This deterministic allocation gives each branch a unique port.

Save as port_for_branch.py

#!/usr/bin/env python3
import sys

branch_name = sys.argv[1]
hash_val = sum(ord(c) for c in branch_name)
django_port = 8001 + (hash_val % 99)
print(django_port)

Usage

$ python3 port_for_branch.py sidebar
8047                                   # sidebar gets port 8047

$ python3 port_for_branch.py dark-mode
8023                                   # dark-mode gets port 8023

Key insight: Same branch always gets the same port. Deterministic = no conflicts, no remembering.

If you skip this: Multiple dev servers will fight for the same port, causing confusing "Address already in use" errors.

Step 5: mosh Connection

mosh (mobile shell) handles network transitions seamlessly. WiFi to cellular? Phone sleeps? Connection resumes instantly.

Connect using mosh

mosh --ssh="ssh -p 47892" user@100.x.x.x

WiFi to CellularCONNECTION LOSTSeamless

Spotty connectionFROZEN / TIMEOUTSurvives

  • Connection shows "mosh" in your terminal client
  • Turning WiFi off then back on doesn't disconnect
  • Switching from WiFi to cellular maintains session

Troubleshooting

Common issues and their fixes. Click to expand each problem.

mosh doesn't forward SSH agent - Git push fails

Problem: When you try to git push through mosh, GitHub authentication fails because mosh doesn't forward your SSH agent.

Solution: Manually add your key inside the tmux session:

# Inside tmux, when you need to push:
ssh-add ~/.ssh/id_ed25519           # Add key manually
Identity added: /home/user/.ssh/id_ed25519

git push                            # Now works!

Permanent fix: Add ssh-add ~/.ssh/id_ed25519 2>/dev/null to your ~/.zshrc after the tmux block.

Notification hook doesn't fire

Check these in order:

  1. Script executable? Run ls -la ~/.claude/hooks/poke-notify.sh - should show -rwx
  2. jq installed? Run which jq - should return a path
  3. POKE_TOKEN set? Run echo $POKE_TOKEN - should show your token
  4. settings.json valid? Run cat ~/.claude/settings.json | jq . - should parse without error
  5. Hook matcher correct? Must be exactly "AskUserQuestion" (case-sensitive)

Test manually:

export EVENT_DATA='{"tool_input":{"questions":[{"question":"Test?"}]}}'
~/.claude/hooks/poke-notify.sh question
mosh connection fails to establish

Problem: mosh hangs or times out when connecting.

Common causes:

  1. mosh-server not installed on VM: 
    sudo apt install mosh
  2. UDP ports blocked: mosh uses UDP 60000-61000. Open these in your firewall: 
    # UFW (Ubuntu)
sudo ufw allow 60000:61000/udp

# Or check if it's a Vultr/cloud firewall issue
  3. Wrong SSH port specified: Make sure -p matches your actual SSH port
tmux session doesn't persist after disconnect

Problem: When you reconnect, your tmux session is gone.

Check:

  1. Did the VM reboot? Check uptime
  2. Did you exit tmux cleanly vs just disconnecting?
  3. Is tmux running? pgrep tmux

Prevention: The auto-attach snippet in ~/.zshrc handles this, but make sure you disconnect by closing the app, not by typing exit in tmux.

Port collision between worktrees

Problem: "Address already in use" when starting dev server.

Solutions:

  1. Use the port allocation script: See Step 4 above
  2. Find what's using the port: 
    lsof -i :8000
  3. Kill the process: 
    kill $(lsof -t -i :8000)
Git worktree shows "already checked out"

Problem: Can't create worktree because branch is checked out elsewhere.

Solution:

# Find where branch is checked out
git worktree list

# Remove old worktree (if directory deleted but not unregistered)
git worktree prune

# Force create if you really want it
git worktree add --force ../myproject-sidebar feature/sidebar

Alternative Notification Services

The Poke webhook pattern can be adapted for other services. Just change the curl endpoint and payload:

ServiceNotes
ntfyOpen source, self-hostable, free
PushoverOne-time $5 purchase, reliable
Telegram BotFree, requires bot setup
Discord WebhookFree if you live in Discord

Example: ntfy adaptation

#!/bin/bash
QUESTION=$(echo "$EVENT_DATA" | jq -r '.tool_input.questions[0].question')
MESSAGE="$PROJECT_NAME: Claude needs input - $QUESTION"

# ntfy instead of Poke
curl -d "$MESSAGE" ntfy.sh/your-private-topic
