.md Files = Superpower

Source: @gregisenberg (Greg Isenberg) Date: 2026-01-04 Status: VIRAL

The Principle

Markdown files as structured knowledge bases for AI tools:

  • Better context management
  • Automation via skills
  • Agentic workflows
  • 80% better first-try outputs

2. MDFlow (mdflow.dev)

Treat .md files as CLI-executable scripts:

---
model: sonnet
---
Write a short poem about {argument}.

Run: mdflow poem.claude or ./poem.claude.md

Pipe outputs: ./tree.gemini.md | ./md-to-html.gemini.md

Mental Model

"View .md as building blocks for an AI-enhanced personal OS - like hiring specialists that reference your curated context on demand."

Key Principles

  • Structure beats chaos
  • Upload once, benefit forever
  • Composability (pipe outputs)
  • Minimalism (few files yield 99% outperformance)

Caveats

  • Skills may not auto-trigger on ambiguous queries
  • Large .md files can hit context limits
  • MDFlow requires CLI comfort + API keys

Example brand-voice.md

# Brand Voice Guide

## Tone
- Professional but approachable
- Direct, no fluff
- Technical when needed, clear always

## Values
- Clarity over cleverness
- Substance over style
- Action over theory

## Vocabulary
- Use: "build", "ship", "iterate", "learn"
- Avoid: "leverage", "synergy", "paradigm shift"

## Examples
Good: "Here's how to fix it"
Bad: "Let's unpack the situational dynamics"
