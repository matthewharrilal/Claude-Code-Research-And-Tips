The 6-Component Stack

1

LSP - Language Server Protocol

IDE-level code intelligence that gives Claude the same superpowers your editor has:

  • Real-time diagnostics - Catch errors before running
  • Type information - Understand the shape of data
  • Go-to-definition - Navigate codebases intelligently
  • Autocomplete context - Know what's available

Why it matters: Without LSP, Claude is coding blind. With it, Claude has the same visibility into your codebase that you do.

2

Hooks - Custom Automation

Hooks intercept Claude's actions at key moments, enabling custom automation:

PreToolUse   --> Intercept before tool execution
PostToolUse  --> Validate after tool execution
Stop Hook    --> Prevent premature exits

Key insight: The Stop Hook is crucial for the unhobbled pattern. It intercepts Claude's attempts to end the session and redirects to continuation logic.

3

Subagents - Task Delegation

Instead of one agent doing everything, delegate specialized tasks to isolated subagents:

  • Each subagent has its own clean context
  • Prevents context rot in the main agent
  • Enables parallel work on independent tasks
  • Specialized agents for testing, documentation, refactoring

Pattern: Main agent orchestrates, subagents execute. Main agent stays lean, subagents handle heavy lifting.

4

Adversarial Validations

Robust testing and checks that keep outputs honest. Claude will drift without external validation:

  • Test suites that must pass before "complete"
  • Type checking as a verification gate
  • Linting for code quality enforcement
  • Self-review prompts - "What could go wrong with this?"
Without validators, Claude will drift. It will confidently mark features "complete" without proper testing. Build validation into the workflow, not as an afterthought.
5

Ralph Wiggum Loops

Persistent, iterative loops for continuous autonomous work. The pattern that enables overnight coding:

while :; do
  cat PROMPT.md | claude
  # Check for completion promise
  # If found, break
done

The key: Each iteration spawns a fresh Claude instance with zero context pollution. Continuity comes from external state - git, files, progress logs.

See Fresh Context Per Iteration for the underlying philosophy.

6

2-Way Voice (STT/TTS)

Hands-free, voice-driven interaction that enables truly untethered development:

  • Speech-to-text for input - dictate instructions while walking
  • Text-to-speech for output - hear progress updates
  • Enables "cardio coding" - development during exercise
  • Like pair programming with a voice assistant

Note: Voice requires custom apps/wrappers - not native to Claude Code yet. Community solutions exist.

Ralph Loop Integration

The Ralph Wiggum loop is the backbone that ties the stack together. Here's how it enables overnight autonomous coding:

Key Principles

  1. Never let the agent stop prematurely - Intercept exits with hooks/loops
  2. Offload state to files/git - Not session memory, for infinite context
  3. Validate aggressively - Self-correct with adversarial checks
  4. Go voice-first - For untethered, human-like flow

Caveats & Gotchas

Significant Upfront Setup Required

The Issue: The full unhobbled stack requires meaningful configuration and tooling setup.

Mitigation: Start with the official Ralph plugin (/ralph-loop, /cancel-ralph). Add components incrementally. You don't need all 6 components on day one.

Path: Ralph loop first -> Hooks second -> Validators third -> Voice last

API Rate Limits & Costs

The Issue: Autonomous loops can burn through API credits quickly.

Mitigation: Monitor usage actively. Set budget limits. Use the --max-turns flag. Consider off-peak hours for long runs.

Typical cost: $10-12 per operating hour for heavy autonomous work.

Risk of Infinite Loops

The Issue: Without proper exit conditions, loops run forever wasting tokens.

Mitigation: Always define a "completion promise" - explicit criteria for when to stop. Include timeout conditions. Monitor for stuck states.

Pattern: Write "TASK_COMPLETE" to a file when done. Loop checks for this marker.

Voice Requires Custom Apps

The Issue: STT/TTS is not native to Claude Code CLI.

Mitigation: Use community-built iOS/Android wrappers. Consider Whisper for STT, system TTS for output. This is the highest-friction component.

Don't Do This

Don't rely on default single-session prompting - Leads to compaction and loss of context over time.
Don't skip validators - Claude will drift without external checks. What looks complete may not work.
Don't ignore exit conditions - Infinite loops waste tokens and produce garbage. Always define "done."
