Unit 1 -- Core Curriculum

The Eight Foundational Modules

These modules form the core curriculum. All subsequent coursework builds on them. Complete before enrolling in any elective.

ORCH 101
Context Is the Primary Constraint

Claude's context window (~200K tokens) is finite. Quality degrades as it fills. Every pattern optimizes around this single constraint.

ORCH 102
External State > Internal Memory

Store state in files, git, and databases. External state survives session death and can be read by fresh agents.

ORCH 103
Fresh Context > Extended Sessions

Quality of 20 five-minute sessions exceeds one 100-minute session. Deliberately restart with clean context.

ORCH 104
Orchestrator/Worker Separation

The Iron Law: Orchestrators coordinate, workers execute. Never mix. Clean separation prevents context pollution.

ORCH 105
Atomic, Verifiable Tasks

Break work into smallest testable units. Each fits in one context window with explicit acceptance criteria.

ORCH 106
Quality Gates Before Commit

Typecheck, test, lint -- then commit. Broken code compounds. Early failures save rework.

ORCH 107
Isolation for Parallel Work

Git worktrees for file isolation. Separate branches per feature. Domain-specific instances.

ORCH 108
Explicit Communication Channels

Progress files agents can read. Structured task definitions. Clear criteria. Hooks for notifications.

Unit 1.2 -- Common Errors

Frequently Failed Examinations

Students who fail typically exhibit these patterns. Study them before your first practicum.

Common Error: Context Rot

Single long sessions where quality degrades. Claude forgets decisions, contradicts itself. Correction: Fresh context per iteration + external state. Always.

Common Error: Orchestrators Writing Code

An orchestrator agent that implements. Context fills with details, loses oversight. Correction: The Iron Law. Coordinators never execute.

Common Error: Missing Verification

Marking work complete without testing. Broken code compounds. Correction: Quality gates before every commit. No exceptions.

Semester 3: Practicum

The Ralph Loop -- Autonomous Study

Independent study begins. The Ralph Loop provides autonomous iteration with fresh context. External state carries knowledge between sessions. The student can now study while sleeping.

#!/bin/bash
MAX_ITERATIONS=${1:-25}

for i in $(seq 1 $MAX_ITERATIONS); do
  echo "=== Study Session $i ==="
  OUTPUT=$(cat prompt.md | claude --dangerously-skip-permissions)
  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    echo "Coursework complete!" && exit 0
  fi
done
Before: Manual Study

Assignment incomplete.

After: Autonomous Study

./ralph.sh 25
Start the session. Sleep.
Wake to completed work.
Student time: 10 minutes.
