From apprentice to master -- a craftsperson's progression through orchestration, context management, and agent engineering
The Tool Wall -- Core Instruments

The Eight Principles

These are the tools on the wall. Every pattern in this workshop is built with them. Learn what each does before picking one up.

Principle 01

Context Is the Primary Constraint

Claude's context window is finite (~200K tokens). Quality degrades as it fills. Every pattern here optimizes around this single limitation.

Principle 02

External State > Internal Memory

Store state in files, git, and databases. External state survives session death and can be read by fresh agents.

Principle 03

Fresh Context > Extended Sessions

Quality of 20 five-minute sessions exceeds one 100-minute session. Deliberately restart with fresh context.

Principle 04

Orchestrator/Worker Separation

The Iron Law: Orchestrators coordinate. Workers execute. Never mix. Clean boundaries prevent context pollution.

Applied Practices
Principle 05

Atomic, Verifiable Tasks

Break work into smallest testable units. Each task fits in one context window with explicit acceptance criteria.

Principle 06

Quality Gates Before Commit

Never mark work "done" without verification. Typecheck, test, lint -- then commit. Broken code compounds.

Principle 07

Isolation for Parallel Work

Git worktrees for file isolation. Separate branches per feature. Domain-specific Claude instances prevent conflicts.

Principle 08

Explicit Communication Channels

Progress files agents can read. Structured task definitions. Clear acceptance criteria. Hooks for notifications.

Correction Marks -- Common Mistakes

What the Apprentice Must Unlearn

The master's corrections are not criticism -- they're accumulated wisdom. These are the naive approaches every apprentice tries first, and why the master says no.

Correction: Context Rot

The naive approach: Single long sessions where quality degrades over time. Claude "forgets" decisions, contradicts itself, gets slower.

The master's way: Fresh context per iteration + external state. Never let the workspace get cluttered.

Correction: Orchestrators That Code

The naive approach: An orchestrator agent that also tries to implement. Context fills with details, loses the big picture.

The master's way: The Iron Law. The foreman doesn't pick up a hammer. Orchestrators coordinate, workers execute.

Correction: Workers Spawning Workers

The naive approach: Worker agents that spawn their own sub-workers. Infinite recursion, loss of control.

The master's way: Workers report back. Only the orchestrator delegates. One hierarchy, clear authority.

Level 2

Specialization

The apprentice learns that different tools serve different purposes. Subagents provide isolated, focused contexts. A security reviewer reads but never writes. A test writer focuses only on tests. Hooks automate repetitive tasks.

Key Technique

Isolated contexts mean each agent stays focused. Specialization produces better work than a generalist doing everything.

Level 3

Autonomy -- The Ralph Loop

The apprentice's first real project. The Ralph Loop grants autonomous iteration with fresh context. State survives across iterations in external files. The apprentice can now sleep while work continues.

#!/bin/bash
MAX_ITERATIONS=${1:-25}

for i in $(seq 1 $MAX_ITERATIONS); do
  echo "=== Iteration $i ==="
  OUTPUT=$(cat prompt.md | claude --dangerously-skip-permissions)

  if echo "$OUTPUT" | grep -q "<promise>COMPLETE</promise>"; then
    echo "Done!"
    exit 0
  fi
done
Before: The babysitter

You babysit Claude for 90 minutes. Manual approval. Constant supervision. Can't step away.

After: The autonomous craftsperson

./ralph.sh 25
Go to sleep.
Wake up to completed feature.
Human time: 10 minutes.

At the Workbench -- A Night in the Workshop

The Overnight Shipping Story

6:47 PM, Wednesday. You've been staring at the Jira board for three weeks. The authentication feature keeps slipping. Tonight, that changes.

You write the PRD -- 5 user stories, each atomic, each with verifiable acceptance criteria. You start the Ralph loop. You grab your jacket and leave.

The workshop works through the night.

Iteration 1: Fresh instance. Reads the plan. Implements the user model. Commits. Iteration 2: Fresh context. Reads the learnings. Implements login. Commits. Iterations 3-5: Database, form, tests. Each fresh. Each committed.

Iteration 6: All stories pass. The loop exits.

7:15 AM, Thursday. Five clean commits. CI green. The app works. Three weeks of Jira tickets -- shipped while you slept.

Level 4-5

Journeyman -- Reliability and Parallelism

The journeyman adds explicit acceptance criteria, progress tracking, and learning accumulation. Success rate climbs from 60% to 95%. Then comes multi-agent coordination -- CC Mirror provides hub-and-spoke orchestration. Multiple workers execute in parallel. Right model for each task.

The Journeyman's Edge

The difference between apprentice and journeyman is not skill -- it's judgment. Knowing WHEN to use each tool, not just HOW. Which model for which task. Which architecture for which scale. The decision frameworks become instinct.

Level 6-7

Master -- Factory Scale

Git worktrees provide complete file isolation. Parallel Ralph loops run multiple features simultaneously. Three features that took three hours sequentially now complete in one hour.

At Level 7, the Gas Town architecture: persistent named workers, self-improving systems, automatic decomposition. The master runs a factory. Team equivalent of 3-5 mid-level developers. Continuous 24/7 operation. The factory learns from its own runs.
