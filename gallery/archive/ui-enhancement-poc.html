Demo 3: Card Hover Physics

3D tilt based on cursor position

CSS (fixed hover state)

Hover me

Fixed translateY + scale
300ms transition

GSAP (cursor-tracked 3D)

Hover me

3D tilt tracks cursor
Elastic spring return

/* CSS Version */
.card {
  transition: transform 0.3s ease;
}

.card:hover {
  transform: translateY(-8px) scale(1.02);
}
// GSAP Version
card.addEventListener('mousemove', (e) => {
  const rect = card.getBoundingClientRect();
  const x = (e.clientX - rect.left) / rect.width;
  const y = (e.clientY - rect.top) / rect.height;

  gsap.to(card, {
    rotateY: (x - 0.5) * 20,
    rotateX: (y - 0.5) * -20,
    ease: "power2.out",
    duration: 0.3
  });
});

  }
});

Quality Comparison

Aspect CSS Approach GSAP Approach
Spring Physics Pre-calculated bezier approximation True elastic/back easing
Overshoot Limited, calculated at design time Dynamic, configurable bounce
Cursor Tracking Not possible (static states only) Full position-based animation
Stagger Control Manual delay per element Automatic stagger with timing
Bundle Size 0 KB (native CSS) ~60 KB (GSAP core + plugins)
Performance GPU-accelerated by default Optimized, but JS execution

Recommendation

Use CSS for simple state transitions (hover effects, visibility toggles) where perceived quality difference is minimal.

Use GSAP for premium interactions where physics feel matters (copy button morphs, tab indicators, scroll reveals) and where the 60KB bundle is justified by UX improvement.
