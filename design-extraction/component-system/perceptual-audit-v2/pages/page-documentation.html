Getting Started with Claude Code

A comprehensive guide to setting up your development environment, understanding context management, and building effective autonomous workflows.

Project Structure

Every Claude Code project follows a consistent structure. The key files—CLAUDE.md, PROMPT.md, and PROGRESS.md—work together to manage context across sessions.

Recommended Layout 
my-project/
├── CLAUDE.md          # Project context and constraints
├── PROMPT.md          # Current task definition
├── PROGRESS.md        # Session state persistence
├── src/
│   ├── index.ts
│   └── lib/
│       └── helpers.ts
└── tests/
    └── index.test.ts
Note
The CLAUDE.md file is read automatically at session start. Keep it under 500 lines for optimal context usage.

The Basic Workflow

The simplest effective pattern is the Ralph Wiggum Loop—a continuous iteration cycle where Claude reads from a prompt file and writes results back to state files.

bash 
1# The Ralph Wiggum Loop
2while :; do
3  cat PROMPT.md | claude-code
4  sleep 2
5done
Tip
Add --continue to preserve context between iterations. Use sleep 2 to avoid rate limiting during overnight runs.

The loop continues until manually interrupted. Each iteration reads the current prompt, executes work, and updates state files that persist for the next iteration.

Gotcha
Never use git add -A in automated workflows. This can accidentally commit sensitive files like .env that should never enter version control.

Writing Your Context File

The CLAUDE.md file defines project-wide constraints. It should be concise, focusing on what Claude needs to know rather than comprehensive documentation.

markdown 
1# My Project
2
3TypeScript project using Bun runtime.
4
5## Commands
6- Test: `bun test`
7- Build: `bun build`
8
9## Style
10- Prefer explicit types over inference
11- Use early returns for guard clauses
12- No abbreviations in variable names
Tip
Focus on constraints, not instructions. Tell Claude what patterns to follow, not how to think through problems.
