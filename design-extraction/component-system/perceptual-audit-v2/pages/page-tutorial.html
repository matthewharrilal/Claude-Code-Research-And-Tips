Tutorial

Build Your First Autonomous Workflow

A hands-on guide to implementing the Ralph Wiggum Loop pattern. By the end, you'll have a working autonomous agent that iterates without supervision.

Before You Begin

This tutorial assumes you have Claude Code installed and a basic project structure in place. If not, complete the Getting Started guide first.

Step 1: Create the State Files

The Ralph Wiggum Loop requires two state files: PROMPT.md defines the current task, and PROGRESS.md tracks state across iterations.

markdown 
# PROMPT.md

## Current Objective
Refactor the authentication module to use JWT tokens.

## Constraints
- Do not modify the database schema
- Maintain backward compatibility with existing sessions
- Write tests for all new code

## Context
See PROGRESS.md for current state.
Tip
Keep PROMPT.md focused on a single objective. Multiple goals in one prompt leads to context confusion.

Step 2: Run the Loop

With state files in place, start the continuous iteration loop. Claude will read the prompt, execute work, and update progress on each cycle.

bash 
# Start the Ralph Wiggum Loop
while :; do
  cat PROMPT.md | claude-code --continue
  sleep 2
done
Essence

The loop's power is its simplicity. Each iteration is isolated. State lives in files. Context is refreshed. Errors don't cascade.

Step 3: Verify and Iterate

Let the loop run for 3-5 iterations. Watch the output. Check PROGRESS.md after each cycle to verify state is being captured correctly.
