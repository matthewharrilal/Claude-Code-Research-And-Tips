Coexistence Test 03 — File Tree + Code Snippet

A well-structured Claude Code project uses specific files for context management. Here's the recommended structure:

Project Structure 
my-project/
├── CLAUDE.md          # Project context
├── PROMPT.md          # Current task
├── PROGRESS.md        # Session state
├── src/
│   ├── index.ts
│   └── lib/
│       └── helpers.ts
└── tests/
    └── index.test.ts

The CLAUDE.md file defines project-wide context. Here's a minimal example:

markdown
CLAUDE.md 
1# My Project
2
3TypeScript project using Bun runtime.
4
5## Commands
6- Test: `bun test`
7- Build: `bun build`
8
9## Style
10- Prefer explicit types over inference
11- Use early returns

Claude reads this file automatically at session start and uses it to inform all subsequent decisions.
