Coexistence Test 07 — Core Abstraction + Info Callout

The most powerful Claude Code patterns emerge from a single core principle that governs how state should be managed across sessions:

Core Principle

External state beats internal memory. Files persist across sessions; Claude's memory does not.

// Anti-pattern: relying on Claude's "memory"
const state = "stored in context window";
// Lost on session restart

// Pattern: external state files
await writeFile('PROGRESS.md', currentState);
// Survives context boundaries
Note

This principle applies to all orchestration levels. Even simple single-session workflows benefit from writing intermediate results to files, since sessions can be interrupted unexpectedly.

The externalization of state is what enables patterns like the Ralph Wiggum Loop to work. Without it, each iteration would start from zero knowledge.
