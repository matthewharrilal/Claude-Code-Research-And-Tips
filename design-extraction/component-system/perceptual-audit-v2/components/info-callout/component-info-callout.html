Info Callout — In Context

When setting up your development environment, the first step is to configure the context file. This file tells the agent about your project structure and coding conventions.

Note

The context file is optional but highly recommended for projects with more than 10 files.

Once configured, the agent will use this context to make more informed decisions about code placement and style.
