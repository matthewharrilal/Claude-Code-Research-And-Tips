## EXECUTIVE SUMMARY

The Info Callout component is **functionally competent but lacks editorial soul**. It successfully provides contextual information with appropriate restraint, but reads as a "generic documentation callout" rather than a KortAI-aligned editorial element.

**Key Strengths:**
- Color restraint (grayscale dominant, muted accent) — 4.5/5
- Container treatment (sharp corners, flat, minimal chrome) — 4/5
- Typography restraint (3 treatments only) — 4/5

**Critical Gaps:**
- KortAI alignment (lacks editorial confidence) — 2.3/5
- Spacing (functional but mechanical, no rhythm) — 2.5/5
- Hierarchy (only 2 levels, too similar weight) — 3/5
- Overall energy (utility component, not editorial) — LOW

**Recommendation:** Refine typography hierarchy and inject editorial personality before approval.

## THE 7 PERCEPTUAL TRUTHS

### 1. Typography Perceptual Truth
**"Info Callout typography feels like a margin note in a technical manual — functional, unremarkable, serving information over impression."**

**Test:** Single-weight body text, uppercase label, inline code distinct but muted.

**Anti-pattern:** Bold "NOTE" label, all-caps body text, decorative fonts.

---

### 2. Spacing Perceptual Truth
**"Info Callout spacing feels like a developer's safe minimum — correct but not considered, even padding without intentional variation."**

**Test:** ~24px uniform padding, 12-16px label-to-body gap, standard line-height.

**Anti-pattern:** Cramped padding (<16px), excessive gaps, uneven horizontal padding.

---

### 3. Container Perceptual Truth
**"This container is an editorial margin note — the single-edge accent bar marks territory without enclosing it, like a pull quote in a well-typeset book."**

**Test:** Left border accent only (3-4px), warm cream background, sharp corners, no shadow.

**Anti-pattern:** Full border, rounded corners, drop shadow, gradient background.

---

### 4. Hierarchy Perceptual Truth
**"Info Callout hierarchy is flat and functional — the left accent bar and 'NOTE' label establish attention, but content and label have too similar visual weight."**

**Test:** 2 hierarchy levels only (label + body). Accent bar is primary anchor.

**Anti-pattern:** Bold body text, colored content, competing emphasis, icon + label redundancy.

---

### 5. Composition Perceptual Truth
**"Info Callout is a well-behaved utility element with left-dominant weight distribution — restful energy, no tension, documentation-appropriate."**

**Test:** Left-weighted composition, static energy, predictable rectangle.

**Anti-pattern:** Centered composition, dynamic elements, competing visual weights.

---

### 6. Color Perceptual Truth
**"Info Callout achieves radical color restraint — effectively monochromatic with a single muted accent, feeling like sophisticated editorial choice rather than safe default."**

**Test:** ~99% grayscale, single accent (muted olive/gray border), no saturated hues.

**Anti-pattern:** Blue info color, yellow/red severity colors, pastel backgrounds, multiple accents.

---

### 7. Layout Perceptual Truth
**"Info Callout is a 2-zone vertical stack (label + content) where whitespace alone creates separation — correct but generic, lacking editorial sophistication."**

**Test:** 2 zones clearly defined, no internal dividers, left accent anchors entry.

**Anti-pattern:** Visible dividers, nested containers, competing zones.

---

**"The Info Callout is a helpful aside that prioritizes function over personality — it whispers 'read this' without making a statement about who it is. Competent utility, forgettable identity."**

## NAMED CHARACTER

### The Provident Concierge

**Archetype:** Anticipatory care, professional discretion, quiet preparation

**Why This Name:** The component appears at thresholds — moments before you commit to an action — with quiet, essential context. It doesn't block your path or raise its voice; it leans in with a soft "Before you go..." and hands you exactly what you'll need.

**How to Stay in Character:**
- Warm cream background, never clinical white
- Single-edge accent bar, never full border
- Muted gray label, never bold or colored
- Body text that informs, never demands
- Present once, concisely, then step aside

**Design Mantras:**
- "Before you go, you should know..."
- "The terrace can be brisk. Shall I hold a wrap?"
- "I mention, I don't insist."

## CRITICAL GAPS REQUIRING REVISION

### Gap 1: Typography Lacks Confidence (Score: 2/5 on KortAI audit)
**Problem:** Single-weight gray typography. "NOTE" label is timid.
**Fix Options:**
1. Make "NOTE" label bolder or use serif italic (editorial tension)
2. Use true black (#1a1a1a) instead of gray for body text
3. Add weight contrast between label and body

### Gap 2: Spacing Is Mechanical (Score: 2.5/5)
**Problem:** Uniform ~24px padding with no rhythm variation.
**Fix Options:**
1. Increase top padding to ~32px (create breathing pause)
2. Reduce label-to-body gap to ~8px (tighter coupling)
3. Add more generous bottom padding (~32px) for closure

### Gap 3: Hierarchy Is Flat (Score: 3/5)
**Problem:** Only 2 levels, too similar visual weight.
**Fix Options:**
1. Make "NOTE" significantly different (smaller+bolder OR larger+lighter)
2. Create clear separation between label zone and content zone
3. Consider serif italic for label (editorial voice)

### Gap 4: Missing Editorial Energy (Overall: ~72%)
**Problem:** Reads as generic docs-site component, not premium editorial.
**Fix Options:**
1. Inject confidence through typography decisions
2. Use more decisive accent color (not muted olive)
3. Consider outline text treatment for label
4. The component should feel "designed" not "templated"
