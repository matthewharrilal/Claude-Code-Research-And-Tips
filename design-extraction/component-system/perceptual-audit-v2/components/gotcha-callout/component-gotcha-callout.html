Gotcha Callout — Default
Gotcha

Never run git push --force on shared branches. This rewrites history and can destroy teammates' work.

Gotcha Callout — Warning
Warning

The --hard flag in git reset is destructive. Uncommitted changes will be permanently lost with no way to recover them.

Gotcha Callout — Multi-paragraph
Gotcha

Environment variables set in .env files are not automatically available to subprocesses. You must explicitly pass them or use a tool like dotenv.

This is a common source of "works on my machine" bugs when deploying to production.

Gotcha Callout — In Context

When configuring hooks, you might be tempted to use complex shell scripts directly in the configuration file. This works for simple cases, but quickly becomes unmaintainable.

Gotcha

Inline shell scripts in configuration files are not linted, tested, or version-controlled independently. Extract complex logic into separate script files.

A good rule of thumb: if your hook script is more than one line, put it in a file.
