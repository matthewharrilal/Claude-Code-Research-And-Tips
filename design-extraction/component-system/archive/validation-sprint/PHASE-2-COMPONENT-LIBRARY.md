## KEY DESIGN DECISIONS

### 1. Color Accent System

| Component Type | Accent Color | Meaning |
|----------------|--------------|---------|
| Info | Blue | Neutral information |
| Tip | Green | Positive/helpful |
| Gotcha | Red | Warning/danger |
| Reasoning | Amber | Process/logic |
| Challenge | Purple | Interactive/exercise |
| Card/Essence | Red | Brand/emphasis |

### 2. Typography System

| Context | Font | Weight |
|---------|------|--------|
| Body text | Inter | 400 |
| Labels | Inter | 600 (uppercase) |
| Emphasis titles | Instrument Serif | Normal (italic) |
| Code | JetBrains Mono | 400 |

### 3. Container System

| Container Type | Background | Border |
|----------------|------------|--------|
| Callouts | Tinted cream | Left accent (3px) |
| Essence | Cream | Full border (1px red) |
| Core Abstraction | Dark | None |
| Code | Dark | None |
| Tables | White | Outline (1px) |
| Tasks | Cream | Outline (1px) |

## SOUL TRANSLATION

**Critical principle:** Halftone is ONLY for images. Soul translates to other components through:

1. **Typography Restraint** — Instrument Serif only for emphasis titles
2. **Color Constraint** — Limited palette, red as primary accent
3. **Sharp Corners** — 0px border-radius everywhere
4. **Organized Density** — Clear internal structure
5. **Warm Background** — Cream base (#FEF9F5)

---

## BLINDSPOT CHECK

### What I avoided examining:
1. **Mobile responsiveness** — Components may need adjustment for mobile
2. **Dark mode** — No dark theme variants created
3. **Accessibility** — Color contrast not formally tested
4. **Animation** — No transition states defined

### What I found when investigated:
The family test revealed that components with the same visual DNA (typography, color, corners) feel cohesive even without halftone treatment. The soul transfers through restraint, not decoration.
