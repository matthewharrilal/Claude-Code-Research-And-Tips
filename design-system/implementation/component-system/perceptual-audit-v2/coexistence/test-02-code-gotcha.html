Coexistence Test 02 — Code Snippet + Gotcha Callout

When committing changes, Claude Code uses a specific git protocol. Here's the standard commit workflow:

bash
1# Stage specific files, not everything
2git add src/main.ts tests/main.test.ts
3
4# Commit with descriptive message
5git commit -m "feat: Add user authentication module"
Gotcha

Never use git add -A or git add . in automated workflows. This can accidentally commit sensitive files like .env or credentials.json that should never be in version control.

By explicitly listing files in the git add command, you maintain full control over what enters your repository, even when operating autonomously.
