Understanding Context Windows

Context windows are Claude's working memory. Understanding their constraints is essential for effective orchestration patterns.

Note

Claude Opus 4.5 has a 200k token context window. At approximately 4 characters per token, that's roughly 800k characters of combined input and output.

Tip

Use /context in Claude Code to see your current token usage and available space. This helps you understand when you're approaching limits.

Gotcha

Context window usage is not linear — large files read early in a session stay in context throughout. A 50k token file read at the start affects every subsequent interaction.

Essence

The context window is not storage — it's working memory. Treat it as precious, temporary, and shared between you and Claude.

Challenge

Optimize a bloated context

Take a session where you've read many files and the context is approaching limits. Practice extracting the essential information into a summary file, then starting fresh with only that summary. Compare the quality of results.

Notice how each callout type serves a distinct purpose in the reading experience: Info provides background, Tip offers actionable advice, Gotcha warns of pitfalls, Essence distills wisdom, and Challenge invites practice.
