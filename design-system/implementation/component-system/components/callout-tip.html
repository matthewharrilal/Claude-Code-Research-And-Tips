Pro Tip

Use claude-code --continue to resume from where your last session ended. The agent will reload context from your CLAUDE.md and pick up where it left off.

Editor's Note

The most common mistake is overloading CLAUDE.md with too much context. Start minimal and add only what the agent asks for or clearly needs. Less context produces more focused results.

Workflow Tip

Keep PROGRESS.md as a simple checklist, not a detailed log. The agent only needs to know what's done and what's next. Verbose progress tracking wastes context tokens.

Debugging Tip

If your Ralph loop stalls, check PROGRESS.md for conflicting entries. The agent may be reading contradictory state. Clear the file and restart with a fresh state checkpoint.

Architecture Tip

When designing multi-agent systems, give each agent a single responsibility. A "research" agent should only research. A "synthesis" agent should only combine results. Mixing responsibilities leads to confused context.
