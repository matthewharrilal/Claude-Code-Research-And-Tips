The Core Insight

Feed Stock

"Claude Code is the world's biggest fuckin' ant. Everyone is focused on making their ant run longer... when nature prefers colonies."

Gas Town isn't an agent -- it's a factory for agents. The fundamental shift: stop optimizing a single context window and start building the production infrastructure around it. You're not a programmer anymore. You're the PM of a factory floor.

The 8-role architecture is the heart of the plant. Three processing levels -- Town, Rig, and Human -- contain specialized vessels that transform raw ideas into shipped code.

The 8-Role Architecture

Processing Vessels

Town Level -- Global Singletons

Central Reactor

Mayor -- Chief Coordinator

The primary AI coordinator for the entire town. Receives high-level directives from the Overseer, routes work to appropriate rigs, manages cross-rig coordination. Everything flows through the Mayor.

Model: Opus 4.5 -- Complex coordination requires top-tier reasoning

gt mayor up # Start mayor gt mayor status # Check state gt mayor down # Stop mayor

Deacon

Monitoring

Health monitoring and agent coordination. Watches agent health across the town, handles agent-to-agent handshakes, reports anomalies to Mayor, manages handoff protocols.

Model: Haiku -- Simple health checks

Dogs -- Quality Gate

Quality assurance and safety gates. Reviews work before acceptance, enforces quality standards, guards against regressions. Can reject and rollback work. The safety valves of the plant.

Model: Sonnet -- Quality review requires reasoning

Work flows from Town level down to Rig level -- one rig per project

Rig Level -- Per-Project Instances

Refinery

Decomposition

Receives feature requests from Mayor. Breaks down large tasks into agent-sized atomic work. Assigns to Polecats or Crew. Manages the merge queue with sequential rebasing.

Model: Opus 4.5 -- Task decomposition is architecturally complex

Witness

Observation

Observes and records all rig activity. Watches over polecats, records history and context, provides state to new workers, reports anomalies to Deacon, unsticks drifting agents.

Model: Haiku -- Observation and logging

Human Level -- You

Overseer (You)

Human Boss

Product Manager and decision maker. Define features and priorities, review inbox decisions, approve major changes, provide human judgment. You have an identity in the system, your own inbox, and you can send and receive town mail.

"That's you, Human."

Beads: The Memory Data Plane

Agents suffer from "50 First Dates" syndrome -- they wake up with no memory of yesterday. Beads (225K lines of Go) provides git-backed storage, universal context across all agents, hash-based IDs to prevent merge conflicts, and memory decay through compaction.

Layer 1: Hot Context

  • Current task state
  • Last N interactions
  • Active file references

Layer 2: Warm Context

  • Recent commit history
  • Cross-agent message log
  • Rig state snapshots

Layer 3: Cold Context

  • Full project history
  • Previous run outcomes
  • Archived decisions

Retrieval order: Hot -> Warm -> Cold (only query cold if needed)

The 6 Waves of AI Development

Distillation Stages
Waves 1-3 -- Pre-Factory

Manual to Chat-Based

Traditional coding (1x) through Copilot completions (2x) to chat-based CHOP (5x). The single-ant era. Each wave made the ant bigger, but never questioned whether a bigger ant was the answer.

Waves 4-5 -- Colony Formation

Agents to Agent Clusters

Coding agents like Claude Code (10x) and multi-agent parallel systems (25x). The colony begins to form. Individual agents are powerful; groups of agents are transformative.

Wave 6 -- Factory Scale

Agent Fleets: AI Supervisors Managing Swarms

Gas Town is Wave 6 infrastructure. AI supervisors managing agent swarms. 50x+ productivity. The factory doesn't just use agents -- it IS agents, managing agents, coordinated by agents, with a human PM at the top.

Core Operating Principles

Process Parameters

GUPP

Propulsion

"If there is work on your hook, YOU MUST RUN IT." Work is slung to agent hooks (persistent git worktrees). When an agent starts, it checks its hook. If work exists, execution is immediate.

Zero Framework Cognition

Design

"Claude makes all decisions. The orchestrator executes." All decisions delegated to AI. No heuristics, regex, or parsing in the orchestrator. Infrastructure, not intelligence.

Cattle, Not Pets

Philosophy

Kubernetes thinking applied to agents. Individual sessions are ephemeral. The work graph is persistent. The agent is not the session -- the agent is a Bead with a global address.

Nondeterministic Idempotence

Resilience

Path is fully nondeterministic -- different runs take different routes. Workflow eventually finishes as long as you keep throwing agents at it. The system is resilient to individual component failures.
